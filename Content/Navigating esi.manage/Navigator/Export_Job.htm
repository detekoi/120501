Export Job Definitions

There are two aspects of an Export Job.

  1. Selected Elements - what project items are to be exported.
  2. Projects - which Projects (in the form of a Project Set) are to be included in the Export Job.

 

  1. Selected Elements - Items which are selected will appear in the Export File. These items will be what was defined for the corresponding item - i.e., selecting Production will export all of the Production Segment information from the identified projects.
  2. If any of the Detail items are checked (for AFE, Capital Expenditure or Production), the Line Item information as well as any Details (Schedule Forecast, Field Estimates, Actuals, etc) will also be exported as their corresponding data type (Forecast, Field Estimate, etc.).
  3. All Project Sets in the Budget will be listed on the Projects page of the Export Job. Whichever Project Sets are selected will be included in the Export when the Export Job is run.

 

Note: there are other routes available to Run an Export Job.

  1. From the Export Job itself.
    1. Open the Export Job by double clicking on it in the Navigator
    2. Click on the "Run this Export job" button
    3. Run Export
  2. From a Project or from the Project List.
    1. Click on the "Run this Export job" button
    2. Run Export
  3. Using the esi.manage Integration Service.

When running the Export Job for items 2 or 3, you will be prompted to select an Export Job to use as the definition for the export.

  1. If you are running the export from a Project or the Project List, the Project Sets identified in the Export Job are ignored.
  2. When running the Export Job from the Integration Service, the Project Sets selected in the Export Job define the scope of the Export.
