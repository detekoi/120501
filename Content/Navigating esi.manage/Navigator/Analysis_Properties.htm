Properties Fields

As described in the Analysis Case Overview, companies can define any or all of the Properties within the esi.manage Configuration and decide whether to Lock any or all of the Properties in the Configuration perspective.

 

The Analysis Case is defined on the Properties page and is made up of 7 Sections:

  1. Environment- defines the sensitivity file and evaluation file to be used for the calculations. This allows the user to store and share configurations among many analysis cases and modify them all from one single location. If desired, the settings can remain in the analysis case bay leaving the values set at (none). Note that if you select either a sensitivity file or environment file, some of the settings will be disabled as the values from the corresponding file will be used.
  2. Custom Economic Calculations- in this section you can select the parameters spreadsheet to be used with the spreadsheet economics engine. It may be left blank if spreadsheet economics are not being used.
  3. Data Truncation - defines the Span (number of years) of years to use for the generated Cash Flow, and the Start Date (explicit Point Forward Date or Full Cycle of the Projects). This section also allows the User to define the Economic Truncation method (either Calculated, Asset Economic Limit Date, or No Truncation), whether to use custom user-defined truncation formula (if Calculated), and whether or not there is any Income Loss Carry Forward for tax purposes.
  4. Data Definition - how esi.manage is to generate the Cash Flow; i.e., Forecast or  Blended, Gross, Net, Net Risked; which Price Scenario to use; which Revenue Consolidation to use; whether to apply
  5. Sensitivities - if no sensitivities file is selected, here you specify which Price Scenario to use; whether to apply Escalation; which foreign exchange rates to use (Currency Conversions); if the measures should be scaled according to a user configuredscaling file.
  6. Discounting Options - User defines when the start date for economic discounting; the Discount Rate to apply; whether it is to be Mid Year (only valid with Yearly discounting), and whether to Retroactively apply discounting to cashflows that occur prior to the "Economic Discounting as of" date.
  7. Display Settings - User defines: the granularity (monthly or yearly) of the Cash Flow, and, with monthly data, whether discounting is to be applied Yearly or Monthly; the volume unit or equivalence unit to be used for reporting; and the currency in which to report.

 

Environment

The environment section allows you to pick from the existing sensitivity files and environment files the corresponding settings to be used for calculation. This allows the user to have a separate configuration files independent from the analysis cases and update all related cases from one single location. This section is optional as you can leave the values set at (none) and all the related configuration will be stored within the analysis case making it independent.

Sensitivity File

  • (global) – this option means that you would like to use the file specified globally in the toolbar. If you change the global sensitivity file in between runs to obtain different sensitivity conditions under the same set of parameters.

Note that the parameters under the sensitivity section will be disabled.

  • (none) – use the parameters specified within the analysis case.
  • Specific Sensitivity file – The analysis case will always use the current version of the parameters in the sensitivity file.

Evaluation File

  • (global) – this option means that you would like to use the file specified globally in the toolbar. If you change the global environment file in between runs to obtain different environment conditions under the same set of parameter

Note that the parameters relevant to the environment will be disabled.

  • (none) – use the parameters specified within the analysis case.
  • Specific Environment file – The analysis case will always use the current version of the parameters in the environment file.


Custom Economic Calculations

The models used in esi.manage’s custom economics engine can optionally have a spreadsheet with parameters as inputs to the calculation. Within this section the user is able to select the parameters spreadsheet to be delivered to the custom economics engine.

  • Parameters Spreadsheet – select the preloaded spreadsheet to be used for calculations in the current analysis case. This field is optional and will only have effects on the models that use the spreadsheet parameters feature.

 

Data Truncation

The Cash Flow for the Analysis Case is generated at the Asset Level. The results are then successively aggregated up - to the Project, each successive level of the Fiscal Mode, the Project Set, and finally Analysis Case.

The Analysis Schedule defines the Horizon for the calculated Cash Flow for each Asset - these settings are interpreted for each Asset based on its data.

  • Economic Truncation
    • No Truncation - Allows the cashflow stream defined in Economic truncation Formulas in Configuration to run past its cumulative peak value with no cutoff at the economic limit.
    • Asset Economic Limit Date - This option will use the economic limit date in the asset tab and stop Cash Flow calculations at that point.
    • Calculated - esi.manage will run the analysis case and will locate the date at which the cashflow stream defined in Economic Truncation Formulas in Configuration reaches is cumulative peak value for each Asset. All cashflows after this point will be truncated (for each asset). Note that an asset marked as "cost-only" in the asset tab of Project editor will not be subject to truncation.
  • Economic truncation Formula - By default, this is at the point where the Cumulative Operating Income is at its maximum, however custom formulas can be defined in the Configuration.
  • Analysis Start - The date which defines the start of the Cash Flow calculations.
    • Full Cycle - Starts calculating results for all Projects from the earliest Project State Date of all Projects being analyzed. If two projects start 1 year apart and the analysis case is to run for 10 year span, the project that starts later will only have data for 9 years..
    • Point Forward - Use the Point Forward Date for data acquisition - any data prior to this date is ignored except for the purposes of establishing opening balances for depreciation and DD&A pools.
  • Span (Years) - The number of years from the determined Start Date (Asset specific) to generate the Cash Flow for, in consideration of the Truncate option.
  • Point Forward Date - If Point Forward is selected for Analysis Start, then the Point Forward Date establishes the start date for the Cash Flow.
    • Any Asset data (CAPEX, etc.) prior to this date is ignored.
  • Tax - Toggle to enable/disable carry forward of income loss for tax purposes.
  • Span of Calculation - The extend at which the Cash Flow statement will be calculated to.
    • BTAX only - Cash Flow calculation stops before tax calculations.
    • BTAX and ATAX - Cash Flow calculation includes after tax calculations, but without income.
    • BTAX, ATAX, and Income - Complete Cash Flow calculation.

Based on these settings - for each Asset the Calculation Horizon is determined.

Start Date (first spend or Point Forward Date) to End Date (the latest Start Date + Span), with the possibility that the End Date will be Truncated to an earlier point based on the Economic Limit being reached.

 

Data Definition

The Data Definition section of this page defines how the Cash Flow is to be generated. The settings are effectively the same as would be seen on the individual Production, OPEX, CAPEX reports.

  • Version - Select between Forecast or Blended numbers
  • Modifiers
    • Co. Share - determines whether Working Interest % is included
    • Risked - determines whether risked or unrisked numbers are used
    • Downtime - determines whether asset level downtime is included
    • Curtailment - determines whether group level capacities and downtimes are included
      Note: Before or After Royalties is NOT a choice when running an Analysis case. Economics are always performed on After Royalty cashflows.
  • Blending Options - definitions for dates when the Version is set to blended
    • Use Financial Actuals - selects whether Actuals or Financial Actuals are used when blending
    • Actuals until (aka Last Actuals Date) - If the Production or Capital Definition is to be Blended, this date defines the date when the data should switch from Actuals to Forecast.
    • Field estimates until (aka Last Field Estimate Date) - see Last Actuals date.
  • Reserve Consolidation - Determines which reserve consolidation will be used as configured on the reserves classification page
    • Use Reserve Progression - applies Reserve Progression booking schedule configured in projects.
    • Use Enterprise Assumptions' UOP Reserve Consolidation - when calculating Book DD&A, if the UOP method of depreciation is used, the Reserve Consolidation selected in the Enterprise Assumption UOP depreciation settings will be used instead of the Reserve Consolidation selected for this Analysis case.

 

Sensitivities

The output of the analysis case can be calculated under different assumptions for product price, currency escalation, currency exchange rates and generally scaling measures such as Production, Opex and Capex. This section will only be enabled if the analysis case is not using a sensitivity file or the global options.

  • Price Scenario - esi.manage’s price forecasts have different versions called scenarios. Chose the scenario to be used for economic calculations.
  • Escalation File - all currency inputs in esi.manage’s projects can escalate due to inflation (or other factors) according to an escalation file. Chose the desired file or (no escalation) to be used in the analysis case.
  • Exchange File – Since projects can have their inputs in different currencies a currency exchange file must be provided to report the results in the same currency. Since you can have a virtually unlimited number of exchange files, this can be used to model different scenarios and the sensitivity to currency exchange rates.
  • Scale File – The measures of production, Capex and Opex (fixed and variable) can have a factor applied to them at the time of calculation. Select the scale file or (no scaling) to be applied to this analysis case.

Display Settings

In order to effectively compare the Projects/Assets on an equivalent basis within the Analysis Case, Costs and Revenue need to be generated in a consistent currency.

  • Display - these settings control the display periods in the Analysis, on a monthly or yearly basis. There will be rows for data that is within the Data Truncation period, but outside of the display periods.
    • As described above, the Analysis Schedule defines the evaluation Horizon.
    • The Start Date in the Settings defines the Date for the second line in the Cash Flow Results table:
      • The first entry for any results will be shown as <Start Date.
      • Any results (CAPEX, OPEX, Revenue, etc) that falls into this time Period will be aggregated in this row.
      • For example, using Full Cycle as the Analysis Start method, with a Start Date of Jan 1, 2010, any Project data prior to Jan 1, 2010 will be lumped (in the subsequent Analysis Views) into a single line labelled <Jan-2010.
    • The Period / Duration / Discount table allows different Analysis periods to be defined (the + and - symbols Add or Remove lines from this table).
      • For each Line the Period can be Monthly or Yearly, for a given Duration.
      • While the underlying calculations are always performed on a monthly basis, any values which fall within a Yearly time period will aggregated within the corresponding year in the Cash Flow results table.
      • If the Start Date is not the beginning of the Fiscal Year, all months between the Start date and the beginning of the Fiscal Year will be locked in to be displayed as months.
  • Currency - The currency to be used by this Analysis Case.
    • All Expenditure and OPEX entries, and any calculated Revenue will be converted to this Currency during the analysis.

 

  • Equivalence Unit - Changes what units are being used as the equivalent for production. Set as a default of boe.
  • Volumes - Defines the units that each of your products will be reported in.
  • Real Base Date - The Inflation Reference Date to be used for real value calculations.

Discounting Options

These parameters define how the Discounting is to be applied.

  • Schedule - The Discounting schedule can be set to discount for a specific amount of time, on a monthly or yearly basis.
  • Mid Period discounting will only be applied (if Mid-year has been selected in the Discounting Options) to periods that have yearly discounting.
  • Economic Discounting As Of - This is the Date when Discounting is to be applied.
    • Cash Flow after this date will be discounted, and if the Retroactive option is selected, Cash Flow data prior to this date will be Inflated.
  • Annual Discount Rate - The percent of discount to apply. If Monthly Discounting has been selected for a given period, the Annual Rate will be converted to an equivalent Monthly rate.
  • Modifiers:
    • Mid Year - this option only applies to periods that have Yearly discounting. This option performs the appropriate Mid Period adjustment to the Annual rate.
    • Retroactive - As described above, this option will inflate Cash Flow Data prior to the Economic Discounting As Of date.
