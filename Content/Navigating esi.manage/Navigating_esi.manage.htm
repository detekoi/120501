Navigating esi.manage

There are 3 primary concepts to the esi.manage interface:

Perspectives - a Perspective is a collection of views tailored to a specific task. A Perspective is accessed by selecting the appropriate button from the Perspectives ribbon in the top right of esi.manage.

A Perspective is accessed by selecting the appropriate button from the Perspectives ribbon.

When a particular action is applied against a Project or Project Set; i.e., Report On..., you will be prompted to switch to the appropriate Perspective.

View - a view is a collection of data.  For example, a Project's data is accessed through the Project view, shown at the bottom right of the image below.

For example, a Project's data (content) is accessed through the Project View, shown below at the bottom right.

Navigator/Project Browser - provides access to Projects, or to Saved instances of esi.manage Reports, Project Sets, Analysis Cases, Export Jobs, Import Templates or Peep Import Jobs.  The Navigator/Project Browser views are shown on the left of the image below.  These are visible in all Perspectives that relate to Projects, such as Ranking, Reports, etc.

The Navigator/Project Browser control is shown below at the left.

It will remain visible in all Perspectives that relate to Projects: Reports, Ranking, Analysis, and Projects.

In addition, there are a number of specific User Interface elements.

Buttons - there are two types of Buttons in esi.manage, Action and Toggle.

Action Buttons - these perform a specific action. Hovering over a particular button will provide a fly-by description of the action.

The buttons above the Projects List shown below are all Action Buttons.

The same actions can generally be accessed via the Right Click (context) menu.

Toggle Buttons - these represent 'State'. For example, the button shown in the Project Browser can be toggled.

 

Tables - data entry and display is primarily through tables - such as the Project List shown below.

Columns within Tables can be re-sized as needed.

Columns can also be sorted by selecting the Column label - as shown with the Project column below.

Columns within a Table can also be hidden or re-ordered by selecting the Change Column button.
