Populating the Projects List - Active Selection

For most users, Project Set(s) will be the most convenient way to populate the Projects List in order to view or modify Project Content.

 

More than one Project Set can be selected to be the Active Selection. The current 'Active Selection' is shown in the Navigator by highlighting the Project Sets.

Note: that making a new Project Set the Active Selection 'un-selects' the previous selection.

 

If only 1 Project Set has been selected as the Active Set - its name is indicated in the Projects List title.

There is an alternative route to setting the Active Selection, accessed by the Project Selection button from the top of the Projects Perspective :

 

 

From this dialogue, Project Sets can be toggled on or off. Note that the Project Browser can also be made part of this Selection. Some users (those who work extensively with esi.manage and require the ad-hoc filtering capabilities of the Project Browser) may choose to use the Project Browser as their Active Selection.

There is the option (when only one Project Set is selected as shown above) to Copy the Project Set to the Project Browser. This feature is effective for users who work extensively with esi.manage. This action will copy the configuration of the Project Set into the Project Browser. If the Project Browser is the Active Selection - any changes to the filtering criteria are reflected immediately in the Projects List.
