Filtering Projects

To use the Project Browser, a Budget must be selected as the Source. The Project Browser, shown below, is comprised of a number of Filtering nodes. By default, a Node shown in the list below is not activated until it is Toggled on (the check- mark against the Hierarchy Node). Only then will esi.manage retrieve the items from the database that fall under that node. To make the interaction with the Project Browser as efficient as possible, only enable nodes when you want to filter against their elements.

  • Projects - a list of all Projects in the selected Budget.
  • Hierarchy - your company's Hierarchy, in the tree structure as shown in the Project Details.
  • Product - will expand to the list of products in your configuration.

The remaining nodes will each expand with appropriate entries. The image at left shows the initial configuration of the Project Browser, while the Right shows the state after turning on some Filters.

 

The various Filtering Criteria are applied against the Projects which are part of the selected Budget. The rule for combining Filtering options is:

  • Within a Node - OR - using the image above, projects which have Gas OR Oil as a Production segment would be potentially in the Projects List.  Note this is not an exclusive OR where a filter with more than one of these production streams would be excluded, but rather an AND/OR logic.
  • Between Nodes - AND - satisfy the criteria above, AND have a Forecasted Expenditures in 2015

Items which have been collapsed, but have a select all Filtering applied under them, will show with a Green Check Mark (Region for example).

Items which have select a specific Filtering applied under them, will show with a Double Green Check Mark (Region for example).

The Projects List will refresh with the list of Projects which match the selected criteria. For example, if the Filtering was applied to a specific Expenditure Forecast Year, all Projects that have an Expenditure that occurs in 2015 will be shown.

 

The Right-Click menu from the Project Browser enables you to:

  • Show Active and Inactive Hierarchy Types: View hierarchy elements that are configured to not be Active or shown in the Project Browser.
  • Toggle Selections: toggle between filtered and not filtered, or toggle between static and dynamic filters.
  • Toggle Sub-Items: toggle all of the items within the current sub hierarchy.
  • Clear Sub-Items: removes all filters for Sub-Items.
  • Clear All: removes all filters.
  • Set As Active Selection: sets the selected filter as the Active Selection in the Project Browser.

 

You have the option of creating a Project Set from the current Filtering criteria. Click the create new project set button at top of the Project Browser window . The Project Set is saved for future use and will appear in the Navigator Window.
