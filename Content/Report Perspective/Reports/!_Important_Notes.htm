Important Notes

It is easy to prevent charts from graphing in a number of ways. The following are a list of common errors, and how to avoid (or resolve) them.

  1. Filters exclude all data: If you create mutually exclusive filters, example: Start Date > Jan 1, 2010 and Start Date < Jan 1, 2010, this will obviously exclude everything, and no data will be available.
  2. Filter excludes data sets: If you create a filter for a data set that is above or below the highest/lowest value of the data set, no data will be available.
  3. Null Measure/Series values: If your selected measure has no information for the selected Product, Type, or Basis, the chart will fail. Selecting Gas as a product for a budget with only Oil production values as an example.
  4. Custom Data Binding: If you create custom data bindings, it can severely limit your data output, and will persist through any other changes made. Make sure you remove these if you make changes to the data set or series to avoid skewed data.
  5. Create New Button: DO NOT USE THE "CREATE NEW..." button in the Edit Chart dialogue. A custom dialogue does appear, but this will override your existing data set.
  6. Invalid series units: If you assign multiple attributes to a given series, they must be compatible units. The Y-axis can have up to 2 unit types, using a dual-axis, but the X must only have one.
  7. No Series Assigned: You must drag and drop at least one value to the X and Y axis series. These are not automatically assigned, and must be manually defined before the chart is plottable.
  8. Numbers Calculating Wrong: If an aggregation has been added, it can appear to cause the calculation to be incorrect. These values persist when individual values are changed, and when a chart is saved.
  9. Project Metrics Time Series: If the "Metrics" measure is selected, the reporting period (found in the Dates section) must be set to "Annual".
