The Reports Perspective

The esi.manage Reports Perspective has three elements:

  • The Navigator - where Report templates are stored and organized
  • The Report Parameters Editor - where Report templates are configured and modified
  • The Report Viewer - where the Report results are viewed

The Reports Perspective supports these common functions:

  • Configure, Modify and Delete Report Templates.
  • 'Save' and 'Save As' creates a copy of the template so when the report is run again the configuration remains the same.
  • Save template as Private Report, accessible only for the current user, or Publish Report as a system-wide template.
  • Create new Report folders where new or existing Report template files can be dragged and dropped or copied
  • The ability to control the list of templates in the Project Navigator, each user can choose to see 'Private Files', 'Published Files', or 'My Files' in any combination
  • Report on a Project Set, or Report using the active list in the Project Browser
  • Each Report can be exported to: HTML, PDF, Excel and CSV file formats.

 

Identifying Projects to be Reported On

  1. Select the Budget that contains the Projects to be reported on.
  2. On the Project Selection parameters page, either select one or more Project Set(s) or the Project Browser option.
  3. If you wish to use Static Project Sets (individually selected Projects), ensure the Project Set has been built against the Budget selected for the Report.
  4. Dynamic Project Sets (using Filtering) will be 'applied' against the selected Budget. For example, a Project Set that has been built against the Evergreen Forecast can be selected when reporting against an Archive.
  5. The criteria in the selected Project Set will be used with the Archive, generating a resultant set of Projects which will be Reported On.
  6. The Project Browser option allows ad-hoc Filtering with Reports - whatever filtering criteria is in place in the Project Browser will be applied against the Budget selected for the Report.

 

Special Considerations when Generating a Report:

  1. Depending on your esi.manage user permissions, you may or may not have the ability to Publish your saved Reports to all users.
  2. Your esi.manage User/Project Manager permission settings may affect your access for all projects within esi.manage.  If this is so, when a report is run the Filters Header will show that some Projects have been excluded
  3. These permissions issues can be addressed by your designated esi.manage administrator.
