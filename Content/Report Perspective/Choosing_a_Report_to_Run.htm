Choosing a Report to Run

Reports can be created/run using 3 routes:

  1. Creating a new Report from a default Template.
  2. Running a Report Template (New or Existing) against one or more Project Sets.
  3. Running an existing Report Template.

 

1. Creating a New Report

To create a report template for the first time right-click on Reports, choose New and then Report.

 

The New report browser will open, choose the Report you wish to create and press ‘Finish’.

 

Fill out the Report Parameters for the new Report and press ‘View’ .  New Reports will default to the Project Browser as the list of Projects to run the report against.

 

Alternatively, you can select a Project Set as being the Project Source.

 

2. Running Reports from a Project Set

To Report on a Project Set right-click on the Project Set Icon and choose ‘Report on Project Set(s)’.

Reporting from a Project Set is a feature that is available to all esi.manage Perspectives that have access to the Navigator view. It can generate either a New Report using the data from the Project Set or it can substitute the selected Project Set Projects into a saved report.

Note: One or more project sets can be selected. The project set filters are run against the budgets selected for the report. All projects found by the filters are used in the report.

 

2.1 Using an Existing (Saved) Report

For Existing Reports, choose the ‘Use Existing Report’ option and press ‘Next’.  Then choose your Report from the list of previously saved templates and press ‘Finish’.  This will open the Report template and auto-generate the Report if the parameters have been correctly set.

 

 

 

2.2 Creating a New Report

To create a New Report, choose the ‘Create New Report’ and press ‘Next’. Then choose the Report that is to be generated and press Finish.  After the Report template opens, fill in the Report Parameters, and Press the ‘View’ <Image - Button View>  Button.
