Ranking Metrics

Metrics are a way of evaluating the Projects; what can be selected as a Metric are:

  • Measures - Capital, Production
  • Performance Metrics - Exit Rate, F&D, Lifting Cost
  • Economic Metrics - NPV, DROI, IRR, etc.
  • Attributes - any Project attribute that has been configured to be Rankable.

For most metrics you will need to specify the period that the Metric is to be evaluated over. This duration is comprised of:

  • Milestone - this identifies how the Start Date for the metric should be determined on a Project by Project basis. Some of the options include: Budget Start Date, Custom Milestone, Earliest Project Date, First Onstream Date, etc.
  • Offset - how many Intervals are to be calculated from the selected Milestone for each project.
  • Interval - the periodicity (Days, Months, Years) that is to be used in conjunction with the Milestone and the Offset.

Most metrics can be calculated as Risked or UnRisked.

For metrics that require Revenue, a Price File can be selected.

To support the calculation of the Score for each Project there is the ability to specify:

  • Weight - relative weighting between Metrics.
  • Ceiling and Floor - a way to control the interval used for calculating the Score - metric values that fall outside of the Ceiling or Floor don't skew the results.
  • Target - specify a Target for the metric (i.e., allowed Capital Spend for the current Budget Year). The Gap between Target and Result is reported in the Summary.
  • Score - metrics can be calculated and used for Targets, but not contribute to the Score for each Project. For example, Capital is often a Metric with a Target, but may not be scored for each Project.
