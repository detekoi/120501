Configure PEEP Custom Columns

Esi.manage has the ability to store and edit data that is kept on the Custom Tab in PEEP. This is accessed through the Configure PEEP Custom Columns option in the PEEP Interface Drop down.

To add a new custom column, press the green plus icon located in the popup window. This will create a new line where a description for the column can be added. When adding Custom Columns it is important that the Description used in esi.manage matches the name used in PEEP exactly. This is due to the way in which custom columns are mapped between the two systems.

There is check box for each item that specifies whether or not the data in the columns should shift as the project shifts. For example, something like a custom tax rate or custom price file should not shift with the project as they are tied to specific years. Something like a custom well count, or reserves numbers should be flagged to Shift with Asset as they are relative to the timing of the Project itself.

The sort order will specify which columns appear in which order, with the lowest values appearing at the left end of the table and the highest numbers appearing on the right.

To enable the custom columns tab in the Project Editor, the tab must be turned on in the esi.manage Preferences. The option is found in the Project Editor section as shown above.

Editing or viewing the data in the custom columns is done through the new PEEP Custom Columns tab in the Project Editor. Here data is stored when imported from PEEP and can be edited before sending the project to PEEP with a Synch or and Export action. Additionally, if a case is being created from esi.manage data can be entered into these columns to push to PEEP without the case or data existing beforehand.

Note: Columns must be included in the PEEP mapping file to properly transfer back and forth between PEEP and esi.manage. The mapping for these items requires knowledge of Java RegEx patterns and Java String format notation so it is recommended that 3esi is involved in configuring the mapping file for these new columns.
