Export to PEEP

One or more projects can be Exported to PEEP from the Projects View.

  1. Select the Project(s) to be Exported in the Project View.
  2. Initiate the export action by either invoking the context menu and choosing the Export Project(s) to PEEP action or by clicking the Export Project(s) to PEEP button.
  3.  A dialogue box will open confirming the export action. There are a number of parameters too configure for the export.
    1. Select a Global Parameter from the drop down list.
    2. Select the target Scenario to export to for non-scenario consolidation projects.
    3. Select the Production Parameters that define the production stream(s) you want to export.
    4. Select the Capital Parameters that define the capital profile you want to export.
  4. Click OK to continue or Cancel to abort.

 

The Export to PEEP can also be done at a Project Set level by right-clicking the project set and choosing Export Projects to PEEP.  The dialogue box above will open and proceeds the same as above from there.

Note: The export to PEEP fails for Projects with names that a greater than 50 characters as PEEP only allows case names of 50 characters while esi.manage allows project names of 100 characters.

It is important to ensure the esi.manage economic group is an exact replica of the PEEP model name in order for the interface to run.

Users can specify discount and escalation dates via the (DefaultValue section of the) mapping file. For discount (i.e. real value) dates, choices are as follows:

  • ProjectStart (default) = esi.manange project start date;

  • FiscalYear = Start of the fiscal year of the esi.manage project start;

  • DiscountDate = "Economic Discounting as of" field from Default Economic Properties in esi.manage configuration;

  • None = Leave it alone for existing PEEP cases or PEEP new case default for new PEEP cases

For escalation / inflation date, choices are as follows:

  • ProjectStart (default) = esi.manange project start date

  • BudgetBaseCurrencyDate = esi.manage project's budget base currency date in budget configuration table

  • None = Leave it alone for existing PEEP cases or PEEP new case default for new PEEP cases
