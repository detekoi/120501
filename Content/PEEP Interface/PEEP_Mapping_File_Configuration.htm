PEEP Mapping File & Configuration

In order for the PEEP interface to work, a mapping file must be configured which links the esi.manage measures to the measures and associated fields in PEEP. The setup of the mapping file would normally be done by a 3esi consultant.

The Mapping File specifies what information is transferred to PEEP from esi.manage and vice versa. Specific measures or project information can be toggled and manipulated, such as WI, NRI, Custom Columns/Attributes, Tax, etc..

esi.manage has to be pointed to the correct mapping file. This is done by selecting Window>Preferences>PEEP configuration and opening the folder icon next to 'PEEP to Manage mapping File' and selecting the mapping file.

 

An example of the mapping file can be found in the esi.manage installation files.

esi.manage\3.XX PackageWithPeep\Sample_Files

The mapping file is a csv file with 3 separate columns unless indicated otherwise.

  • Column A - Mapping type.
  • Column B - Name of the type in PEEP.
  • Column C - Name of the esi.manage type. Note the type abbreviations set in the configuration have to be used in this column.

Multiple PEEP Configurations

esi.manage supports multiple PEEP environments. Users can now define which PEEP ODBC data sources they use, and can quickly select between them. The current source is displayed in the esi.manage status bar (which refreshes on every user initiated PEEP action).

By default esi.manage will look for file called esi.manage.peepenv placed in the same directory as the esi.manage executable (esi.manage.exe). Alternatively, users can specify a different location and different file name by using a -D3esi.peep.environments directive in the esi.manage.ini file e.g:

The format of the PEEP environments file resembles that of Windows INI file. It has a mandatory section name enclosed in square brackets, with subsequent lines in the format peepOdbcSource=value.

There could be maximum on 7 peepOdbcSource lines in one section. One for each ODBC source specified in PEEP Preferences. Their names match those in PEEP Preferences, namely:

  • Models
  • Cases
  • Global Parms
  • Prices
  • Currency
  • Reports
  • Wells

esi.manage supports multiple PEEP environments and multiple models. The mapping file can now supports an "include" statement: any line that starts with an “@” sign. It is assume that what it follows an @ sign is a relative or absolute path to another mapping file. Include statements can be nested without limits.

In order to support multiple PEEP environments and multiple models, two special macros have been introduced which can be used in include statement:

  • $PEEP_ENV$ (resolves to Mapping_File_Location \ PEEP_ENV)
  • $PEEP_MODEL$ (resolves to Mapping_File_Location \ PEEP_ENV \ PEEP_MODEL)

Each model can have a separate mapping file.
