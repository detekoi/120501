Offsets

Offsets allow users to apply delays (measured in days) between activities with dependencies on each other.  Two types of time offsets can be specified for each measure:

  1. Desired Offset – This is the intended amount of time the user would like to see applied between measures
  2. Actual Offset – This is the actual amount of time being applied between the scheduled activities that have been planned

Offset values can only be entered into the ‘Desired Offset’ column.  These values can then be applied to the ‘Actual Offset’ values, and thus applied between the scheduled measures by using the ‘Match Desired Offset’ button.  

Match Offset Date - To change the Desired Offset date to match the Actual Offset date, highlight the desired dependency by clicking on it and select “Match Actual Offset”.

In order to change the Actual Offset date to match the Desired Offset date, follow the same steps as matching offset date but instead of selecting “Match Actual Offset", select “Match Desired Offset”.

 

Synchronize Dependency Offsets

To synchronize dependency offset dates of all measures (change Actual Offset to match the Desired Offset), select “Synchronize Dependency Offsets”.

During this process, esi.manage synchronizes the offset date accordingly and shifts the dates as desired.  See below for before and after synchronization. Note that esi.manage updates the columns accordingly.
