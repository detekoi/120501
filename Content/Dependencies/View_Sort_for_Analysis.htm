View & Sorting for Data Analysis

Data within the Dependencies tab can be viewed in various ways. Either from a complete stand-point (all listed measures and dependencies) or by sorting based 3 different data criteria:

 

  1. Reference Date – This is the earliest date of the measures displayed. By changing the refrence date all measure dates will shift in accordance to the current actual offset. Once the dependencies tab is saved or the user navigates to a different tab within the project editor the date changes will become persistent
  2. Analysis Case –This correlated to any existing Analysis cases the user has created against this project data. Any measure exports or modifiers in the Analisis case will be reflected on the data
  3. Production End Dates – Allows the user to specify how the data is viewed:
  1. Technical Limit Date: This refers to the Offstream date, where Production ends as a result of hitting an Offstream date or Offstream rate.
  2. Asset Economic Limit Date:  This refers to the user input or calculated asset economic limit found in the reserves section of the production tab. Note: calculation using the economics engine can be triggered using the “Update Asset Economic Limit” function in the project toolbar.

*Specifying these criteria will allow the measures data in the dependency table to be sorted and displayed more efficiently for different workflows and individual focuses
