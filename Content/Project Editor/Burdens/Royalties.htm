Royalties

  • Royalties are accessed through the burdens tab of the project editor.
  • They are stored as a time-vector at per-ASSET, per-PRODUCT level which allows for modelling various Royalty calculations.
  • Royalties for each product and asset are captured for each price scenario.
  • Sliding scale or price triggered royalties can be configured for a particular economic group (as a function of jurisdictions).

 

There are several buttons to support Royalty Configuration:

  • Show Years with Manual Data only - this will filter the table to rows with User entered data.
  • Find a Specific Date - will expand the table at the specified Date.
  • Copy and Paste - Copy Data to, or Paste From the clipboard (see below).

 

Configuring Royalty Percentages

  1. Select an Asset from the drop down list.
  2. Select a Product from the drop down list.
  3. Enter the Royalty percentages against the different price scenarios.
  4. When a Value is entered, it will be used for all future time periods (for that royalty type) until another User Supplied value is found.
  5. Manually entered values appear in a white cell, while System generated values appear in a coloured cell.
  6. The Pin All Burdens for this Asset option will prevent the Royalty Percentages from being moved when an Asset or Project is Date Shifted.

 

Copy and Paste Royalties

  1. Data copied into the royalties table from excel must be for the whole row and include the first column in the correct date format. The royalty values must be formatted as a %.
  2. Therefore, it is necessary to copy either a single row or a whole years row from the royalty table to excel in order to get the correct format in excel.
  3. Then create your entries in excel using the copy, fill down etc.
  4. Paste the entries as whole rows (including the date column) back to the royalties table in esi.manage.
