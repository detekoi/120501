Asset Scaling

 

This is a site specific feature so must be switched on in Configuration >System >Miscellaneous >Site-Specific Features.

Asset Scaling will allow companies to Scale the contribution of Assets within a Project. Where assets represent outcomes in a project this is the probability of that outcome being achieved. When running normal economics, the user will let the default behaviour (apply scale) be used. In economics, however, the Scale will be applied late (on outputs) vs early (on inputs). Asset Scaling modifier is shown in the Analysis Case Editor and Default Economic Property Configuration.

 

 

 

When economic scale factors are being applied, the all asset-level cash flow elements are multiplied by the corresponding asset scale as the last step of calculation except for the following which are not scaled:

  • production quality
  • effective price
  • effective price equivalence
  • custom

 

 

Set an Asset Scale

Asset Scaling will be set within the asset tab of a project, after enabling the feature within the Site Specific features, the box in red below will be populated. This box is what will define the scale applied to the asset throughout esi.manage.

 

Common Issues

The feature is intended to support a workflow where every asset is entered multiple times, once for each possible outcome. The asset scales then serve as an outcome weight factor.When working in this way, beware of the following implications:

  • Measure-level scaling is incompatible with all group-level and non-linear concerns such as production curtailment, tier pricing, royalties, etc.
  • Economics-level scaling is compatible with asset-level non-linear concerns, but is incompatible with higher-level concerns (e.g. hierarchy-level spreadsheets, UOP deprecation, curtailment, etc).
  • When analysis case asset-scaling in on, all economic outputs at the asset level will represent fractional assets.
  • All measure calculation outputs will also only show fraction values for each asset.
