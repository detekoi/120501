Project  Overview

 

  1. Description: Project Name.
  2. Code: If a project code was assigned when the project was added, this will be reflected here.
  3. Status: Reflects the stage the project is at. Default options include Approved, Budgeted, Completed, In Progress, Inventory, and Proposed.
  4. Manager: The assigned project manager for the project. This can be changed by selecting a new manager from the "Project Manager" pick list in the second column of fields in the editor below. The Project Manager is the main security level, and largely determines the visibility and ability of users to read and/or write to the project. See the User/Project Manager Access page for more details on using Project Managers to control security, or the User Roles section for more details on access rights.
  5. Location: Text field for the geographical position of the project.
  6. Template Type: used for Source Template Projects to control how the Virtual Projects will be created, either individually or as Assets grouped by year.
  7. Source Template: if the Project is a Virtual Project, this indicates the Source Template project.
  8. Start Date: the earliest Start Date within the Project.
  9. Created By: Log of who initially generated the project.
  10. Created Time: Log of when the project was initially generated.
  11. Comment: Text field for any extra information that should be noted about the project that was not included in the standard fields.

 

A Projects Start Date is used in several places throughout esi.manage, such as:

  • Template Scheduler
  • Date Shifting
  • "Earliest Date" in Economics
  • Integration with esi.schedule
  • Clone Asset

The logic used to determine the Start Date for a Project is outlined below.  An asset's start date is taken to be the earliest date from among the following items:

  • CAPEX header and detail start dates
  • AFE detail start dates
  • production forecast onstream dates
  • production non-forecast detail start dates
  • If none of these dates are present, the asset's start date is taken to be the containing budget's start date.
  • OPEX, spud, rig release, ownership, reserve booking dates and any other dates are all expressly ignored for this calculation.
  • A project's start date is the earliest of its assets' start dates.
  • If a project has no assets, then its start date is the containing budget's start date.
