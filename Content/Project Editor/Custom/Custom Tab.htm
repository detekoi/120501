Custom Tab

The costume tab woks in conjunction with esi.manage's Measure Allocation features for the custom economics engine. Within the custom tab you are able to select a group of projects based on a hierarchy attribute to group projects and execute the logic in the specified model.

Possible use cases include but are not limited to:

  • Cost Allocation for Facility Hubs.
  • Royalties Allocation for ring fence fiscal regimes.
  • Any calculations that requires measures to be aggregated at a specific level before individual asset level calculation.

Parameters

  • Name: - a given name to the allocation model
  • Asset: - the asset that will group all the project conforming the group
  • Group: - The hierarchy attribute that describes the group
  • Model: - The excel file with the custom Allocation Model
  • Parameters: - An optional parameters file to pass to the model

Allocation Model

The allocation model adheres to the same format and rules as the regular custom excel economic models. See Custom Economic Calculations

The main difference is that the Allocation Models have 2 main tabs for performing the calculations and a new row type called INN:

  • Group: - values brought into the model with the Type(column A) INN will be the consolidation of all the projects described by the group attribute. The row type IN its still available and will bring in the values from the project where the model is attached.
    In the group tab you should perform all the high level calculations that will the be sent to all assets that are members of the allocation group.
  • Member- calculations in this tab are executed after the group calculations and repeated for each asset of the group. In other words, the Group calculations are done for the asset where the model is attached and the Member calculations are done for all assets whose project has the hierarchy attribute describing the group. You are free to point the calculations in the Member sheet to values in the Group sheet.
