Ownership Overview

  • The Ownership Tab provides the function to add time-variant working interest.
  • The Ownership tab allows one to input WI% by project or by asset (subset of a project) and for all measures or individually (Expenditure types, Products or OPEX).
  • By entering data into a cell, esi.manage will automatically populate the subsequent months or years with the same number.
  • WI% is inherited down the Ownership levels and down the Measures. For example, if the project level had ownership configured, the asset level would automatically inherit the WI% from the Project level unless it was manually overwritten. Likewise, WI% configured in All Measures would feed down to the other measures who would inherit the data. Inherited WI% is denoted by a tinted shading in the WI% fields, see below.
  • When data has been written to the ownership tables, a pencil icon will appear. See below, next to Asset for example.
  • Ownership can also be configured across multiple projects that sit in the same hierarchy group. This is done in the default Ownership page of the Configuration perspective.

In the screenshot above, there are several pieces of information can be taken away from it besides the obvious meaning of the values shown.

  1. Because Bakken Base Well 1 is highlighted yellow, its value is being inherited from the level above it.  In this case, the only level above Asset/All Measures is Project/All measures.  This means that at the highest possible level, on the Project/All measures screen, there is a value of 20% entered.  If you were to create a new asset then, it would automatically inherit this 20% WI%.
  2. There are no pencils next to the measures (Expenditure, Fixed OPEX, Production), so you can assume that the values entered on this page will be inherited by all the levels below it without being changed.  This means that for each measure associated with each asset, it will have the WI% displayed here for the asset it is associated with.
