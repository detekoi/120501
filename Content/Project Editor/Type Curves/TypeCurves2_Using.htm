Using Type Curves

In summary, there are four major steps that must be completed before the ultimate purpose of a Type Curve can be realized. They are:

  1. Creating Type Curves: this is done at the budget level, and establishes the base curves applicable to all projects in the budget.
  2. Creating Patterns: this is done at the project level, in which modifiers are applied to existing Type Curves to create customized Patterns.
  3. Creating Assets: this is a necessary step prior to creating Recurrences, which are tied to specific Assets in a project.
  4. Creating Recurrences: this is done at the project level, in which modifiers are applied to existing Patterns to create customized Recurrences that represents the Production and Expenditures of designated assets in the project.

When these steps are completed, Production analysis and Expenditure analysis can be generated without the need to create Productions and Expenditures within the project.

 

Type Curves

Type Curves for Production, Expenditures, and Royalties are created in the Project Perspective, with the processes much like creating Production, Expenditures, and Royalties in the Project Editor. When Type Curves are created, they reside in the budget level. All curves created in a budget can be accessed by all projects within that budget. Patterns

Within the Project Editor, a Type Curve Pattern can be created using an existing Type Curve. The resulting Pattern will be based on the Type Curve, as well as any modifiers applied to the Type Curve as part of creating the Pattern. In the example below, Type Curve 1 was used to create Pattern 1, a Production Pattern that has a count of 2 as a modifier. As a result, Pattern 1 is identical to two Type Curve 1 placed one after the other, with the curve spanning over 24 months.

 

Recurrences

Within the Project Editor, a Type Curve Recurrence can be created using an existing Pattern. The resulting Recurrence will be based on the Pattern, as well as any modifiers applied to the Pattern as part of creating the Recurrence. In the example below, Pattern 1 was used to create the first Recurrence on the list, a Production Recurrence that has an occurrence of 2 as a modifier. As a result, the Recurrence has an identical curve to Pattern 1, with the values of the Recurrence curve double that of the Pattern 1 curve.

 

With Recurrences in place (in the case of the example, a Production Recurrence), the project now has an asset with Production values tied to it, without adding anything to the Production tab. When a Production Analysis is created for the project in the example, the Production curve from the analysis result is identical to the Recurrence curve.
