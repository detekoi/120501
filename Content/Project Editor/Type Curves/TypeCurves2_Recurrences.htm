Recurrences

The next step of making existing Type Curves usable is to create Recurrences in the project-of-choice. Recurrences are created by taking an existing Pattern and apply modifiers to the Pattern curve in order to manipulate the magnitudes of the curve values, the start date, and any delays to the modifiers to the Recurrences. Each Recurrence is tied to an asset in the project-of-choice, therefore there must be an existing asset in order to be able to create a Recurrence curve.

There are two types of Recurrences: Production and Expenditures. A Production Recurrence is based on a Production Pattern (which in turn contain a Compound Production Type Curve), whereas an Expenditure Recurrence is based on an Expenditure Pattern (which in turn contain an Expenditure Type Curve).

In order to create Recurrences, there must be existing Patterns residing in the same project-of-choice, as well as existing assets in the project. For this example, the following Patterns are created for the example project (the Pattern names describes the actual types of Patterns).

 

To create Recurrences, open an existing project where the Patterns are created, and navigate through the Project Editor to find the tab named "Type Curves". Within the 'Type Curves' tab, click on the 'Recurrences' tab to open the page.

 

Create a new Recurrence by selecting the 'Add Recurrence' button on the right side of the page. A new row appears in the table. The caution symbol reminds the user to provide all necessary Recurrence information to the table.

 

The user can specify the following details for the new Recurrence:

  1. Asset: this is an existing asset that the Recurrence created is tied to. Like adding Expenditures and Productions in the Project Editor, each item is assigned to an asset. When an Expenditure Analysis or Production Analysis is generated for a given asset, any Expenditures or Productions from Recurrences belonging to the asset is pooled and summed up into the analyses under that asset.
  2. Pattern: this is an existing Pattern that is either a Production Pattern or an Expenditure Pattern. The Pattern provide the basic curve that is further modified to become a Recurrence.
  3. Occurrences: this integer value quantifies the number of times the existing Pattern curve is "stacked" onto the Recurrence curve. Occurrences are combined with the Cycle Time modifier to specify the amount and frequency of "stacking" Patterns onto the Recurrence curve.
  4. Start Date: this sets the base start date of the Recurrence curve that is modified by the delay applied to the existing Pattern.
  5. Cycle Time(days): this integer value quantifies the days of delay between each Occurrence "stacking" event.

 

Asset

To assign an asset, double click on the Asset cell, and a dropdown menu listing the existing assets appears. For this example, the asset named "Asset 1" is to be selected as the asset that this Recurrence belongs to.

 

Pattern

To assign a Pattern, double click on the Pattern cell, and a dropdown menu listing the existing Patterns appears. For this example, the Pattern named "Production Pattern" is selected as the base Pattern for this Recurrence.

 

Start Date

Unlike a Pattern, the start date of a Recurrence curve can be explicitly specified by the user. The start date of the Recurrence is similar to the default start date of a Pattern, in which both start dates are subject to the delay modifier of the base Pattern. For this example, when the Recurrence start date is set as the same default start date of the Production Pattern (July 3, 2014), the resulting start date of the Recurrence curve is the same actual start date of the Production Pattern (June 3, 2014).

Occurrences and Cycle Time

The shape of the Recurrence curve can be customized by "stacking" the base Pattern curve on top of itself repeatedly. The Occurrences value determines the number of times such "stacking" occurs, while the Cycle Time value determines the number of days after the start date before the stacking occurs. To demonstrate how Occurrences work, the following steps shows the transformations of the Recurrence curve as a result to various Occurrences and Cycle Time settings.

When the default Occurrences and Cycle Time values are applied (1 and 0, respectively), the resulting curve is identical to the base 'Production Pattern' curve with initial oil production of 240 bbl/d in both June 3, 2014 and June 3, 2015, and final oil production of 120 bbl/d in both June 2, 2014 and June 2, 2015.

 

When the Occurrences value is changed to 3, the resulting Recurrence curve has tripled initial oil production (from 240 bbl/d to 720 bbl/d) as well as tripled final oil production (from 120 bbl/d to 360 bbl/d), while the same curve shape is maintained. This shows that at each point in time, the y-axis Production value is increased 3-fold. However, unlike the scaling factor applied to Pattern curves, the mechanism of the increase is addition, not multiplication. The application of Cycle Time demonstrates this point.

 

When the Cycle Time value is changed to 30, the resulting Recurrence curve formed 3 distinct "layers" of the base 'Production Pattern' curve.

 

The appearance can be described by the steps in which this Recurrence curve was constructed:

  1. For the first 30 days, the curve follows the base 'Production Pattern' curve, starting with an initial oil production of 240 bbl/d. This is Occurrence #1.
  2. After the first 30 days, which is the same as the number of days specified in the Cycle Time modifier, an additional base 'Production Pattern' curve is added on top of the existing Production rate after 30 days. At day 31, the Production rate is equal to the Production Pattern at day 31 plus the initial Production rate of the Production Pattern. This is the start of Occurrence #2.
  3. After another 30 days, another Occurrence cycle is reached. An additional base 'Production Pattern' curve is added on top of the existing Production rate after 60 days. At day 61, the Production rate is equal to the Production Pattern at day 61, plus the Production Pattern at day 31, plus the initial Production rate of the Production Pattern. This is the start of Occurrence #3, the final Occurrence of year 2014.
  4. With no more Occurrences, the Recurrence curve continues its decline with each passing 30 days of Cycle Time until the Compound Production Type Curve duration of 12 months is reached.
  5. After 12 months, the Recurrence curve increases by 120 bbl/d of oil production as occurred in the 'Production Pattern' curve.
  6. After 30 days, an Occurrence cycle is reached. An additional base 'Production Pattern' curve is added on top of the existing Production rate after 12 months and 30 days. At 12 months and 31 days, the Production rate is equal to the existing Recurrence curve Production rate plus the Production Pattern at the end of 12 months. This is the start of Occurrence #2.
  7. After another 30 days, another Occurrence cycle is reached. At 12 months and 61 days, the Production rate is again equal to the existing Recurrence curve Production rate plus the Production Pattern at the end of 12 months. This is the start of Occurrence #3, the final Occurrence of year 2015.
  8. At the start of 2016, the first of the three "stacked" Patterns stops Production. Due to the delay applied by 30 days of Cycle Time, the second "stacked" Pattern stops Production 31 days into 2016, and the third and final "stacked" Pattern stops Production 61 days into 2016.

 

Other Modifiers

The user can also specify the Reserve Consolidation of the Compound Production Type Curve used to generate the base Pattern curve of the Recurrence, if different curves are created within different Reserve Consolidations of the same Compound Production Type Curve. In the case of the example, only the "TestInc - 1P" Consolidation contains a Compound Production Type Curve, therefore it must be set at that Consolidation.

If royalty is applied to the base Pattern of the Recurrence as a Secondary Type Curve, the following must be done:
