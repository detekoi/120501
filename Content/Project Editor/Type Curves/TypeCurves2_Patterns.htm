Patterns

The first step of making existing Type Curves usable is to create Patterns in the project-of-choice. Patterns are created by taking an existing Type Curve and apply modifiers to the Type Curve in order to manipulate the start date, magnitudes of the curve values, and any repeating of the Type Curve.

There are two types of Patterns: Production and Expenditures. A Production Pattern can contain a primary Production Type Curve, as well as multiple secondary Type Curves (which can be Production or Royalty Type Curves). An Expenditure Pattern only contains a primary Expenditure Type Curve.

In order to create Patterns, there must be existing Type Curves under the budget where the project-of-choice resides. For this example, the following Type Curves are created for the example budget (the Type Curve names describes the actual types).

 

To create Patterns, create or open an existing project in the budget where the Type Curves are created, and navigate through the Project Editor to find the tab named "Type Curves". Click the tab to open the page.

 

Create a new Pattern by selecting the 'Add Pattern' button on the left side of the page. A new Pattern with the default name "New Pattern" appears in the table on the left side. Double click on the name to re-name the new Pattern. For this example, two new Patterns are created (named "Production Pattern" and "Expenditure Pattern").

 

With the new Pattern selected/highlighted, select the 'Add Detail' button on the right side of the page. New items appear in the first row of the table on the right side. The caution symbol reminds the user to assign a Primary Type Curve to the Pattern.

 

The user can specify the following details for the new Pattern:

  1. Primary Type Curve: this is an existing Type Curve that is either a Compound Production Type Curve or an Expenditure Type Curve. This is the only required detail for a Pattern to be created, and must be specified before the Secondary Type Curve.
  2. Secondary Type Curve: this is an existing Type Curve that is either a Derived Production Type Curve or a Royalty Type Curve. This is only applicable if the Primary Type Curve is a Compound Production Type Curve.
  3. Delay (days): this value specifies the start date of the Pattern curve, by assigning the number of days into the future (of the current date). A negative value can be assigned to make the start date some time in the past.
  4. Scale: this value is a multiplier to the magnitudes of the values that comprises the Pattern curve.
  5. Count: this integer value specifies the number of times the Type Curves in the Pattern would appear. For count values greater than 1, the Type Curve would simply appear consecutively, with the time duration of the curve extended accordingly.

From this point on, modifiers are applied to customize the base Pattern curves created from adding the Primary and Secondary Type Curves.

 

Delay

For a new Pattern, the default start date of the curve is the date when the details of the Pattern are added. To specify a start date different from the current day, whether it is from the past or the future, a delay can be applied to change it to a desired start date. The delay value is the number of days to add or subtract from the default start date to shift to the desired start date. For this example, 'Production Pattern' was created on July 3, 2014 as shown in the screen shot above. In order to set the start date to June 3, 2014, a delay of -30 is entered to the delay cell (June 3, 2014 is 30 days prior to July 3, 2014).

Scale

The new Pattern curve inherits the Production or Expenditure values from the Primary and Secondary Type Curves added to the Pattern. While the overall shapes of the Pattern curves as inherited from the Type Curves cannot be modified, the magnitudes of values in the Pattern curves can be changed as a whole by applying a scaling factor. The scaling factor (the user-defined number entered in the scale cell) is a multiplier to all values in the Pattern curves. This effectively keeps the curve the same while multiples all values in the y-axis of the curve by the scaling factor (however, the appearance of the curve may change due to changes in the y-axis range). The Pattern curves can be scaled in the following ways:

  • Scaled up by applying a scaling factor greater than 1
  • Scaled down by applying a scaling factor less than 1
  • Leave unchanged by leaving the scaling factor at 1 (default value)

For this example, when the scaling factor is set at the default value of 1, the oil production of 'Production Pattern' starts at 100 bbl/d, and ends at 50 bbl/d:

 

When the scaling factor is changed from 1 to 3, the initial oil production triples to 300 bbl/d, and the final oil production also triples to 150 bbl/d:

 

Count

At this point, with the Primary Type Curve, Secondary Type Curve, delay, and scale applied, the result is an intermediate Pattern curve that contain single instances of the customized Primary Type Curve and Secondary Type Curves that begins and ends at the Primary Type Curve duration boundaries. The duration of the Pattern curve can be extended further by applying the count, which is a multiplier value for the number of consecutive times the intermediate Pattern curve is repeated. Type Curve Dependencies

The user can also specify “dependencies” between different Primary Type Curves within a Pattern, Similar to dependencies of true measures, Primary Type Curves can be assigned dependency relationships such that a Type Curve would not begin until the end of the preceding Type Curve. Also similar to true measures dependencies, dependencies can be established between the two different types of Primary Type Curves (Production and Expenditure).

The Type Curve dependencies are specified using the following four buttons (only applicable to Patterns with multiple Primary Type Curves):

  • Move Up: the selected Primary Type Curve switches position with the Primary Type Curve immediately above (does not apply to the first Primary Type Curve in the list)
  • Move Down: the selected Primary Type Curve switches position with the Primary Type Curve immediately below (does not apply to the last Primary Type Curve in the list)
  • Move Left: the selected Primary Type Curve loses dependency on the Primary Type Curve immediately above (does not apply to the first Primary Type Curve on the list, or any Primary Type Curve not already dependent on another)
  • Move Right: the selected Primary Type Curve gains dependency on the Primary Type Curve immediately above (does not apply to the first Primary Type Curve on the list)

While Move Up and Move Down changes the order of the Primary Type Curve listing in a Pattern, Move Left and Move Right changes (turns on/off) the dependency relationship between Primary Type Curves.

Other Modifiers

The user can also specify the Reserve Consolidation of the Compound Production Type Curve used to generate the Pattern curve, if different curves are created within different Reserve Consolidations of the same Compound Production Type Curve. In the case of the example, only the "TestInc - 1P" Consolidation contains a Compound Production Type Curve, therefore it must be set at that Consolidation.

If royalty is applied to the Production Pattern as a Secondary Type Curve, the following must be done:

  • Select a Price Scenario where the Royalty Type Curve has assigned royalties to
  • Check the 'Apply Royalty' box

For this example, the Royalty Type Curve included as a Secondary Type Curve has a 20% royalty for oil under the "Royalty - Medium" Price Scenario. With "Royalty - Medium" and 'Apply Royalty" selected, the resulting 'Production Pattern' curve has a 20% drop in production compared to previous.
