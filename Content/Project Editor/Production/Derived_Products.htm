Derived Products

Derived Products is a mechanism for defining a Production segment (the Derived Product) as a Ratio to another Production segment (the Primary Product). There are several aspects of the model which should be considered when creating a Derived Segment.

  • A Derived Segment is associated against an Asset, and defined in terms of a Primary Product.
    • The Ratio applies to all Production segments of the Primary Product, which are associated with the same Asset.
  • The Shrinkage for the Primary Product segment(s) dictates when the Derived Product's production is reported.
    • Shrinkage > 0 - the Derived Product's production is only reported for Sales Definitions.
    • Shrinkage = 0 - the Derived Product's production is reported for all Production Definitions.
  • There are two Ratio models:
    • Fixed - the Ratio does not change with time.  This can be achieved by entering a single ratio into the derived production ratio table, and selecting Constant interpolation.
    • Variable - there is a time series of Ratios that can be created. This is done by entering >1 ratio in the derived production ratio table.
    • Note that Shrinkage is not time-variant, using the Variable Ratio model with a non Zero shrinkage can present numerical differences when comparing results generated with different Definitions.
  • There are two types of Derived Products
    • By-product - a secondary product derived from the primary product, which is affected by the primary products shrinkage.  By-products will always have zero
      Gross "unshrunk" production.  By-products will not appear in Production analysis unless Production definition is "shrunk".
    • Associated product - A production stream that is derived as a ratio from another stream, and is unrelated to the primary products shrinkage.  Associated products
      can have non-zero gross "unshrunk" production.
  • The onstream date of the derived product can be controlled in two ways with the use of the Start Event parameter:
    • Onstream - The start date for production will be the related to the onstream date of the primary product. Variation to the onstream date of the primary product can be achieved through the use of the offset.
    • Calendar - The onstream date of the derived production will start on the date specified in the field Start Date. Note that offsets still apply and will reference the Start Date
  • Defaults

    • Use Defaults? toggle can be used to automatically fill the derived product ratio(s) based on defaults that are set against a hierarchy element (eg. Region) in the Configuration.

    • See the Defaults section for more information on setting derived production defaults in the Configuration section.

 

The Variable model is designed for situations where you want to report Yield, which may change over time with facility upgrades. In most situations, the Fixed model is appropriate and should be configured at a Ratio consistent with the Primary Product's shrinkage.

Creating A New Production Segment with a Derived Product(s)

  1. Create the Primary Segment - in the screen shot below this is the Gas segment.
  2. For the Derived Segment - select the Asset and Product, and choose the Decline Type as Derived. The derived product line will appear in green font.
  3. Select the Production Rate units for the Derived Segment. This selection does not specify how the Ratio will be defined.
  4. esi.manage supports the creation of one derived product from another derived Product, for example defining Gas as a Ratio to the Primary Oil production, and then Deriving Condensate from the Gas derived segment.
  5. Click away from the derived header line and then click back into it to access the derived details pane in the bottom left hand corner.

 

In order to define the ratio(s), select the row you want to edit, the details will show up in the bottom left pane.

 

There are two approaches to defining the Ratio - Fixed or Variable.

A Fixed Ratio is a single value that will be applied to any Production of the Primary product, for the Asset.

If Shrinkage has been specified on the Primary Segment - the Ratio should likely be consistent with Shrinkage, in terms of the Units Conversions in your database.

 

Variable Model

As described previously, the Variable model is designed for situations where you want to represent Yield changes.

 

  1. A Variable Ratio is a time series of Fixed Ratio's.
  2. The Offset defines when changes in the model occur. The 0 entry (shown above) would indicate the starting Ratio.
  3. The Interpolation method defines the shape of the ratio curve between two successive entries.
