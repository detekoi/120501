Production Downtime

Downtime, along with production volumes and shrinkage, are entered in the production editor screen.

Click on the Downtime tab at the top of the upper table in the production editor screen.

 

 

  • Asset - Select the Asset the downtime will be applied against
  • Set the Start and End dates.
    • If no dates are provided, the downtime will be applied to the entire length of the asset.
    • If a start date is provided, but no end date, the downtime will be applied to the entire life of the asset after that date.
    • If an end date is provided, but not start date, the downtime will be applied the entire life of the asset, up until that date.
    • If both start and end dates are provided, the downtime will be applied within those dates.
  • Down percent - Enter the percentage of downtime for the given period
  • Scheduled (Y/N) - Select whether the downtime is scheduled or unscheduled. unshceduled downtime is applied first, and unscheduled downtime second, by default.
  • Pinned (Y/N) - If a downtime entry is "pinned", its date will not change if the Project or Asset dates are shifted.
  • Reason - Open area for entry to capture the purpose or reasoning of the downtime.

  • Enter the Percent that the production will be reduced to during the downtime.  By default, unscheduled downtime will be applied first, and scheduled downtime will be applied against the remaining production (ie. 50% unscheduled and 50% scheduled downtime will have 25% of total production [100*50% = 50, then 50*50% = 25). If Nonstandard Downtime is enabled (see Site Specific Features), the total downtime for any period will be added cumulatively, however will not exceed 100%. (ie. 50% scheduled and 50% unscheduled downtime will have zero production in that period [ 100 * (50%+50%) = 0] ).
  • Select whether the downtime is scheduled or unscheduled.
  • Click outside the current line in the table to save the changes.

 

Nonstandard Downtime:

When Downtime is entered, the default setting is for the reserves that are lost to be added onto the end of the Forecast.

This option can be changed to discard Reserves that are lost as a result of downtime.

  1. Switch to Configuration perspective
  2. Enable "Nonstandard Downtime"

 

  1. Click outside of the line to commit the change.  
  2. This change requires the program to be restarted, so you must close and re-open esi.manage.
  3. Run the Production Analysis again, and the Reserves that were lost due to downtime are no longer added.

If you wish to have the reserves lost due to downtime added on, that can be done manually, or be disabling the "Discard Downtime Production" in the Configuration perspective.
