Archives

Archiving

An Archive (which becomes a Budget) is an exact copy of all Project Data within a Forecast at a specific point in time. As part of the archiving process, a relationship is established between the copied data and the original source data. This allows for subsequent comparison (Variance Reports) between the Forecast (source) and the Archive.

There are two types of Archives:

  1. Closed – a Closed Archive cannot be modified at all.
  2. Editable – an Editable Archive can be selectively modified, by users with appropriate Permissions. The typical purpose of an Editable Archive is to save the state of the Forecast Data, but allow subsequent importing of Field Estimates, Actuals or Financial Actuals at a later date.

 

When Should Archives be Created

There are several points in time where it is appropriate to create an Archive, either as a record of the state of your Evergreen Forecast either prior or just after an event, or for subsequent variance reporting and tracking.

  • Budget – when your company has completed the Budgeting process for a given year. This Archive (typically Closed) will provide the ability to report Variances between the Current point and what was Budgeted.
  • Month or Quarter End – either point in time is appropriate, however, if you don’t plan on using the Monthly archives then don’t.
  • Acquisition or Divestiture – this would be a point in time where it would be appropriate to have a State snapshot of the Forecast prior to the Event.

 

Creating an Archive

Archives can only be created by users with appropriate Permissions.

To create an Archive:

  1. Switch to the Configuration Perspective.
  2. Using the Tree View:
  3. Select Budget/Projects
  4. Budget
  5. Budgets
  6. Then select the Current Forecast from the Budgets Table
  7. Based on your current needs, select either the Closed Archive or Editable Archive button.
  8. You will be prompted as to whether you wish to Create the Archive.
  9. By Default, the Archive will be created as either:
    1. Budget Name Archive Date
    2. Budget Name Editable Archive Date

A Closed Archive Budget will have all the same settings as its Parent except:

  1. The Open Flag is set to False and is non-modifiable
  2. The Visible Flag is set to True, but it is modifiable
    • If you wish to make a Closed Archive not appear in the Project Browser or Reports, then set Visible to False.
    • A user with sufficient permissions to modify the Configuration can make the Closed Archive visible at any time, if necessary.

In most situations, the settings on the Editable Archive will be the same as for the Parent.

  1. To prevent User Modification (but not Imports) to Project Data in the archive. In this case set the Read Only flag to True.
  2. To prevent Imports against a particular measure (Production, OPEX, CAPEX), set the corresponding Lock Actuals flag to True.
  3. If you have created an Editable Archive (as opposed to a Closed Archive) for the purpose of Importing Actuals at a later date, once those Actuals have been imported set the Lock Actuals flag to True. This will prevent inadvertent importing at a later date.
  4. If it becomes necessary to import again, you can always Uncheck the flag to perform that particular import.
  5. When you have completed any Imports, you can Close an Editable Archive which will prevent any changes to the data.
