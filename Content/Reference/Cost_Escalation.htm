Currency Escalation - Inflation

Currency Escalation is a way to represent future costs (Expenditures, OPEX, Resources) based on amounts entered in today's "dollars".

There are 3 parts to the configuration of Escalation.

  1. Budget Base Currency Date - this sets the Date that the Escalation Factors are defined against. A Base Currency Date of Jan 1, 2010 means that monetary amounts have been entered in 2010 "dollars".
  2. Currency Escalation File - multiple files can be created to enable evaluation of Projects under a number of different scenarios. An Escalation file contains Measure Currency Escalation entries as described below. In order to apply Escalation to calculations, an Escalation file must be selected.
  3. Measure Currency Escalation - Annualized percentages defined for each CAPEX Type, and OPEX and Resources, on a Jurisdiction by Jurisdiction basis.

Using an All OPEX (Fixed) entry as an example:

  1. $10000 / month for the period Jan 1, 2010 to Dec 31, 2025
  2. Budget Base Currency Date = Jan 1, 2010
  3. Annualized Currency Escalation = 10% at Jan 1, 2011

Note: Escalation is entered against a particular Month (Jan 1, 2011) as an Annualized amount, but is applied in calculations on a monthly Basis. The configured Escalation File is applied such that an entered percentage is used for all periods in the future, until another user entered percentage is encountered.

Jan 2010: 10,000 as the Escalation does not come into effect until Feb 1, 2010

Feb 2010: 10,000*((1+10%)^(1/12))

Mar 2010: 10,000*((1+10%)^(2/12))

Jan 2011: 10,000*((1+10%)^(12/12))

Feb 2011: 10,000*((1+10%)^(13/12))

Jan 2012: 10,000*((1+10%)^(24/12))

The Annualized Percentage (10%) is converted to a Monthly rate, and is applied based on the number of Months from the 'Start' of the Escalation. Feb 2010 has 1 month of escalation applied, while Jan 2011 has 12 months - which results in the full 10% increase from the 2010 Amount. Similarly, Jan 2012 is 10% more than Jan 2011 (12,100 versus 11,000).
