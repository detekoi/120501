OPEX Calculations

There are three ways to Forecast operating Costs:

  • Fixed - a one time, or recurring expense that is not dependent on Production. A Fixed OPEX entry can be related to a specific Product, which will influence the calculated Lifting Cost.
  • Variable - expenses that are a function of the Production amount, such as processing, transmission or tariffs.
  • Price-Based - expenses that are a function of the Price Forecast amount, per the Price Profile and Product chosen.

 

OPEX Modifiers

  • WI% - your company's share of the Operating Expense
  • NPS% - the likelihood that the Expense will be realized. Generally this applies to new wells, where the NPS% is the Likelihood that well will be successful.

There are no 'specific' line item modifiers for Variable OPEX; WI%, NPS% and Royalties (NRI%) will all be based on the values for the related Production Segments.

 

Reporting Period versus Data Period

All OPEX Entries (Fixed or Variable) have a Start and End Date. These fields define:

  1. For Fixed OPEX: the valid Period for the Expense, and how to interpret the Gross Amount in terms of the Time Period.
  2. For Variable OPEX: the valid Period for the Expense.

When OPEX calculations are requested (Reports, Analysis Cases, Ranking, etc.), the Configuration defines a number of Calculation Periods. Each Period has a Start and End Date (Calculation Start, Calculation End), and a Calculation Periodicity.

 

Fixed Operating Cost Calculations - Forecast

The following fields for a Fixed OPEX entry determine the calculated Forecast OPEX for a particular Calculation Period. The key element that influences the OPEX Calculations is Time Period which specifies how the Gross Amount is to be interpreted.

For all Time Periods, the Gross Amount can be converted to an Average Daily Rate. This is used when the Calculation Period does not align with the Valid Period for the OPEX, or when the Calculation Periodicity is less than Monthly.

The Time Period choices are:

  • NULL - the Gross Amount is a lump sum. It will be converted to an Average Daily rate (based on the End Date - Start Date), .
  • Daily - this is a Daily Rate.
  • Weekly - the Gross Amount will be converted to an average Daily Rate, unless the Calculation Periodicity is Weekly.
  • Monthly - if the Calculation Periodicity is less than Monthly, the Gross Amount will be converted to an average Daily Rate.
    • If the Periodicity is Monthly or greater, the Gross Amount will be used as a Monthly amount; i.e., if the Calculation Period is Quarterly, the OPEX will be 3 x Monthly Gross Amount.
  • Quarterly - if the Calculation Periodicity is less than Monthly, the Gross Amount will be converted to an average Daily Rate.
    • If the Calculation Periodicity is Monthly, the Gross Amount will be divided by 3 (3 months/quarter).
    • If the Calculation Periodicity is Annual, the Gross Amount will be multiplied by 4.
  • Annual - if the Calculation Periodicity is less than Monthly, the Gross Amount will be converted to an average Daily Rate.
    • If the Calculation Periodicity is Monthly or Quarterly, the Gross Amount will be converted to either an average Monthly rate (divided by 12) or a Quarterly Rate (divided by 4).

The remaining fields that influence calculations are:

  • Start Date, End Date, and Duration- this defines the Period that the entry is valid for.
  • Amount (per period) - the Amount of the Expense
  • Currency - the Currency Units for the entry. Based on the Currency units conversions, the calculated amount will be converted from the Source to Output Units.

Forecast OPEX calculations can be Configured to apply an Escalation file, in which case the calculated amount for a particular Calculation Period is adjusted based on the Base Currency Date (part of the Budget), and the Currency Escalation factors defined in the particular Escalation file.

Variable Operating Cost Calculations - Forecast

The following fields for a Variable OPEX entry determine the calculated Forecast OPEX. The key elements are:

  • Amount - the variable OPEX Rate ($/volume)
  • Volume Unit - the Production Unit (volume) for the Rate.

The Asset and Product selections for the Variable OPEX entry identifies which Production Segments provide the Forecast Production amounts. The calculated Production volume (in the Variable OPEX volume unit) will be used to determine the OPEX amount.  If Product is left blank, then all production associated with the asset is converted to an equivalence unit and the OPEX amount will be calculated on the total production in that equivalence unit.

The remaining fields that influence calculations are:

  • Start Date, End Date, and Duration - this defines the Period that the entry is valid for. Production which occurs outside of the valid period will have no calculated OPEX.
  • Currency - the Currency Units for the entry. Based on the Currency units conversions, the calculated amount will be converted from the Source to Output Units.

See Currency Escalation

Forecast OPEX calculations can be Configured to apply an Escalation file, in which case the calculated amount for a particular Calculation Period is adjusted based on the Base Currency Date (part of the Budget), and the Currency Escalation factors defined in the particular Escalation file.

 

Operating Costs - Actuals

All 'Actuals' are represented as Fixed OPEX entries. Typically these entries have been brought into the Project via an Import process resulting in:

  • Start and End Dates - Jun 1, 2010 to Jun 30, 2010
  • Gross Amount - 10000
  • WI% - 85% (whatever is appropriate for the particular Cost Center or Well)
  • Type - All Operating Costs (or any other OPEX Type)
  • Product - optional (can be set for Lifting Cost calculations)

Generally the Time Period is NULL as the Actuals entry represents a lump sum for the given period. If a Time Period is selected for a particular entry, the calculation methodology described in Fixed OPEX calculations is applied.

Operating Expenses utilizes the Definitions, Types and Blending logic for Production.

 

Lifting Cost Calculations

The Lifting Cost (Dollar/volume) calculation is accessible as a Project Metric, and is an optional reporting form for OPEX Reports. The basic form of the calculation is:

  • Dollar / Volume = OPEX Amount / Production (in selected volume units)

Fixed OPEX (Forecast and Actuals) can optionally be associated with a specific product, while for Variable OPEX the product association is mandatory.

When a Product has been identified for an OPEX entry, the corresponding Product's production is used in the calculation.
