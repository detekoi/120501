Measure Scale

Measure Scale is used to scale measures when running Analysis Cases, Expenditure Analysis, Opex Analysis or Production Analysis.

The nature of the scaling can be anything that should be modelled that is not in the predefined options of esi.manage.

The Measure Scale works by selecting a hierarchy type from the drop down menu, and then one or more specific values of that hierarchy type on which selected measure or measures (Expenditures, Opex, Production) will be scaled when using this Scale File in an Analysis.

Any number of Scale Files can be configured combining Measure Scale Hierarchies values and Measures.

A new Measure Scale file is created by selecting the button in the Configuration section (Defaults >> Measure Scale). You will be prompted to provide a File name which can be referenced when you want Measure Escalation applied to esi.manage analysis.

Data is entered by a selected Hierarchy, in a particular escalation file, for a specific measure. Each Measure type will be available for its own escalation rate.

Opex is available separately by Fix and Variable, and also by Opex type.
Expenditures is available by Expenditure Type.
Production is available by Product.

The multiple scales in different levels will be inherited and combined. The result is the effective measure that will be applied when the Scale File will be used in an Analysis.

To show the Effective Scale, the button must be clicked. Otherwise, the numbers shown in the table are the ones we have entered, or the default value ‘1’.

There is an option to Duplicate an Escalation file, in order to keep much of the data and only modifying a select number of entries.
