Currency Escalation Configuration

Currency Escalation is a way to represent future costs (Expenditures, OPEX, Prices) based on amounts entered in today's "dollars".

There are 3 parts to the configuration of Escalation.

  1. Budget Base Currency Date - this sets the Date that the Escalation Factors are defined against. A Base Currency Date of Jan 1, 2010 means that monetary amounts have been entered in 2010 "dollars".
  2. Currency Escalation File - multiple files can be created to enable evaluation of Projects under a number of different scenarios. An Escalation file contains Measure Currency Escalation entries as described below. In order to apply Escalation to calculations, an Escalation file must be selected.
  3. Measure Currency Escalation - Annualized percentages defined for each CAPEX Category, and OPEX and Resources, on a Jurisdiction by Jurisdiction basis.

 

A new Currency Escalation file is created by selecting the button. You will be prompted to provide a File name which can be referenced when you want Escalation applied to esi.manage calculations. See Currency Escalation Calculations for a more detailed description of how the supplied values are used in calculations.

Data is entered by Jurisdiction, in a particular escalation file, for a specific measure. Each Capex type will be available for its own escalation rate, Opex is available as a single option (cannot have different escalation rates for different Opex types), and Price is also available as a single field (cannot have different escalation rates for different products).

 

The Escalation Table propagates user entered values for all time periods into the future, until another user supplied value (by measure) is encountered.

There is an option to Duplicate an Escalation file, keeping much of the data the same and only modifying a select number of entries.
