Enersight Mapping File

In order for the Enersight Integration to work, a mapping file must be configured specifically for the Enersight and esi.manage data. The mapping file for the Enersight Integration can be found in the Sample Files folder of the esi.manage installation folder (eg. 3esi\esi.manage\Sample_Files\Enersight\Mapping.csv)

The mapping file specifies what information is transferred from a PDS into esi.manage, defaults for many options, as well as translates values in Enersight to values in esi.manage. Generally speaking, the mapping file follows a three column format (except where indicated otherwise).

  • Column A - Mapping type
  • Column B - Enersight value
  • Column C - esi.manage value

Each section of the mapping file will start with a line that begins with a number sign (#) which will indicate what the section is for. The "#" denotes a comment line, and this line will not be considered by esi.manage when importing data. Additionally, there may be comments on the format of the section (eg. four columns, defaults, etc.), or examples of what the values could be.

The mapping file is read from top to bottom, with each row being interpreted sequentially. For this reason it is important to maintain a one-to-one mapping in all cases. Any mapped values where more than one Enersight value is mapped to a single esi.manage value, only the last values will be imported. For example, if three capital classes are mapped to a single esi.manage expenditure type, only the capital in the third class listed will be imported.

When deciding what information to include in the mapping file, there are a few things to consider. First, only values that do not already match between Enersight and esi.manage need to be mapped. If the value is the same in both systems, it will assume that is the mapping (eg. bbl as a production unit). Second, if there is unnecessary or incomplete information in the mapping file, you may not need to remove it. Incorrect/incomplete mappings will be ignored if there is no data coming from Enersight that requires it to be mapped (eg. inert gas as a product in Enersight with a blank value in the esi.manage column, will be ignored if there is no inert gas production in Enersight). Lastly, any row can be commented out to be ignored by esi.manage by entering "#" at the start of column A.

Currently, the following data can be imported. Also listed is the location of the source data in Enersight as well as the location of the data once imported into esi.manage.

  • Attributes
    • Enersight: user data
    • esi.manage: project hierarchy or project custom attribute
  • Capital
    • Enersight: capital
    • esi.manage: expenditure
  • Production
    • Enersight: Decline/Prod
    • esi.manage: Production
  • Revenue
    • Enersight: Other Income and Process
    • esi.manage: Production (with effective price to calculate the same cost or additional revenue)
  • Opcost
    • Enersight: Opcost
    • esi.manage: Opex
  • Ownership
    • Enersight: Land
    • esi.manage: Ownership
  • NRI
    • Enersight: Land
    • esi.manage: Ownership (may be hidden of site specific features not configured correctly)
  • ProductQuality
    • Enersight: Units
    • esi.manage: Configuration > Product Qualities
  • Royalty
    • Enersight: Economics or Sev&AdVal
    • esi.manage: Burdens > Royalties
  • Burden
    • Enersight: Other Royalties (eg. positive GORR)
    • esi.manage: Burdens
  • Tax
    • Enersight: Economics or Sev&AdVal
    • esi.manage: Burdens
  • RoyaltyInputs
    • Enersight: Royalty calculator inputs
    • esi.manage: project custom attributes

Note: The mapping file is a .csv (comma delimited) file. Although these files can be edited in Microsoft Excel, occasionally it will insert commas into empty cells which will cause the mapping file to no longer work. For this reason, it is often best to edit the mapping file in a text editor (eg. Notepad++), or keep a backup copy of the mapping file before making any changes.

Defaults
This section is for setting default values for fields that are required in esi.manage, but do not exist in Enersight. it is also used to set universal values for the import, such as currency, and how to treat variable opex. All rows that exist in the Sample file must be populated.
DefaultValue ProjectManager Name of Project Manager that exists in esi.manage null All imported opportunities will be given this Project Manager
DefaultValue ReserveClassification Name of Reserve classification in esi.manage null All production will be entered into this Reserve Classification
DefaultValue RoyaltyScenario Name of a Price Scenario that exists in esi.manage null Effective Price and Effective Royalty Rates will be imported into this Price Scenario
DefaultValue ProcessFeeValues "Negative" or "Positive" null

Negative - will import negative production and maintain a positive price to represent process fees

Positive - Will import positive production values and a negative price to represent process fee values
DefaultValue PdsCurrency Currency abbreviation from Enersight null If no Currency unit is set for the PDS, this currency will be used.
DefaultValue VariableOpcostByProduct "true" or "false" null

True - variable opcost will be summed by product and imported as a single line item for each product

False - Variable opcost will be imported grouped by Enersight opcost class (requires not having >1 product mapped to a given class and designated as variable)
DefaultValue ProductionKind "AsSold" or "AsProduced" null

AsSold - production passed to esi.manage will be after shrinkage, and match the "As Sold" values in enersight reports.

AsProduced - production passed to esi.manage will be unshrunk and match the "As Produced" values in enersight reports

It is important to note that variable opex will also be based off of the Production Kind that is designated here.
Action Configuration
This section allows you to choose which pieces of data you want to import into esi.manage. The values are all entered on a single row, each in a new cell. Any item removed from the list will not be changed when merging data.
ActionConfiguration ImportFromEnersight Measure 1 (eg. Attributes) Measure 2 (eg. Capital)

Enter a new value in each column on the same row until all desired import data is listed. The options are: Attributes, Capital, Production, Revenue, Opcost, Ownership, NRI, ProductQuality, Royalty, Burden, Tax, RoyaltyInputs

Note: NRI overrides Ownership and Effective Royalty percentages when performing calculations in esi.manage.
Action Options
This section indicates the behaviour when loading projects that already exist in esi.manage. This row can be commented out for a complete delete of the existing projects in esi.manage and a reload of new data as per the mapping file.
ActionOptions ImportFromEnersight Merge null When merging a new import with projects that already exist in esi.manage, all data that is listed in the ActionConfiguration section will be deleted from the existing projects in esi.manage, and reloaded. Data that exists in esi.manage (eg. Capital) that is not listed in the ActionConfiguration line will not be changed.

Products
This section is where product names are lined up. They must be a one to one mapping.
ProductMapping Enersight product name esi.manage product description null Any product that is being saved into the PDS must have a mapped esi.manage product.
Volume Units
This section is to indicate which esi.manage units to use for each Enersight volume unit. This is particularly relevant for units in Enersight that are used for both energy based (gas) and mass based (liquid) products.
VolumeUnitMapping Enersight production unit esi.manage production unit

Mass, Energy, or Equivalence

 Enersight often uses the same production unit for both Gas and Oil, however esi.manage must use different units for energy and mass based products. In a situation where Enersight uses the unit "m3" to represent both, in esi.manage we will have one called "m3" with "Mass" in column D, but also "m3 (gas)" with "Energy" in column D.
Currency
This section is used to indicate the currency unit in esi.manage that represents different magnitudes of a currency unit in Enersight.
CurrencyMapping Enersight base currency unit (eg. USD) scale unit (eg. m$) esi.manage currency unit for the given scale (eg. kUSD) The example mapping can be interpreted as the base currency of Enersight is USD, and when multiplied by m$, is equal to kUSD (thousand US Dollars) in esi.manage.
Product Quality
If the product quality units do not match in esi.manage and Enersight, they are mapped here.
ProductQualityUnitMapping Enersight product quality unit (eg. GJ/e3m3) esi.manage product quality unit (eg. GJ/e3m3 (gas)) null This mapping is only required for product qualities that include a production unit that does not match between Enersight and esi.manage, such as the "m3" unit mentioned in VolumeUnitMapping.
Expenditure Types
This section maps the Enersight capex class to esi.manage expenditure type description. It is important to note here that there should not be more than one Enersight class mapped to a single esi.manage expenditure type, as only the last mapping in the list will be loaded.
ExpenditureTypeMapping Enersight capital class (eg. class1) esi.manage expenditure type (eg. Drilling) null The Enersight capital class is typically denoted by "class#", however custom capital classes can exist that have been named. On the capital cost definitions page in Enersight, the value to enter in column B is the value in the drop down list directly to the right of the Name.
Opex Type Mapping
This section is for mapping fixed opex classes in Enersight to fixed opex types in esi.manage, IF the VariableOpCostByProduct value is set to "true". If the value is set to "false", all opex types should be mapped here. If the value in the Enersight PDS is saved as a $/production unit, it will be imported as variable opex in esi.manage. If it is $/unit time, it will be imported as fixed opex. It is also important to note here that these should be a one to one mapping for the same reason as expenditure types.
OpexTypeMapping Enersight opex class (eg. class1) esi.manage opex type (eg. Monthly Well Cost) null Same as with the expenditure type mapping, the opcost class can be found in the Opcost Definitions page of Enersight. The esi.manage opex type should be the description, not the code.
Variable Opex Type Mapping
This section is for mapping variable opex when the VariableOpCostByProduct value is set to "true" in the defaults section of the mapping file. In this scenario, all opex in Enersight that is denoted as having the Opcost Type of "Variable" and having the same product will be summed and imported as a single variable opex line item in esi.manage. An opex type will need to exist in esi.manage that is not also used in the previous section, "OpexTypeMapping". An example would be to create a separate variable opex type for each product.
VariableOpexTypeMapping Enersight product (eg. oil) Esi.manage opex type (eg. Variable Oil Opex) null Same as with the expenditure type mapping, the opcost class can be found in the Opcost Definitions page of Enersight. The esi.manage opex type should be the description, not the code.
Income Mapping
Revenue that is generated in Enersight as a result of processing fees are modelled in esi.manage as revenue from a production stream. Products must be created in esi.manage that this income can be mapped to, for each product that has processing revenue generated.
IncomeMapping Enersight product (eg. Gas) Esi.manage product (eg. Other Gas Income) null The esi.manage product used to represent this additional revenue must not be used elsewhere to model actual production (ie. do not use the product "Gas" here, as any production imported earlier in the mapping file will be overwritten).
Process Fee Mapping
Similar to Income Mapping, this section is for mapping costs as a result of processing fees. Products must be created in esi.manage that these costs can be mapped to.
ProcessFeeMapping Enersight product (eg. Gas) Esi.manage product (eg. Other Gas Costs) null The esi.manage product used to represent this additional revenue must not be used elsewhere to model actual production (ie. do not use the product "Gas" here, as any production imported earlier in the mapping file will be overwritten).
Revenue Rate Mapping
This section is similar to the Currency Mapping section. It indicates what currency rate in esi.manage is for what magnitude of currency rate in Enersight.
RevenueRateMapping Enersight currency unit (eg. USD) Magnitude (eg. m$) esi.manage revenue rate (eg. kUSD/d) The esi.manage revenue rate can be set to the same as the currency mapping, and best practice is to keep the periodicity of the revenue rate as daily.
Revenue Price Profile Mapping
Esi.manage support default price profiles for products, however Enersight can provide multiple different revenue rates in a single PDS. This section allows the user to map magnitudes of currency to esi.manage price profile units.
RevenuePriceProfileMapping Enersight currency unit (eg. USD) Magnitude (eg. m$) esi.manage revenue rate (eg. kUSD) Similar to Currency Unit Mapping
Tax Name Mapping
Several types of taxes can be calculated in Enersight. These can be renamed on their way into esi.manage.
TaxNameMapping Enersight tax name (eg. state) esi.manage tax name (eg. State Tax) null  
Value Mapping
The value mapping section allows a user to map some values out of Enersight that are not user data, to either a fixed field, hierarchy attribute, or project custom attribute. The two fields in esi.manage that can be used are "Asset Type" and "Asset Subtype". Column B indicates whether you want to import the asset type or subtype from Enersight, the Column C indicates which location in esi.manage you want to put it, and the Column D indicates the name of the esi.manage field (where relevant). See below for examples
Value Mapping EsiEsOpportunity.type assets.type   This will import the asset type from Enersight into the asset type in esi.manage. The asset type must already be configured in esi.manage
Value Mapping EsiEsOpportunity.type hierarchyAttributes Well Type This will import the Enersight asset type into the project hierarchy field called Well Type. The asset type must already be configured in esi.manage as a possible selection for Well Type
Value Mapping EsiEsOpportunity.subtype hierarchyAttributes Well Type This will import the Enersight asset sub-type into the project hierarchy field called Well Type. The asset sub-type must already be configured in esi.manage as a possible selection for Well Type
Value Mapping EsiEsOpportunity.type customAttributes Project Type This will import the Enersight asset type into the project custom attribute field called Project Type. If this custom attribute is not a "text" attribute, but a picklist selection, it must already be configured in esi.manage.
