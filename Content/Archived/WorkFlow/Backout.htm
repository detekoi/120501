Backout

Backout is a 'Site-Specific' Feature, and as such, has been developed under a specific set of assumptions. The corresponding Backout calculations are appropriate only for those specific set of assumptions. Unless this feature is enabled in the Configuration, none of the screens shown below will be visible in a Project.

The Backout screen under the Production editor allows you to calculate the actual benefit of bringing a new well online into an existing system that will cause other assets in a connected system to have to decrease their output.

The Backout screen can be accessed from the "Backout" tab in the Production editor or by clicking the Backout barrel icon in the toolbar.

 

To Add Backout

  1. Select an Asset that is affected, and which product is effected.
  2. Set the start date of the Backout period by setting an Offset Period and Offset Amount.
    • Example of Offset: An asset with production coming onstream January 1 with an Offset Amount of 6 and an Offset Period of Months would see the Backout period start July 1.
    • If the Offset Amount was 10 and the Offset Period days, it would start January 11.
  4. Enter the duration of the Backout by setting a duration and selecting a period. Based on a combination of Offset and Duration, Start and End dates will auto-populate.
  5. The Backout factor is the amount of production lost as a result of the Backout. This needs to be entered *in the same unit* as the selected production was entered for the selected asset.

 

The bottom of the page shows the other calculations (risk, working interest, royalty) being applied to the record in question.

Note: in esi.manageTM, the opportunity cost of the new well is attributed to the well causing the backout, not against the wells actually operating at reduced capacity. See disclaimer at the bottom of the Backout page show above.
