Production Baseline Configuration

If Production Baseline has been enabled, an additional Data element is added to the Production table.  

  • The Baseline element is a Boolean and indicates the Production Segment that will represent the Base (existing) production profile

Note: this column is at the end of the row by default

Only one Production segment can be identified per Asset/ Production SubCategory / Zone combination.

 

Uplift Segments

An uplift segment is configured as any other Production segment. However, the interpretation of some elements has particular significance when modelling Baseline.

  • Baseline Flag is False - which indicates that this is an Uplift segment (you will not be allowed to tag more than one Asset/Product combination as the Baseline segment).
  • IP - the Initial Production Rate.
    • If Total Uplift, this IP must include the Base production. For accurate modelling, this value should be the sum of the Base Production Daily Rate (at the anticipated Uplift Onstream Date), plus whatever incremental production is anticipated from the activity.
  • OnStream Date - indicates when the Uplift is Forecast to come Onstream. No Production will be reported prior to the earliest Onstream Date for the Uplift Segments.
  • OffStream Date / Rate - the implication of these values is dependant on which type of Uplift (Total or Incremental) is being used.
    • As Total Uplift includes the Base, there will be NO production reported after the Onstream Date/Rate for the Uplift Segment.
  1. Multiple Uplift segments can be created against the same Asset/Product.
  2. Gaps and Overlaps are allowed between segments. If there is an overlap in the Dates (the Offstream Date for the first Uplift segment is later than the Onstream Date of the second Uplift segment), the Uplift contributions are additive.
    • If Total Uplift is being used, and both Uplift Segments include the Base in their IP's, the Reported Uplift production will likely be larger by the Base contribution during the overlap period. For example: Baseline = 100, Uplift 1 = 120, Uplift 2 = 130. With no overlap in dates between Uplift Segment 1 and 2, the reported Uplift amount will be 20 and then 30. If there is an overlap, the reported Uplift will be 150 during the overlap period. This may not be the anticipated result.
