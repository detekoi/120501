Project Grouping

After pressing Next> the Project Grouping window will appear.

Multiple attributes can be assigned to either Project-Level grouping or Asset-level grouping. Here you can group attributes to be aggregated into separate projects based on those attributes, or funnel groupings into assets. Leaving Project-Level attributes will result in a single project being generated. Similarly, empty Asset-Level attributes do the same for assets.

Project-Level attributes and Asset-Level attributes can be set in a grouping order by using the 'up' and 'down' arrows.

For example, placing Project into the Asset-Level Attributes will create a single project with an asset for each original project.

 

Alternatively, if Area is placed into the Project-Level Attributes, multiple projects will be created aggregating the projects with common Area into its own project.

"Preserve projects as assets" functionality of the external aggregator can be accomplished by selecting "Project" as an asset level grouping attributes. To get the same functionality as "Group assets only", you need to select "Project" as the project-level attributes and leave asset-level attributes empty.
