Project Aggregator

The project aggregation function allows you to combine multiple projects into a single project aggregating all the assets and measures (production, OPEX and CAPEX) into a new project (or projects).

 

Aggregation Exclusions

Field estimates and Actuals are not part of project aggregation. Derived by products will not be aggregated.

 

Production Aggregation

The following fields must be the same in production in order to aggregate them:

  • Product
  • Production Rate
  • Quality Unit
  • The production must be monthly.

The aggregated production data will be tabular.

All derived associated products will be aggregated as normal products.  Derived by-products will not be aggregated

The Shrinkage, POS%, and overall quality will be calculated.

The quality from a production segment that uses equivalence units is discarded during project aggregation. As a result, for the period of time where the production segment uses equivalence units, no quality is calculated.

 

Production Exclusions

  • Comments, 'Dependent On' and Price Profile will not be aggregated.
  • Downtime will not be aggregated.

 

Expenditure Aggregation

The following fields must be the same in Expenditure in order to aggregate them:

  • Type & Category
  • Status
  • Currency

The expenditure must be on a monthly basis.

Gross expenditure amount and PD% will be calculated.

AFE linkage, Comments, dependencies, negative started, and ended are ignored.

 

OPEX Aggregation

The following fields must be the same in Fixed OPEX in order to aggregate them:

  • Category
  • Type
  • Status
  • Product
  • Currency

Time period must be monthly.

POS% and Gross amount are calculated.

 

The following fields must be the same in Variable OPEX in order to aggregate them

  • Category
  • Type
  • Status
  • Product
  • Currency
  • Production volume unit

Amount per volume is calculated.

If no production, Variable OPEX will be ignored.

Status of the OPEX in aggregation result projects is always NULL.

Variable OPEX(es) are recalculated with respect to Use Sales Volume, and the flag will be always set in the result.

 

Ownership Aggregation

The ownership for expenditures, production and fixed OPEX will be calculated.

Ownership is calculated at an asset level.  An overall ownership is calculated for each type of measure by gradually summing with the weight corresponding to its part in the total of the measure.  Ownership for a measure in a project will be ignored if there is no data for that measure.
