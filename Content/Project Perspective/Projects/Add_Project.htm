Add Project

For every new development that occurs within a given budget, you can create a project to track it. Projects are composed of a single asset or series thereof, along with the financial, reserves, OPEX, timing, hierarchy, KPI and production information related to it.

 

To Add a Project:

  1. Start by opening the Project Perspective.
  2. Since you are adding a new project, you do not have to worry about any filtering at the moment, so you can ignore the Project Filter and skip to the Projects view.
  3. In the table on the right, (bear in mind it may be empty), right click and select "Add Projects". The Add Project screen will appear.

 

  1. Fill out the Project Overview form. Required fields are listed in bold. If you check the Import Hierarchy field, a dialogue will appear and you can browse to the location of the hierarchy import file. This allows you to import an existing hierarchy.

 

  1. If you did not check the Import Hierarchy option, clicking the Next button after populating the required fields will bring you to the Project Hierarchy screen.

 

  1. Fill out the Hierarchy form. All the fields are required, however are automatically set with a default values for hierarchy elements to prevent manual selection each time.  See "Hierarchy" in this file for more on that - then click "Next"
  2. Select "Finish" to create the Project.

 

Delete Project

You can delete a project at any time. You will not just delete it from your view, but from the database. Deleting an item for you deletes it for everyone.

There are two ways to delete a project:

  1. Highlight the project in the "Filtered Projects" view, then right click and select "Delete Project"
  2. Highlight the project in the "Filtered Projects" view, then use the "Delete Project" button (shown above).

 

A warning message appears confirming you want to delete the selected record. If you are sure, click "OK".

You may not be able to delete some projects because they are related to other projects or budgets.
