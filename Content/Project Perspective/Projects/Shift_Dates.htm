Shift Dates

In esi.manage you have the option to either Update Dates (only change the project or asset start date), or Shift Dates (allows you to move several dates at the same time). This can occur at several levels. Shifting at an upper level moves all dates contained under that. The levels are Project, Asset, and Expenditure. Below is a diagram of the relationship between the levels:

  • Project: The Project level is the highest roll-up you can shift on. This shifts all Assets, Expenditures, and milestones contained under that project.
  • Asset: Shifting on an Asset will shift related milestones as well as dates for Expenditures that the asset has associated Wellcount Dependencies on.
  • Expenditures: If you wish to Shift Dates of Expenditures only, but not the asset or project, this must be done manually.

This page explains how to shift dates at the Project level. For lower roll-up level shifts, refer to the respective portion of the Project Content subject.

 

To Shift Project Dates:

Project dates can be shifted using the Change Project Start Dates button.

  1. Choose all Projects from project list that you want to shift dates for.
  2. Click the " Change Project Start Dates" button. This screen will appear:
  3. Choose whether you want to Shift the Project or Asset start dates
  4. Enter either a new start date or the number of days you wish to shift each project or asset.

Shifting Dates will only cause all dates below the level to beshifted (ie. shifting Asset date will move asset, expenditure, and production dates, but will not change the project start date).

Update Dates:

Project dates can be updated using the Change Project Start Dates button.

  1. Choose all Projects from project list that you want to shift dates for.
  2. Click the " Change Project Start Dates" button. This screen will appear:

  3. Choose whether you want to Update the Project or Asset start dates
  4. Enter either a new start date or the number of days you wish to shift each project or asset.

Updating Dates will only cause the single date you choose to change. No other values will be changed within the project.

There is a business rule in the application where you cannot have an Asset start date prior to the Project start date. If you encounter this error, simply update the other date first (ie. if updating the Project start date to be after the Asset start date, simply update the Asset start date first).

Refer to "Shift Date Options" for additional options when shifting dates.
