Creating the Template Projects

Template Projects to be used as the Source for the Template Scheduler must be created in a Budget that has been designated as a Template Source. This flag is set in the Configuration for the particular Budget.

Any of available approaches - Creating a New Project, Cloning, or Cloning an Asset can be used to create a Template Project.

 

As shown, a Template Project can either be:

  • New Projects - any new Projects created via the Scheduler from this Project will contain only one instance of the data within this Project, i.e.; one drilling activity and the corresponding Production, OPEX, Reserves, etc.
  • New Assets Divided into 1-year Projects - once the Scheduler has been instructed to create the Virtual Projects, it will create individual Assets for each of the activities in a single project, based on the Year. For example, if 12 instances were specified for a given year, there will be 12 copies of this project's data within the single Project - individual Assets with the corresponding Project Data scheduled accordingly.

 You could have two copies of each of the Template Sources - one with New Projects and the other with New Assets. The New Projects version could be used for Scheduling the next year's activities, while the New Assets could be used for the Plan in subsequent years.
