Template Well Scheduler Overview

It is becoming increasingly common that some portion of a company's Drilling program will include a large number of relatively fast, repetitive Drills in a specific geographic region. esi.manage provides a powerful feature (Project Template Schedule Editor) to assist planners in the analysis and scheduling of these drills.

This tool can be used to create a schedule for any type of Project (conventional or program drilling).

 

The basic process is:

  1. Create a Budget(s) that will contain the Project Templates, and identify this Budget(s) as being Template Projects Only.
  2. Create (or clone) individual Projects into that Budget. Provide the necessary information to represent the Activity and it's outcome - i.e., Expenditure amounts and duration, Production profiles, Reserves, OPEX, and Resources.
  3. Each Project has a Template Type - either New Projects (in which case each Virtual Project is created as a stand-alone), or New Assets Divided into 1-year Projects (each new Asset that falls within a specific year will be contained within a single Project)
  4. With that Budget (Template Well Source) set as the Active Selection, open the Template Scheduler Editor from the Projects Perspective. You will be required to select a Target Forecast that the Virtual Projects will be created in, and the granularity that you wish to schedule the projects (Daily, Monthly).
  5. The Scheduler will be initialized with whatever Template Projects you had visible - for example if you had used the Project Browser filters to show only Projects from a certain Asset Team or Region, those are what will be displayed in the Scheduler.
  6. The Data Matrix interface is used for this feature - the values that are entered into a particular calendar period indicate how many instances should be created. You can use the Copy and Paste capabilities of the Data Matrix to populate the grid.
  7. Once the desired number of wells have been entered, the Save button will result in the Projects and Assets being created with the indicated frequency.
  8. The Projects that are Created are termed 'Virtual Projects' - their data is real but they retain a relationship to the Type Well that they were based on. This connection allows subsequent changes to the Schedule to replace the Virtual Projects accordingly. Updates to a Template Well's information (different Production characteristics for example) can be replicated in the Virtual Projects by re-applying the Schedule.
  9. As long as a Project is Virtual, it cannot be worked with directly. Once the plan for a particular period in time has been established, Projects can be Promoted from a Virtual to a Real Project. After this, the Project can be edited - and it will not be impacted by any subsequent use of the Scheduler.
