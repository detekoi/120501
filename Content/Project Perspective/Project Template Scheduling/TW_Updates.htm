Updating Virtual Projects

Projects created via the Template Scheduler are created as a type of project called a Virtual Project. This means that these Projects have a relationship with the Template Project that they were created from.

If you open the Template Scheduler at some time in the future (after Virtual Projects have been created), esi.manage will find all Virtual Projects for each of the selected Template Projects and display them in the Scheduler, based on the current Periodicity and Focus Period. The Instances at this point will be Black - these projects exist.

 

  1. To Change the existing Schedule, enter new values for the Template Project in the Time Period you want to change. As you type in values, the Scheduler will show that there are Changes to be saved - the new values will be Blue and the Scheduler shows the *.
  2. When you save the Schedule, only the instances in Blue will result in a change to the Virtual Projects. Within any period that is changed - existing Projects will be deleted and new instances created.
  3. If you wish to drive a Change to the Type Well Source down through all Virtual Project instances, you can select the corresponding lines of the schedule and click on the wrench icon labeled “Rebuild the selected template schedules”. After confirming that you wish to recreate to the latest template, esi.manage will regenerate the schedules of all the selected lines.
