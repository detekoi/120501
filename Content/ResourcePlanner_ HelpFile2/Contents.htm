Resource Planner Perspective

The Resource Planner


The Resource Planner is an integrated module within esi.manage™, it is recognized as its own perspective and will be deployed with every version of esi.manage. This tool is used for Resource Utilization, Resource Scheduling and Planning.

Launching the Perspective the first time

 

The Resource Planning perspective is accessed via the ‘open perspective’ button situated on the perspectives bar.

 

You will now see that you have moved into the Resource Planning perspective


 

 

 

 

 

 

 

 

 

 

 

 

 

The Resource Planning perspective is made up of five views as follows (refer to figure above):

 

  1. Asset Selection (includes Expenditure / activity selection)
  2. Resource Selection
  3. Schedule Builder
  4. Resource Line
  5. Gantt Settings

 

 

Asset Selection


The asset selection screen appears in the top left hand part of the Resource Planning perspective. The Asset Selection view allows the user to select which assets and activities (expenditures) to be represented in the Schedule / Gantt Chart. Assets to be considered are selected by pre-defined Project Set or by Budget. Note: Project Sets with filters on hierarchy or attribute types not supported by Resource Planner and will not appear in the list.

 

 

 

Budgets

Budgets are a snapshot of a collection of projects over a set series of time. Each project must belong to a Budget. To select a project in Resource Planner, either a Budget or Project Set must be selected first.

 

Project Sets

Project sets are a key part of the asset selection process. Before you start using Resource Planner, verify that assets within projects are grouped into appropriate project sets such as by resource line or rig line  (i.e. a group of rigs or other resources for which you want to simultaneously schedule activities).

 

Expenditure Selection

The Asset Selection view also requires the user to select one or more expenditures (or activity type) to schedule against. The user can select multiple activities or expenditures. The expenditure list is populated from the list of expenditure types setup in the configuration perspective.


Once the asset and an associated activity or expense relating to the selected asset has been selected, the different combinations of projects, assets and expense lines appear in the Schedule Builder and in the ‘Unallocated’ line within the Resource Line, ready to be scheduled against a resource. Please note if the selected expenditure items do not appear in at least one asset, a yellow warning icon will appear with an associated warning message.

Limitations between single and multi-expenditure selection

When selecting expenditures you may choose to allocate against many or you may allocate just against one expenditure. There are certain limitations which will be activated when selecting one or the other, for single expenditure selection this will allow you to use the Auto-Build function within the Schedule Builder. When choosing multiple expenditure this will allow for the viewing of the relationships between different activity types but the Auto-Build function will be disabled.

 

 

 

 

 

Clear Expense Durations

If expense activities have been allocated to a resource, this will clear the allocation and move the expense items back to an unallocated bucket. This is done by pressing the Clear Existing Expense Allocations button, shown below.


 

 

 

 

 

 

Resource Selection

The Resource selection view is found in the bottom left hand corner of the Resource Planning perspective. The resource selection view has ‘Select all’ and ‘Clear all’ buttons to make it quick and simple to select a large number of resources simultaneously. There are also two drop down buttons, one relating to the Capacity File and the other relating to the Category of the Capacity File. This allows the user to navigate quickly to the correct Capacity File if there is a long list of Capacity Files to choose from.


Once the Capacity File and Category has been selected, the list of resources within the Capacity file will be displayed with a checkbox besides them. As you click on each check box, a new Resource Line will appear in the Resource Line view ready for the assets and activities defined within the Asset Selection view to be scheduled against.

 

 

 

 

Resource Selection

Resource Selection is a two-step process, first we must select our Capacity File, and this will be set up in the Configuration Perspective, this will contain our Resources. The second step within the process is the use of the second drop down menu, and this will contain our Categories. Categories will further define our Resource Selection, these will define which type of resource we will need to use, ie human vs equipment.



 

Duration Modes


 

There are three types of Durations modes, Manual, Expenditure and Asset Attribute, these will affect the duration of the expenditures placed upon the Resource Line.

 


The first of these is Manual Duration timing, this will mean that we are able to manually choose the duration of the expenditure through the use of the “Duration (days)” text box. To manually select our duration we will first select the “Manual” node within our Resource Selection View, we must then select our “Duration”. This can be done by inserting a number into the “Duration (days)” text box. Doing this will set all of the activities durations to the selected number.

 

 


As we can see here the manual duration of the activities is set at 10 days and will be shown through the Resource Line.

 

 

 

 

 

 

 


The second duration mode is Expenditure Duration timing, this will mean when this is selected the activities durations will be based off of their own current expenditure duration/timing.



As it can be seen here the duration of the BC Dry Hole Scenario – Drilling is 13 days, this will be reflected within the Resource Line as seen below.

 


The Asset Attribute duration mode is the final of the three, when this is selected the activities durations will be based off of an Asset Attribute that will be pre-configured or will be set up by a super user. This asset attribute will apply a defaulted number to the duration of specified capital/expenditure types, but the start date of the activities will be populated by the expenditure start dates.

 

Shown above is an Asset Attribute created for the Resource Planner, this is done through the Configuration Perspective and will be done by a super user. This attribute sets the Cycle Time for Completions to be 25 days.



 

 

 

Resource Modifier Flag

The Resource Modifier Flag is show in esi.manage as the Apply Resource Efficiency check box, this will apply the Resources Efficiency Modifier. The Efficiency Modifier is set up through the Configuration Perspective and is applied to a singular resource, this is defaulted at 1.0, this will set the efficiency in which a resource operates. This is an important feature because when used it changes the how fast/efficient the Resource can complete the activity assigned to it.


 

 

 

 

Clear Resource Assignments


The ‘Clear Allocations Now’ is used to reverse the process of the ‘Assign Resource Now’ button, this button will move the allocated activities within the Resource Line back into the ‘unallocated’ resource line.

 

 

 

 

 

 

Gantt Chart-Resource Line


The Resource Line view is found in the upper right hand section of the Resource Planner perspective by default. It is an interactive Gantt Chart-like view of the resource schedule. The system automatically synchs the data of the Resource Line and Schedule Builder views. The Resource Line’s Gantt Chart display activitives based on the ‘Gantt Settings” and the timeline settings can be controled by the shortcut bar located on the right of the Resource Line view.

Drag and Drop options


The drag and drop functionality provides a very easy way to adjust the schedule and automatically updates the start and end date of the activity and production. It does this by invoking the asset date shift functionality when you move the bars. This way, when you move the bars around it is moving not just that expenditure, but all of the expenditures, production, resources and Opex associated with that asset. The shift is “anchored” to a single activity.

 


The Drag and Drop settings icon allows the User to select between to options: Drop Side by Side on Same Resource, and Insert into timeline shifting assets to make space. Side by side is the default behavior,  when the User drops an activity onto the Gantt chart, the activity lands below the mouse, creating an over allocation if necessary. 

 

 

 

 

 

 

 

 

The new mode is “Insert into timeline shifting assets to make space”.  When you drop your allocation it will move itself to immediately after the intersected allocation. This “insert” function will insert the activity into the resource line and insert “padding” or a selected amount of gap days on either side of the inserted activity. The rest of the schedule for that entire line is shifted to make the next allocation start immediately after the allocation you just dropped plus your padding.


 

 

Crosshairs


The ‘Crosshairs’ feature can be triggered by clicking the Crosshairs button on the top right:

 

Once triggered, whenever the mouse pointer is in the Gantt Chart timeline, the following details are highlighted (starting top left and continuing clockwise) around the crosshairs-like graphic: Activity start and end dates; Current time below the Crosshairs; Resource and Project name – Asset name.

 

 

Timeline Controls


You can modify the appearance of the Gantt by adjusting the time frame in view. This can be accomplished by clicking the buttons to the top-right of the Gantt.

In order from top to bottom these options are:

  • Zoom In: Zooms the dates down to the next level of granularity (from yearly to 6-month; weekly to daily, etc.)
  • Zoom Out: Zooms the dates out to the next level of granularity (from 6-months to a year; daily to weekly, etc.)
  • Granularity: Select a time slice to show - weekly, monthly, yearly...etc. The smallest granularity is Days and the largest is Decades.
  • Change Horizon: Will open a dialogue box to select the Start date, End date, and display granularity of the Gantt
  • Scroll System Time to Visible area: Zooms to today's date, with a red line down the center of the Gantt demonstrating where that date is.
  • Scroll to a Specific Timepoint: Specify a date to scroll to.
  • Show All Activities: Zooms in/out of the view so that all selected timelines are visible on the screen.
  • Show Earliest Activities: Scrolls to the very first date in the selected data set.
  • Show Latest Activities: Scrolls to the very last date in the selected data set.

Schedule Builder

In the default layout, the Schedule Builder is found in the bottom right hand area of the Resource Planning perspective. The Schedule builder is a table of rows and columns. The rows represent each combination of project asset and activity / Expense to be scheduled across the different resources. There are also a series of default columns containing different criteria about each activity. These columns can be sorted so that the assets / activities at the top of the list will be scheduled first. This makes it very quick and simple to produce an order for the schedule.


 

 

 

Auto build schedule

When automated scheduling is preferred over manual, users can utilize the ‘Build Schedule’ feature available at the bottom of the Schedule Builder view. Users can either leave the check box ticked to set the start date to equal to the earliest asset date currently selected, or, alternatively, uncheck the tick box and set a specific start date.


 

 

 

 

Scheduling order

Once the user has defined the start date for the schedule, the next step is to sort the assets and activities based on scheduling requirements. Any of the visible Schedule Builder columns can be used to sort the data, simply by clicking the header of the desired column (clicking the column header more than once toggles between sorting by ascending and descending orders.) Or users can utilize the default Schedule attribute column, ‘Scheduling Priority’, to enter specific priorities then sort by that column. The schedule order will be determined by the order that the projects are listed in within the Schedule Builder.


 

 

 

 

 

 

Adding and Editing attributes


Users can change the order, add, and remove columns by using the ‘Change Viewable Columns’ feature. This function is associated with one of the powerful features of Resource Planner, which is modifying (multiple) Schedule and Asset Attributes (Project Attributes are read-only) across projects in tabular fashion. This means users can copy the visible tabular data from Schedule Builder to an external tool such as Microsoft Excel, collaborate with others outside of esi.manageTM then paste the modified data back in Resource Planner’s Schedule Builder to save the schedule permanently in esi.manageTM.

 

 

Copy to Clipboard



To maintain data integrity, the copy functionality will automatically include the attribute/column set that Resource Planner deems as the key for each row, which is: Project name, Asset Name, and Expense (Expenditure Type). To access the copy function we will press the           button which will copy the highlighted/selected area of the schedule builder to the clipboard.


 

 

 

Paste from Clipboard

The paste function can be accessed through the use of the      button which will paste the information from your Clipboard into the Schedule Builder. If a user is pasting information to Schedule Builder, the column headings and the key attributes’ (Project, Asset, & Expense) values are expected for each pasted row; otherwise, the paste will fail.

 

 

 

 

Color coding/ Gantt Settings


The Gantt Setting view is located on the right of the screen, it contains four mini-areas: Bar Text, Bar Color, Bar Fill Pattern and Bar Border. These mini-areas allow the scheduled assets to be customized for the users viewing of them.

 

 

 

 

Bar Text


The Bar Text allows the user to change the text that is displayed on the activity bars, the user can choose from three choices; Project and Asset Name, Project Name only and Asset Name only.

 

 

 

 

 

 

Bar Color


The bar color functionality allows the user to custom color the activity bars using a range of rules and attributes. Once the attribute and rule type have been set, the user needs to apply a color to the rule. Either this can be a solid color, or a shaded color with a different color blended in the top and bottom of the activity bar.

 

 

Bar Fill Pattern


The bar fill functionality allows the user to customize the pattern within the activity bars using a range of rules and attributes. Once the attribute and rule type have been set, the user needs to apply a pattern to the rule.

 

 

 

Bar Border


The bar border allows the user to customize the border based on the same rule and attribute types shown above for customizing the color of the bar fill. The user can set the width and color of the border.

 

 

Rule types

There are 7 different types of rules you may use to customize the bars within the Resource Line. The rules are Default, Asset Name, Hierarchy Attribute, Custom Attribute, Expenditure Type, Read Only and Multiple Conditions

 

 

Default

Change the default color of the activity bars


 

 

 

 

Asset Name


Color code activities that share the same asset name. This requires the assets to have a common fragment within the name.

 

 

Hierarchy Attribute


Color code activities that share a common Hierarchy attribute. User needs to select the attribute name and the matched value.

 

 

 

Expenditure Type


Customize display of the activities based on the activity’s Expenditure Type.

 

 

 

 

 

Read Only


Non-editable activities

 

 

 

 

 

Multiple Conditions


These rules support complex constraints of a scheduling process. It groups multiple rules together and applies a condition to the group.
