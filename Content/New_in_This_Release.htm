Version 3.15

  1. Economics
    • Can now define Burden Calculation Logic, based on CAPEX, OPEX, Production, and other Burdens.  These can appear in the Income Statement and reduce Net Revenue, as well as Cash Flow reports and Analysis charts.
    • Can specify Royalty Price Triggers, which will show in Economics.
    • Grouping of Economics in Configuration: Default Currency, Royalties, Taxes, Burdens, and Depreciation, can now be defined in one location, and a single model including all variables can be set as default for a node in the hierarchy.
    • Alberta Royalty Framework now supported, including New Well Royalty Rate, Natural Gas Deep Drilling Program, and Deep Oil Exploration Programs.
  2. Production/Reserves
    • Define and consolidate Reserves (1P, 2P, 3P, P10, P50, P90, etc.).  This can be done by configuring Reserve Classifications for input, Reserve Consolidations for reporting and analysis, and using either Classifications or Consolidations in Reports, Economic Analysis, or Measure Export.  
    • Tabular data, in addition to being entered by real month, can be entered by ideal month (30.4 days) instead.
    • Project Reserves tab is now within the Production tab
    • Define time variant Shrinkage per Product, per Asset, on Shrinkage tab.
  3. Add Default data to hierarchy nodes, in Configuration.
  4. Project Aggregator: Aggregate multiple projects into a single project, to allow for a higher level view.
  5. Add hierarchy items to Project list table, for easier sorting and data checking.
  6. Simplify Gantt Chart, to automatically open and display Forecast data at the Project level.
  7. PEEP, Aries, and Measure Export interfaces updated.
  8. Clone Project now includes Actuals, whereas Copy Project is Forecast only.
  9. Can carry forward Income Loss, so depreciation is not lost if Economic Limit is reached prior to depreciation completing.

Version 3.12

  1. Time-variant Foreign Exchange files have been implemented.
  2. The Pricing model have been expanded to include Price Profiles which allow users to combine multiple Price Forecasts based on absolute and/or relative production amounts in order to model production contracts.
  3. Forecast Production has been migrated from a line-by-line segment basis to a compound, curves basis.  A chart has been added to the production tab in the project view to visualize these new curves.
  4. Default Ownership values can be entered for different hierarchy values, allowing you to specify a default WI (and NRI) for any configured hierarchy attribute.  This is done from the defaults node in the configuration.

 

Version 3.11.1

  1. Archives of template budgets will retain their template status.  This change includes copies of template archives.
  2. Projects cloned into a template budget will become template projects.  Conversely, cloning a template project into a non-template budget will convert the project into a real project.

 

Version 3.11

  1. Working Interest has been removed from the line Item level.  It is now inherited from Ownership.
  2. The Analysis Perspective has a new view, the Income Statement.
  3. The PEEP Interface has been updated to include new features around the importing and exporting of projects, synchronizing projects with PEEP and Importing PEEP Reports.
  4. Document Attachments have been added to a new tab in the Project Details view.  esi.manage can now store relevant reports against related projects.
  5. A new Concession Group Class has been added.  Economic Concession Groups have had their first implementation in the form of the Australian PRRT.  Curtailment Groups are now a part off the Allocation Group Class.

 

Version 3.10

  1. A new Ownership module has been introduced.  Working Interest has now been centralized and is inherited across measures and assets.
  2. Groups have now been introduced. The current implementation allows for Curtailment to be utilized across multiple projects in the group.
  3. Probability of success can now be entered as a negative number to allow for better integration with Peep.
  4. The way that esi.manage interfaces with Peep has been updated to allow for better synchronization.
  5. Production Backout has been removed.  It has been replaced by Curtailment.

 

Version 3.5/3.6/3.7/3.8/3.9

  1. In 3.9 the production calculations with respect to Production Downtime have been modified to retain the total reserves amount.
  2. The Pricing Model has been significantly extended. Price Adjustments (time series) are configured in a y = mx + b form, allowing offsets or multipliers to be defined against a reference price forecast. Adjustments can be configured on a jurisdictional basis, as well as within an individual project.
  3. Tabular Production data can now be viewed and edited in the Data Matrix.
  4. Derived Products can be defined as a ratio (single value or time series) to a Primary Production segment.
  5. The Measure Export has been expanded in terms of Grouping and Data Options.
  6. Currency Escalation (Inflation) has been added, allowing Expenditure Types and Operating Expenses to have an inflationary factor defined for future periods. Reports, Analysis Cases and Export Jobs can reference a particular Escalation File and have it applied to the calculated results.
  7. Production can now be represented and reported in terms of Quality (Heating Value or Density) in addition to Volume.
    • Product Quality can be overridden at the Production Segment level.
    • Import files (csv and xml) have been updated to support Quality.
    • Project and Measure export include Quality.
    • Quality can be accounted for in Economics calculations.
  8. Multi-Currency support at the individual Line Item. Expenditures, AFEs and OPEX lines can be entered in any Currency Unit, not just the Budget Currency unit.
  9. Configurable Metrics are allowed - including Ratios between one or more values.
  10. A Two way interface with PEEP has been developed as part of the Asset Import. This link enables an esi.manage Asset (and Asset Attributes) to be populated from a PEEP case, and updated data from esi.manage to be sent back on request.
  11. Report colours are configurable.
  12. Support for has been provided for multi-select Open and Delete from the Projects List.

 

Version 3.4

  1. All reports have been migrated to the new Format.
  2. The Data Matrix allows the Creation of New Expenditures or OPEX entries.
  3. AFE Actuals and Field Estimates can be viewed in the Data Matrix.
  4. An option has been provided on the Template Well Scheduler to force each instance to Start on the First of the Month.
  5. A new external Import tool is provided (currently OPEX only) that allows CSV imports from outside esi.manage. This import routine allows both Fixed and Variable OPEX to be imported, with the ability to create new or update existing OPEX entries.
  6. A new external Export tool  has been provided to generate a CSV based export of calculated measures, in a variety of definitions and types, for a user defined number of periods. Grouping information can be requested. This export routine can also be invoked external to esi.manage.
  7. Negative production decline rates are supported.
  8. A Fiscal Year End option has been included, which will allow reports as well as the Data Matrix and Template Scheduler to display column labels using either a Calendar or Fiscal reference.
  9. The Paste feature for Tabular Production has been extended to allow select replacement of existing data (one or more columns, anywhere in the table).
  10. Two new Production Definitions (Gross Risked Raw and Net Unrisked Raw) have been added.

 

Version 3.2 / 3.3

  1. Performance Improvements - a behind the scenes change in the calculation machinery that has provided significant performance improvements. Reports, Ranking, Exports all realize the benefits of this work.
  2. New Layout in Reports - we have cleaned up the look and feel of the reports, making them more consistent, easier to read and have thrown in a few additions: Alternating Row Shading and a Customizable Logo.
  3. Navigation and Project Set Enhancement - major improvements and further integration of Project Sets into Reports, Ranking, the Gantt and Economic Analysis to simplify getting access to the data you need.
  4. Data Matrix - New Project Editing Paradigm - a spreadsheet like data editor has been introduced which allows for editing Capital and Operating Expenses across multiple Projects at one time. The new data grid has a number of familiar spreadsheet features like Copy & Paste, Undo, Filters and Auto Filters, highlighting for Totals, etc.  
  5. After Tax Economics - in 3.0 we introduced Discounted Cash Flow and Economic Metrics including Net Present Value, Internal Rate of Return, Discounted Return on Investment, and Payout. After Tax calculations have now been added. Multiple Jurisdictions, Government and Local Taxes, Depreciation and Amortization have been added.
  6. Economics Enhancements - Save and Recall the Results from Previous Economics Runs. Configurable Templates for Cashflow Analysis.
  7. Project Ranking - multi-selection for toggling Projects in/out of the Summary. In addition, the Ranking Summary can be toggled to update Manually, allowing you to make a number of changes prior to kicking off the calculations again..
  8. Bulk Editor - a utility to quickly update many Projects at one time - Hierarchy Attributes, Project Manager, or Shift Dates.
  9. Shift Tabular Production - an Icon added in the Tabular Production Table to Shift the Onstream Date. All entries are moved accordingly.
  10. Auto Re-size Columns - you can re-size all columns in a given table by double clicking in the grey box in the upper left corner of the table.
  11. UnAFE'd amount - on the Expenditure page, each line item will display (at the bottom of the page) the amount of the Total Expenditure that is not covered by the AFE(s) associated against it.
  12. Template Project Scheduling - define Project Templates and use the Template Well Scheduler to create instances of the Template within specified time period. These Virtual Projects can be updated with a new Schedule or Project Content, or Promoted to actual Projects at appropriate points in time.
  13. Sufficient Funds Test for AFE creation - a Site Specific Feature, this allows the creation of a new AFE to be Tested against the combination of Requested AFE Amount, Forecast Amount and Existing or Allocated Amount. Different levels of roll up can be applied to the evaluation.
  14. Project Analysis View - an OPEX Analysis View has been created using the same framework as the Production and Expenditure Analysis views. In addition, a Global Analysis View has been created that will operate against the Projects List.
  15. Hide Project View Pages - the ability to hide and restore pages in a Project has been provided.
