Ranking Criteria

A Ranking Criteria file is defined independently of the Projects being Ranked. This allows a set of criteria to be defined, and then applied to different groups of projects as needed. See Project Ranking List for a description of running a Ranking file.

 

There are two folders, for Public and Private ranking Criteria Files.

  1. Public files are available to anyone, but changes can only be made by the original creator.
  2. Private files are available only to the creator.

When a new Ranking File is created, there is the option to create it as a Public File. Alternatively, a Private Ranking file can be Published to make it available to other users. See Publish Workflow.

 

New Ranking Criteria

 

1. Press the "Create new ranking criteria" button found above the file list.

 

2. Give the Ranking File a name. Specify whether the file is available to all users (public). Leaving public unselected will result in a Private Ranking Criteria.

3. Define the Custom Milestone, which you can use as a start or end date that supersedes the default milestones.

5. A Analysis Casecan be used to incorporate specific Analysis cases into ranking.

Note : if you select an invalid milestone, i.e. Economic Point Forward for an operational metric (Production), the calculations will default to the Custom Milestone for the Operational Metric Calculations.  Similarly, if an operational milestone is selected for an economic metric the calculations will default to full cycle.

 

After filling in the required name and milestone fields, click the "Finish" button.

Next, add the Metrics for the projects.

The Metric drop down list will be comprised of Metrics that have been previously created within the Configuration (see Project Metrics):

The remaining entries are used for the statistical analysis of the Project Score:

  • Weight - defaults to 1 for every metric. Higher values for a metric will skew to results towards this metric. A weight of 2 = 200 as a maximum score.
  • Ceiling (Floor) - establishes a maximum (minimum) value for the metric; calculated metrics which are above (below) this value are scored at the Ceiling (Floor). This prevents outliers from skewing the results.

Managing Ranking Files

Once a Ranking file has been created, there are a number of management options:

  • Publish - make a Private Ranking File Public
  • Edit - this allows the ranking file to be modified to adjust the Milestone dates and scaling.
  • Copy - make a copy (Private) of the selected Ranking file.
  • Delete - deletion of public or user specific private Ranking files.
