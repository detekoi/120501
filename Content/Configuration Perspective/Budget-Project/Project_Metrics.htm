Project Metrics

Project Metrics can be calculated in two locations:

  1. The Key Metrics page of an individual Project.
  2. As a Ranking Metric.

All Metrics must be Configured through the Project / Project Metrics node within the Configuration. Economic Metrics can be selected as a "source" for a Project Metrics which will make them available to Project Ranking.

 

Creating a New Metric

There are three steps to the creation of a Project Metric:

  1. Provide a Name for the new Metric.
  2. Specify the Lower is Better flag (which is used by the Ranking module for scoring).
  3. Configure the Components for the Metric.

In the following descriptions, the Horizon refers to the time period defined by the Event and the Time Period. In the screen shot above, this would be for 12 months beginning with the First Onstream Date in the Project.

The choices for each of the Drop Downs will be dependent on the particular component; however, some of the typical choices are:

 

Aggregation

  • Total - cumulative amount for the particular component
  • Daily Average - cumulative divided by the horizon
  • Final Daily Rate - the Daily Rate at the end of the horizon

 

Duration

  • Earliest Project Date - the first date found within the Project (typically the first start date of an Expenditure).
  • Budget Start - the Start Date for the Budget (within the Configuration).
  • First Onstream - the earliest Onstream Date within the Project.
  • First Reserves Booking - the earliest Book Date of a Reserves segment within the Project.
  • First Spud - the date on which the ground is first drilled.
  • Custom Milestone - this is a configurable date within a Project Ranking file.
  • First Spend - the earliest Start Date of an Expenditure within the Project.
  • First Rig Release - the date on which a rig last conducted well operations.
  • Full Cycle - this applies to Economic Metrics.
  • Point Forward - this is used when Point Forward Economics is being used, this date is set within a Ranking File.

Type and Definition are the appropriate options for Production, Reserves, OPEX and Expenditures.

    Using an Economic Metric

    Economic Metrics to be used as a Project Metric (for Ranking) must exist as a configured Economic Metric.

     

    You have the option to configure other Parameters (Full Cycle, Discount Date, etc.) differently than the Default Economic Properties.
