Expenditure Types

Every Expenditure Line in a Project must have an Expenditure Type. This association serves two purposes:

  • Filtering and Grouping - Reports can be Filtered to, or Grouped By specific Expenditure Types.
  • Depreciation - each Expenditure Type can have a Depreciation Model configured, on a per Jurisdiction basis.

The appropriate 'number' of Expenditure Types is dependent on two factors:

  1. If you plan on utilizing the Economics engine for After Tax calculations, and the Depreciation models are significantly different between Activities.
  2. The granularity that your company Reports at or Tracks. Keep in mind that Project Hierarchy (Exploration or Development) can provide a differentiation between Exploration Drilling versus Development Drilling by using the Hierarchy element for Grouping or Filtering.

Each Expenditure Types entry is made up of 3 data elements:

  • Abbreviation - up to 8 characters. This is what is used in Capital Imports to identify the Expenditure Type for a particular Expenditure line. These must all be unique.
  • Name - up to 50 characters. This is how the Type is represented in the Project, as well as within Reports. These names must all be unique.
  • Category - each Expenditure Type must belong to an Expenditure Category, which can also be used for Grouping and Filtering.

 

 

Table Options

 

To Add An Expenditure Type

  1. Scroll to the last line in the table (it will be empty).
  2. Enter an Abbreviation (up to 8 characters) - it must be unique amongst all Abbreviations.
  3. Enter the Expenditure Type Name (up to 50 characters).
  4. Select an existing Category from the drop-down list.
  5. Click outside the line to save the change.

 

Changing Expenditure Types

Once an Expenditure Type has been used within a Project, there is only one option for changes:

  • Rename the Abbreviation or Type

Note: If you Change the Group an Expenditure Type belongs to, it will result in the Existing Expenditures having a NULL Type (the previous Category is retained, but the Type no longer belongs to that Category). This can be manually cleaned up in Editable Archives, but this change will happen within closed archives as well.

Deleting an Expenditure Type that is in use is not allowed.
