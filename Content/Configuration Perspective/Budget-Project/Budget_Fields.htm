Budget Fields

 

Field

Description

Name

This is how the Budget will be identified in the Project Browser, and Reports.

Type

The type of budget will determine a number of options in different areas of the program. There are 3 types of budgets: Budget, Forecast, and Archive. If some fields in the Project screen are greyed out, it is likely because they do not apply to the current budget type. All newly created budgets are of the "Budget" type - and more will be discussed on these later.

Parent Budget

If the budget is a Forecast or Archive, this tracks what budget they were created from.

Open

Closed and Archived budgets cannot be edited, and this is reflected by the check disappearing.

Start Date

The Start and End Dates for the budget will be initialized in any new Reports, and are used to define the Horizon for calculating Project Metrics.

End Date

See above, it cannot be prior to or equal to the Start Date.

Last Actuals

Represents the 'end date' of the last 'Actuals' (CAPEX, AFE, Production, OPEX) that are in the budget. This is set by the administrator, and is used by default as the Last Actuals Date for blending.

Last Field Estimate

See Above.

Create Date

The date that this budget was created in the system, it is unchangeable.

Currency

The type of national currency the budget figures work in. This is especially important in international organizations. *The values are not automatically converted if you change the currency halfway through. If you create a budget with Canadian dollars as the primary, with $10,000 listed for an expense, then send it to an office in Europe and change the currency to the Euro, it will show $10,000 Euros budgeted for the same project.

Default Currency File

Automatically set to "default", contains conversion information between currency.

Product

The primary product of the budget. Default products are Oil, Gas, NGL, Liquid, Sulphur, and Condensate.

Production Unit

The unit of measure used to show the quantity of production.

Reserve Unit

The unit of measure used to show the quantity of Reserves

F&D Unit

The unit of measure used to show the quantity forecast by the F & D estimates.

Capital Efficiency Unit

The unit of measure used to show the Capital Efficiency.

Depth Unit

Unit of measure for well depth.

Visible

Whether the Budget is to be visible in the Project Browser, or for Reports.

Lock Project Code

Only available for automatically generated Project Codes - this will prevent the Code from being changed.

Project Code Prefix

A fixed Prefix can be used as part of the Project Code - for example, the year could be used to identify when the Project was created in esi.manage. This can be modified as needed, however, it will not update the codes for existing Projects.

Project Code Seq Width

The number of digits long your project code will be after the Prefix.

Lock Asset Code

Only available for automatically generated Asset Codes - this will prevent the Code from being changed.

AssetCode Prefix:

A fixed Prefix can be used as part of the Asset Code - for example, the year could be used to identify when the Asset was created in esi.manage. This can be modified as needed, however, it will not update the codes for existing Assets.

Asset Code Seq. Width

The number of digits long your Asset code will be after the Prefix.

Read Only

This is generally applied against Editable Archives (any closed Budget cannot be edited). This allows the Archive to remain Editable, but only when the administrator enables this.

Lock Production Actuals

This prevents Production Actuals from being modified by any user.

Lock Financial Production Actuals

This prevents Production Financial Actuals from being modified by any user.

Lock CAPEX Actuals

This prevents CAPEX (and AFE) Actuals from being modified by any user.

Lock OPEX Actuals

This prevents OPEX Actuals from being modified by any user.

Template Projects Only

This identifies the Budget as being the 'source' for Projects to be created by the Template Well Scheduler.

Escalation File

Defaults to (no escalation). This will be the Default Escalation file to be used for the Budget.

Base Currency Date

If escalation is being used, this defines the reference date that corresponds to Escalation file.
