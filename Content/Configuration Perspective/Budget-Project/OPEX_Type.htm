OPEX Type

Every OPEX Line in a Project must have an OPEX Type. This association allows Filtering and Grouping - Reports can be Filtered to, or Grouped By specific OPEX Types.

The appropriate 'number' of OPEX Types is dependent on the granularity that your company Reports at or Tracks.

Keep in mind that Project Hierarchy (Base Production or Development) can provide a differentiation between the Processing Costs for Base producing wells, versus new wells by using the Hierarchy element for Grouping or Filtering.

Each OPEX Types entry is made up of 3 data elements:

  • Abbreviation - up to 8 characters. This is what can be used in OPEX Imports to identify the OPEX Type for a particular line.
  • Description - up to 50 characters. This is how the Type is represented in the Project, as well as within Reports. The same Description can appear in different Categories, provided the Abbreviations are unique.
  • Category - each OPEX Type must belong to an OPEX Category, which can also be used for Grouping and Filtering.
  • Nominal - define whether the OPEX type is subject to reporting and analysis in Nominal or Real basic.

To Add An OPEX Type

  1. Scroll to the last line in the table (it will be empty).
  2. Enter an Abbreviation (up to 8 characters) - it must be unique amongst all Abbreviations.
  3. Enter the OPEX Type Description (up to 50 characters).
  4. Select an existing Category from the drop-down list.
  5. Click outside the line to save the change.

 

Changing OPEX Types

Once an OPEX Type has been used within a Project, there is only one option for changes:

  • Rename the Abbreviation or Type
  • Deleting an OPEX Type that is in use is not allowed.
