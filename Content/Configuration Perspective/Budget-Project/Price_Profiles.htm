Price Profiles

The purpose of a Price Profile is to tie a Price Forecast to a production stream at the project level.  A Price Profile is made up of one or more tiers.  A tier ties a Price Forecast to a specific relative or absolute value of production.  The minimum number of tiers that a Profile requires is one.

 

Adding Price Profiles

Adding a new Price Profile can be done by clicking on the “Add a Price Profile” button  

After clicking here you will see the following dialog box appear, with empty fields. In the example screenshot below, the following fields have been filled in:

  1. Profile Name – This name must be unique amongst other Profile names
  2. Fiscal Model – This is the level at which you are creating the Profile. If you choose Global, it can be selected as the Price Profile for any Project in your database. If you choose to set it for a specific Jurisdiction, you cannot use this Profile for Projects that do not fit underneath that Jurisdiction.
  3. Production Unit & Currency – These are used to filter down the list of Forecasts available to choose from in the bottom window of this dialog box. You will only see Forecasts that use the same values available as options to populate the Price Profile with prices.
  4. Default – Check this box to enable the bottom window. This is required if you want to automatically populate the Profile with the prices laid out in a Price Forecast. The options that are available to you to depend on the production unit and currency selected above.

You can also add a comment to the profile to add more details to the profile.
