Project Hierarchy

The Project Hierarchy within your esi.manage database has been implemented to match your company's organizational structure and business processes. As your business moves forward, additional Hierarchy Elements may be required, and potentially Hierarchy Types. The Hierarchy is used extensively in esi.manage: Reports can be Filtered to, or Grouped By specific Hierarchy elements. The Project Browser, and by extension Project Sets, use these elements for Filtering.

The Hierarchy Configuration utilizes linked Tables:

  • Hierarchy Type is the Group Name - which is what will appear as the label in the Hierarchy for a particular Project.
  • Hierarchy Elements are the Instances - the options that will be in the drop down list for a particular Hierarchy node.

The Hierarchy system supports Nesting - for example the Field Hierarchy Type has Reservoir as the Parent. This means that for each Field Element entered, the Parent Reservoir Element must be selected. This allows esi.manage to Filter the lists accordingly within the Project - once a particular Reservoir has been selected, only Fields with that Parent will be shown in the Field drop down.  See the bottom of the page for more information on Hierarchy Element Defaults.

  • Required means that this is a Mandatory Hierarchy Type - an Element must be selected within a Project for it to be created.
  • The Sort Order determines in what order the various Hierarchy nodes  (Types) will be presented within the Project, as well as within the Browser.
  • Include Code - this toggle indicates that the Hierarchy Element Code is to be included in the data that is being Exported via the Project Export.
  • Included in Integration - this toggle indicates that the Code can be used for text matching purposes within the Integration Perspective.
  • Included in Project Browser - By clicking this checkbox, the hierarchy type will be displayed as an additional column within the project list.
  • Inactive Flag: This field lets the user turn off old hierarchy elements that cannot be deleted due to being used in locked or archived projects. Inactive hierarchy elements will not show in the project browser, project sets, or as Groupings in Reports.

 

 

Table Options

 

To Add A Hierarchy Type

  1. Scroll to the last line in the table (it will be empty).
  2. Enter the Hierarchy Type Name (up to 50 characters).
  3. If this is a Mandatory entry, check Required
  4. Set the Parent (if applicable)
  5. If you want to include the Project Code (see above), click "Include Code"
  6. Finally, set the sort number.
  7. Click outside the line to save the change.

 

To Add A Hierarchy Element

  1. Select the Hierarchy Type in the top table.
  2. Scroll to the last line in the Hierarchy Element table (it will be empty).
  3. Enter the Hierarchy Code (up to 8 characters)
  4. Enter the Description
  5. Select the Parent. This will be the filtered list of Hierarchy Elements for the Parent Hierarchy Type.
  6. Click outside the line to save the change.

 

Changing the Hierarchy

Once a Hierarchy Types and/or Elements has been used, the Name, Code, Description can be edited. However:

  • Deleting it is not allowed
  • Changing the Parentage will generally not work because of conflicts with existing Projects

 

The Project and Asset Attributes capability provides a similar type of functionality to annotate a Project with technical, organizational or business information.

 

Hierarchy Defaults

All fields of the hierarchy are required to be filled out for every new project.  To save time selecting hierarchy elements each time, you are able to set a hierarchy element as the default for a particular hierarchy type.  Defaults are denoted by a tilde (~) before the hierarchy element by default.  In the image below, ~No Program is the default Hierarchy element for the Program Hierarchy type.

 

In the situations where there are nested hierarchy types (is. There is a parent/child relationship between two hierarchy types), you must set different default hierarchy elements for each parent hierarchy element.
