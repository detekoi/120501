Reserve Classifications

Configuring Reserves

esi.manage supports configuring Reserves to support a number of use cases.

 

Cumulative Function

By leaving the 'Reserve classifications are incremental' box unchecked when production is entered into the production tab of the project editor, the production data will automatically be populated into the next reserve classification.

This supports:

1P / P90 = Proved Reserves (P1) (Proven developed, Proven Undeveloped, Proven Developed not producing))

2P / P50 = 1P + Probable Reserves (P2)

3P / P10 = 2P +  Possible Reserves (P3)

 

 

Above is a very simplistic example. By entering in 100 into '1P', 100 is also populated into the next classification of '2P' which is displayed in green font to show it was automatically populated. This saves the user from entering the same data into each of the reserve classes as often 2P will be the same as 1P as there is no Probable reserves etc.

We then entered 150 into 3P which is why that column is not green.

Incremental Function

However, in other configurations which need to split out the P1 (Proven Reserves) into incremental layers, it would not make sense to auto-populate the data from one reserve classification to the next.

For example:

P1 - Proved Developed Producing (PDP)

P1 - Proved Undeveloped (PUD)

P1 - Proved Developed Not Producing (PDNP)

P2 - Possible Reserves

P3 - Probable reserves

 

 

In this example the production will not automatically be populated a P1 Reserve Classification to another.  However, P2 and P3 do not have the incremental box checked (not shown), and so will automatically be populated by whatever entry is in any of the P1 Classifications.

 

Reserve Consolidations

Reserves are reported and analyzed in the esi.manage reports and Production analysis view using consolidations. Within each Reserve category, you may wish to model the High (P90 / 3P), Medium (P50 / 2P) and Low scenarios (P10 / 1P) and calculate the mean of the distribution using a Swanson’s mean type consolidation.

There is also a consolidations tab. This contains all the reserve classifications from all tabs to allow the user to create global consolidations if required.

 

Expected Value

On the Reserve Consolidations section, you are able to choose a Reserve Consolidation that is your Expected Value (EV).  The EV is denoted by a gold star.

The EV is important when using the Reserve Progression Feature in Reports. If you choose the Reserve Progression option, as time progresses any Reserve Consolidation you choose to report on (eg. 1P) will progress towards the EV value.

 

Parametric Mean definition

  • The Parametric Mean Definition is used to calculate the mean of the parameters and then calculate the profile as opposed to calculating the mean of the profiles
  • The advantage of this is that it will calculate the mean end point rather than having to take the P1, P2 or P3 points

  • Set up the Parametric Mean in the Reserves Configuration for each Classification.
  • Like all consolidations it can be used for Reporting and Charting

Note: Parametric Mean should only be used for pure Formulaic data – not where Tabular is used

 

 Note: In addition to the above configuration, the default economic properties also requires the reserve consolidation to be set.
