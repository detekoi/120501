Defaults

The Default section of Configuration allows the user to establish global settings within their organization based on corporate or regional breakdown, or other user-defined hierarchy elements.

Default reduces user entry time and data entry errors by automatically filling in information for an end user, once they have created a project within an area of the organization where defaults apply.

 

Overriding Defaults

Consider the simple structure below for the following example:

1. Country

a. State/Province

i. Area

> Project

Defaults are designed to work so that information entered at a deeper level of detail will always override the defaults from the more general level. So if a default was entered for both Country and Province, Province (being a higher level of detail) would be the one applied.

Ultimately, the user can always override a default at the Project level, so they are not forced to use the information set up in Configuration.
