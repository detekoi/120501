Production Defaults

Production Defaults allow an organization to set global values based on a project’s position in the hierarchy for a number of entry fields on the Production screens. Through this method, an organization can set pre-entered values for Downtime, Derived Products, Shrinkage, and Reserves Booking Schedules.

The process for entering each is the same as entering their respective information on the Project Editor screens.

 

To Add A Default Production Value:

1. Open Configuration > Defaults > Production

 

 

2. Select the Hierarchy node against which to assign the defaults (note: this may be read only if defaults already exist)

 

 

3. Select which component of Production to enter a default for by selecting the appropriate tab along the top of the screen:

 

 

4. Select which node in the desired hierarchy you are going to assign values against:

 

5. Enter the default as you would in the Project Editor

Notes:

  1. If you need to change the main hierarchy node at any time, you must first remove all default records assigned to the existing hierarchy selection.
  2. Lower levels override higher levels, so in the example above a project in Texas will use the default set for Texas, rather than one set at the USA level.
  3. For defaults to function, the projects MUST be assigned to the hierarchy selected to drive that default criteria.
