Ownership

Ownership or working interest can be configured for each project in the Ownership tab of the project editor or a default can be set up so that ownership is applied across all projects that have the associated project hierarchy attributes.

For example, the screen shot below shows the Ownership screen in the defaults folder of the configuration. The Hierarchy selection page (Configuration>Defaults>Hierarchy Selection allows you to determine the level that default ownership is applied. For example, you could base it on the Jurisdiction or the region of the hierarchy etc.

 These hierarchy nodes are configured in the Project Hierarchy page in Configuration>Budget Project>Project folder.

 

You can enter one, two or all three of the hierarchy selections. For example, in the example above, I could select Cecilia as the region and not select an area or asset team. I could then apply a WI% which would be applied to all projects with the Cecilia selected as the region in the project hierarchy.

By changing the selection in the Ownership Hierarchy settings list, the associated hierarchy setting will be displayed in the lower table where the WI% is set.

You can then add ownership WI% by Hierarchy to all measures or specifically to different Expenditure types, products or Fixed OPEX.

By typing a WI% value into the cells, esi.manage will automatically populate the same number for all subsequent years and months.

To change from an annual to a monthly view, click the 'monthly' button in the top right hand corner of the lower part of the Ownership screen.
