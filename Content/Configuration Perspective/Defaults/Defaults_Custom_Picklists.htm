Default Custom Attributes Picklist Filters

The default screens for Project and Custom Attribute Picklist Filters provide specialized functionality for companies with long lists of custom attributes.

These screens are used to filter the selection of attributes shown in a given picklist, based on the hierarchy settings of the project they are in.

For example, a company with many Reservoirs may create a custom attribute to identify the reservoir(s) that project is targeting. The Picklist defaulting screen limits the list of Reservoirs to choose from based on where a project is located geographically, and which reservoirs are a valid choice for that region.

 

To Add A Default Attribute Filter:

1. Open Configuration > Defaults > Asset (or Project) Custom Picklist Filter

 

2. Select the Hierarchy node against which to assign the defaults (note: this may be read only if defaults already exist)

 

 

3. Select which node in the desired hierarchy you are going to assign values against:

 

 

4. Select the Custom Attribute Picklist to apply a filter against:

 

 

5. From the options contained in that picklist, select which value(s) apply to the hierarchy node you selected. In the example below, any project created in Canada > Alberta > Cecilia will have the options “ADANAC”, “ALLAN BAY”, “ANNVILLE”, “ARCTOMYS”, “ARTEX-HALFWAY” and “ASHERN” shown in the picklist the next time a user opens it.

 

 

6. Press “Save” or click on another screen in configuration to save the changes.

Note: You can assign filters for multiple picklists by picking another picklist from the drop-down after completing the first. HOWEVER, all filters must share the same hierarchy root node (Region in the example above), you cannot assign Attributes from one picklist against one root hierarchy, and another picklist against a different root hierarchy.
