Fiscal Model

The Fiscal Model page is where default Royalties, Taxes, Burdens, Expenditure Depreciation, and Custom Economic Calculation Spreadsheets are configured to be applied across a Jurisdiction or Economic Group.

Jurisdictions and Economic Groups are a mandatory project hierarchy fields. You can add and delete Jurisdictions and Economic Groups in the Project Hierarchy page of the Configuration perspective.

Countries are normally configured as Jurisdictions and local areas with their own tax, royalty and burden regimes are usually configured as Economic Groups. Economic groups belong to a Jurisdiction so are lower in the hierarchy.

 

In the screen shot above, you can see the fiscal model page with a 'Fiscal Model Hierarchy' showing Jurisdictions at the top level and Economic Groups under each Jurisdiction. The list of Jurisdictions and Economic Groups can be expanded and contracted by clicking on the twisties that appear when you hover your mouse to the left of the green tick besides the Jurisdiction name.

The green circle with a tick icon denotes that data has been written to the Jurisdiction or Economic Group. To clear the data in the fiscal model, select the drop down arrow next to the refresh icon and select 'Clear Fiscal Model Data'.

 

By selecting either a Jurisdiction or an Economic Group in the Fiscal Model Hierarchy, you can then populate the Royalties, Taxes, Burdens and Expenditure Depreciation for the particular Jurisdiction or Economic Group you have selected.

 

Overview

This tab provides a snapshot of the currency, royalties, taxes, burdens and depreciation and where they are inherited from. For example, the screen shot at the top of this page shows the Fiscal model defaults for Alberta which is under Canada in the hierarchy. Therefore any esi.manage projects that are configured with Canada and Alberta in the details page, will automatically inherit the royalties, taxes, burdens and expenditure depreciation for Canada as well as Alberta.

Depreciation is being inherited from Canada, this is because a depreciation model has been configured for Canada but not for Alberta. In contrast, the tax is being inherited from both Canada and Alberta as they both have a tax model configured.

 

Royalties

Royalties can be configured at both country / federal level and at the state / local level. There are two ways that default royalties can be configured in the Fiscal Model page of the defaults section. The first is a fixed percentage by product, month / year and price scenario (See figure below).

Royalties can also be input, in a similar manner, within the Project perspective under a Project’s Burdens tab. These will override the configured royalty amounts. Note that these amounts are input in %.

The second is by price triggers, as discussed below.

 

Royalty Price Triggers

The second method for allocating default royalties is by price triggers.

In the screen shot below shows the Trigger sub tab within the Royalties tab. The default royalties are applied in 'Price Trigger Tranches' so you can model changing royalties rates as prices change.

To configure a price trigger, in the price triggers section of the screen, click into the name field and give the price trigger a name, select which product it is for and the price profile that it will be applied against. One this is completed, the currency and volume unit should be populated automatically.

Next, in the 'Price Trigger Tranches' table, set the price triggers and the associated royalty percentage.

A chart will automatically be plotted to show a graphical representation of the Price Trigger.

 

You can also see that there is a Royalty time line section at the top of the page. This allows the configuration to inherit either the percent or trigger mechanism to a certain date and then a different royalty mechanism or provider up to another data. So the mechanisms for Royalty calculation can change over time.

Taxes

The taxes tab allows the creation of one or more taxes that are applied against operating income.

There are three buttons.

  1. Configure Taxes - click here to create a new tax. Click on 'add tax' and give the new tax a name. Click OK and the new tax will appear as a column on the front of the taxes tab.
  2. Yearly - This is toggles the date range between monthly and annual. Once you have selected the desired periodicity, type in the tax percentage for the new tax type you have created. The number you type in to either the monthly or yearly cell will automatically fill down until you type in a new number.
  3. Refresh

 

Burdens

Burdens can be applied to all projects and assets within a Jurisdiction or Economic Group. The burdens tab is configured in the same way as the custom burdens in the burdens editor within a project (with the exception of the “Allow Asset to Override Schedule” option, discussed below).

  1. Configure Burdens - allows you to create different types of burdens. Pressing the add button provides a list of options to select.
  2. Filter Table -  move the burdens to fit customizable configurations.
  3. Yearly -  see Taxes
  4. Refresh - see Taxes

There is an additional flag when configuring default burdens which is the Allow Asset to Override Schedule flag. While this flag is checked, burdens can be configured at within the Project Editor at an asset-level to override the defaults.

The schedule is completely overwritten while using asset-level burdens, but can be restored to the defaults by clearing the schedule.

The schedule will have a yellow background if it is inheriting default values from the fiscal model.

Expenditure Depreciation

Expenditure Depreciation can also be applied to multiple projects by default in particular Jurisdictions and / or Economic Groups.

Each Expenditure Type is grouped so that depreciation can be applied in the same way to all expenditure types in the group. This saves time as you don't need to configure a depreciation method for each expenditure type.

Depreciation uses the Expenditure Family called Tax Family. If the tax family is not selected, the depreciation will not be calculated and won't show up in the cashflow statement in the analysis perspective.

 

As with the other tabs in the fiscal Model page, the first step is to select the Jurisdiction and / or an Economic Group in the 'Fiscal Model Hierarchy'. Then select the Tax family and the Expenditure group.

Each Expenditure group is represented as two 'types': Tangible and Intangible. These two parts are configured independently for each Expenditure group, however they are connected in that the Total Tangible % + Total Intangible % will always equal 100.

For each Expenditure/Jurisdiction, the values that can be entered are :

  • Family - The Expenditure Family to apply the Depreciation against.  
  • Group - Used to Depreciate a Group of Expenditure Types.  If left blank, the Depreciation will be used as a default for all Expenditure Types, unless otherwise specified in the Depreciation Schedule.

 

Tangible and Intangible Types both have the following selections:

  • Type - Depreciation models (Declining Balance, Straight Line, Sum of Years). These models determine the amount of the original capital cost that is eligible for deduction from the Net Income at the point in time as defined by the Convention.
  • Annual Decline % - Annual Depreciation rate (if Declining Balance is selected as the Depreciation model.)
  • Useful Life (Years) - The duration the Depreciation is applied over.
  • Convention - Defines when the Depreciation calculations are performed - Mid Month, Mid Quarter or Mid Year.

 

The values that are defined for Depreciation in the Configuration are applied by default to any Expenditure (based on the Jurisdiction and or Economic Group assigned to the Project).

Depreciation can be overridden at the Project level in the Expenditure Tab.
