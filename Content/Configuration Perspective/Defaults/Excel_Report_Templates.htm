Excel Report Templates

The Excel Reporting Feature allows generating reports based on the information generated in the Economics Analysis Cases in esi.manage.

Any Excel report format is possible,giving the ability to create custom Excel reports.

 

Excel Report Template Process

There are 4 steps to create and run Excel Reports:

  1. Creating and loading an empty report template
  2. Running an Economic Analysis Case and generating the first report
  3. Creating the custom report template
  4. Generating custom reports

 

Creating and loading an empty report template

  • The first step is to create a spreadsheet with the following worksheets:

 

    "ESI Summary (nominal)”: contain the output of the Summary tab in the esi.manage Analysis case in Nominal numbers.

    “ESI Summary (real)”: contain the output of the Summary tab in the esi.manage Analysis case in Real numbers.

    “ESI Cashflow (nominal)”: contain the output of the Cashflow tab in the esi.manage Analysis case in Nominal numbers.

    “ESI Cashflow (real)”: contain the output of the Cashflow tab in the esi.manage Analysis case in Real numbers.

    “ESI Metrics (nominal)”: contain the output of the Metrics tab in the esi.manage Analysis case in Nominal numbers.

    “ESI Metrics (real)”: contain the output of the Metrics tab in the esi.manage Analysis case in Real numbers.

    “ESI Inflation”: contain the cumulative inflation time series values used in the esi.manage Analysis case, based on the inputs defined in the Currency Inflation page of the Configuration Perspective.

    “ESI Metadata”: contain the report timestamp, user, economic node, and fiscal path of the output.

    “ESI Prices”: contain the time series values of the Price Forecast specified by the user

    “ESI Projects”: contain the list of projects that is part of the esi.manage Analysis case, as well as their comments and fiscal model hierarchy path.

    Any other tabs can be added, free worksheet name and contents. Any formula can be added referencing data in the tabs with the “ESI” prefix.

    Aside from the “ESI Cashflow”, “ESI Inflation”, and “ESI Prices” tabs (see Reference for details on the column header syntax to bring in user-specified measures and economic measures), spreadsheets start out blank at this stage.

 

For the “ESI Cashflow” tab (applicable to both Real and Nominal), users can edit the column headers in rows 1, 2, and 3 of the sheet to specify which economic data to import and include in the report. For any given column, row 1 would contain an “IN” value if the data type specified in rows 2 and 3 is meant to be imported, or remain a blank cell if nothing is intended to be imported into this column. Row 2 contain the data type name that the esi.manage report engine would recognize (see Reference for data type name syntax), and row 3 contain any additional information to specify the data type to include (such as product name, expenditure type, opex type).

 

• The report file needs to be added in the Configuration Attachments page in the Configuration perspective and attach the file. The Type must be specified as “Economic Report” in order for the attachment to be used as an Economic Report Template. This is a repository of Attachment files and can be found at Configuration>Defaults>Configuration Attachments.

NOTE: Any file loaded to Configuration Attachments is a copy has been loaded in the esi.manage database. Changes to the source file will not affect the attached copy.

  • The Excel file must then be assigned into the list of Excel Reports.
    • Go to Configuration>Defaults>Except Report Templates
    • Click the + icon
    • Go to the last line of the list and Click on the “Spreadsheet” field and select the Excel file that will define the report.
    • Assign a name for the report in the “Name” field.

Running an Economic Analysis Case and generating the first report

  • Go to the Analysis perspective and run an Economic Analysis Case.
  • Once an analysis has run, a report can be viewed on the Cashflow tab by right-clicking on any node in the cashflow output. Select the “Generate Report” option and select a report to view the output. A report can also be generated from the Metrics tab in the same manner.
  • Save the file and open it in Excel. The tabs defined in the report (the "ESI" tabs) are not populated with the values from the node that was selected on the Cashflow tab of the economic analysis.

    The other tabs (that will contain the custom report templates) are still blank.

Generating custom reports

Every time a report is run, the "ESI" tabs will be populated for the node selected in the economic analysis and the custom template tabs will be based upon that data. The "ESI" data tabs can be hidden to avoid clutter.
Note that all the custom defined tabs contain not only the data but also the formulas referencing the data in the "ESI" tabs. This can be used for error checking and/or auditing.
The report is independent of the project sets and analysis cases used in the Analysis Case. A report can be generated for any of them.

 

Variance Analysis

Variance analysis can be utilized by generating a report for each scenario, then comparing the reports.

 

Economics Spreadsheet

If economics is run using a fiscal model defined in a Economics Spreadsheet (Custom option in the Configuration>Defaults>Fiscal Model) the Cashflow tab of the report will also contain the columns corresponding to the OUT lines of the Economics Spreadsheet.

The custom defined tabs can reference the Economics Spreadsheet generated data.
