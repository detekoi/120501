Measure Scaling

Scale Files are used to scale measures when running Analysis Cases, Expenditure Analysis, Opex Analysis or Production Analysis.

The Measure Scale works by selecting a hierarchy type from the drop down menu, and then one or more specific values of that hierarchy type on which selected measure or measures (Expenditures, Opex, Production) will be scaled when using this Scale File in an Analysis.

Any number of Scale Files can be configured combining Measure Scale Hierarchies values and Measures.

 

To create a Scale File:

  1. Open Configuration > Defaults > Measure Scale.
  2. Select the Hierarchy node against which to define the scaling.
  3. Select which node in the desired hierarchy the scale will be defined at.
  4. Click on the plus sign.
  5. Give it a name.
  6. Now the new Scale File is defined and is selectable in Scale File drop down menu.
  7. Once a file is selected, scales cab be defined against measures.The Scaling Factor is entered as a factor of the measure. For example, a factor of 1, (which is the default factor in a newly created Scale File) will leave the measure as it is. A factor of 2, would double the measure value, a factor of 0.5 would halve the measure value, and so on.
    Any scaling factor can be applied in any measure and type measure combination.
    Note that the measures that have a scaling factor applied, would be marked with a pen beside it.

Scales can be defined at lower levels in the hierarchy. In this case, the scaling that are defined will be compound with the scales defined in levels above.

I.e, if a scale of 1.15 is defined at a certain level and in the level above a scale of 0.85 is defined, the effective scale would not be 1.15, but 0.9775 (1.15 * 0.85 = 0.9775 )

To show the effective measure (this is, the result of inheriting and combining scales from levels above) the button “Effective Scale” must be clicked.
If the button is unclicked, the number shown will be the defined in that particular level (‘1’ by default).

In the following example (see next screenshot), we have defined for Region “USA” and country “US” the Scale factor of 1.15 for all Expenditure Types, except for the Completion type in which we have defined a Scale Factor of 0.85.

If the button “Effective Scale” is unclicked, the values shown are the ones defined in that particular level (‘1.15’ for all Types, ‘0.85’ for “Completion” type, and the defaulted ‘1’ for the rest)

But if the button “Effective Scale” is clicked, the numbers shown will be the result of inheriting and combining all scales that apply.

In this case the “All Types” will show ‘1.15’ (because we entered it) and the rest will show 1.15 (because they inherited and combined it with ‘1’, 1.15 * 1 = 1.15). The type “Completion” will show 0.9775 (because we it will inherit 1.15 from All Types, and combined it with the 0.85 that we entered, so 1.15 * 0.85 = 0.9775).

The Effective Scale are the actual numbers that will be applied in the Analysis when this particular Scale File will be used.

Note that when the button “Effective Scale” is clicked, the table is read only.
