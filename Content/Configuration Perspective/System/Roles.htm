Roles

The role users are assigned will define what capabilities the user has within esi.manageTM. There are two steps to assigning a role: the first step is to define the permissions of an individual role in the "Roles" screen. The second is the assign the user to the role in the "User" screen.

 

To Create a Role:

  1. Access the Role in the Project Perspective.
  2. Select "Add Role"
  3. Give the role a name.
  4. Click the boxes for the permissions you want the users assigned to that role to have.

*Users also have the ability to Rename, Duplicate or Delete an existing role

 

 

Options for permissions:

  • Configuration - Read: This determines if the user can access the configuration perspective. Leaving this unchecked will prevent them from doing so.
  • Configuration - Write: This determines if the user can make changes to configuration settings. Leaving this unchecked prevents them from changing any configuration settings.
  • Configuration - Import: this determines if the user can import into configuration settings.
  • Public Admin: This provides the user with administrator control of previously created Reports, Ranking Criteria, Project Sets, Analysis Cases and Export Jobs in order to either delete them or remove the from the PUBLIC view.
  • Projects - Read: This determines if the user is able to access the Projects perspective. Leaving this unchecked will prevent them from accessing it.
  • Projects - Write: This determines if the user is able to edit data in the Projects perspective. Leaving this unchecked will prevent them from making changes to any of them.
  • Archive Projects - Write: This determines if the user is able to edit data in the Projects perspective for budgets in Editable Archive mode. Leaving this unchecked will prevent them from making changes to projects in "Editable Archive" budgets.
  • Integration Read: This determines if the user is able to access the Integration perspective. Leaving this unchecked will prevent them from accessing it.
  • Integration Write: This determines if the user is able to edit data in the Integration perspective. Leaving this unchecked will prevent them from making changes to the Integration data.
  • Integration Admin: This determines if the user is able to edit not only their own data, but all data in the Integration perspective. Leaving this unchecked will prevent them from being a super user in the Integration perspective.
  • Publish: This determines if the user can create public Project Sets, Reports, Analysis Cases, Export Jobs or Peep/ValNav Import Jobs. If this is unchecked, they will only be able to create items that are private.
  • Reports - Write: This determines if the user is able to edit Report parameters and/or create reports. Leaving this unchecked will prevent them from doing so.
  • Operational Configuration - Read: This determines if a user can view the defaults section in configuration.
  • Operational Configuration - Write: This determines if a user can edit the defaults section in configuration.
  • Capex - Write: This determines if a user is able to modify Capex data.
  • Opex - Write: This determines if a user is able to modify Opex data.
  • Production - Write: This determines if a user is able to modify Production data.
  • View: The view options grant the user to see the various tabs of a project. Roles can be set up to have access to a select number of tabs within the project editor.

*For a user to edit measure information, they will need both Project write access AND measure write access

*History sub-tabs will respect individual’s measure level permissions

 

To assign a user to a role:

  1. Access the User screen in the Configuration Perspective.
  2. Find the user's window login name in the list.
  3. Double-click the right column in the table.
  4. A list will appear of available roles.
  5. Select a role off the list.
  6. Click somewhere else in the table to save the change.

User titles and permissions will vary by system setup. You can add and rename them to suit your purposes.
