Remote Systems Configuration

This functionality allows multiple databases to transfer complete projects from one database to the other. For the import to be successful, configuration of the source and target database must be identical. The import mechanism works off of Project Sets.

Pre-requisites:

  1. Configuration between source and target databases are identical
  2. Web Services integration must be properly configured and enabled
  3. Target database must be identified in Configuration>System>Miscellaneous>Remote Systems Configuration

Overview

Once all the pre-requisite infrastructure is in place the steps to import are as follows:

  1. Identify target database, then navigate to the Configuration Perspective and open Remote Systems Configuration.

  2. Select "Add" and populate all necessary information.
    • Remote System Name - Text field used to identify system connection info in subsequent steps
    • Base URL - URL connection info to Web Services
    • Project State Attribute - Custom Project Attribute created in both databases. Used to help users identify whether projects in the source database are ready to be brought in to the target database
    • Project State Attribute Value - Pick list values of custom attribute defined above.
      **All fields are mandatory

  3. Once Remote Source is configured and all other pre-requisites have been completed, open the Import Wizard through File>Import and select Project Imports>Projects From Remote Service.


  4. Once the wizard is open you will be prompted to select a Remote System (defined in Step 2).
  5. If all the connections are properly set up, the wizard will prompt to select a Project Set and a Target Budget. Project Set selection will be populated from your source database and Target Budget selection from your target database.
  6. Projects in the selected Project Set will now appear in the wizard. Remote State shows the value of the custom attribute (defined in Step 2) per project from the source database and if applicable Local State shows the values of the custom attributes in the target database.
  7. Select Projects to be imported and press Next.


  8. Once completed a prompt containing the results appears. If import tails, an error message will prompt with what needs to be corrected.
  9. Once successful, all projects will be populated from source system into target budget defined.
