User / Project Managers Access

Project Manager Groups

All esi.manage projects must be assigned a Project Manager. A set of projects with the same project manager forms a security group (a project manager group). Users are explicitly assigned read or write permissions to each project manager group.

If, for a project manager group, the user has:

  • No read or write access
    • User will not see any of the group’s projects. The group’s projects will be invisible to the user
  • Read access but no write access
    • User can see but not edit any of the data in the group’s projects
  • Read and write access
    • User can see and edit the data in the group’s projects

Setting Project Manager Group Permissions

Project Manager Group permissions can be set by user or by project manager. Both methods are described below. The information is only configured once - these views simply present the same information from different perspectives. A user’s Role may over-ride the project manager group settings. A Read Only user will only be able to view Projects, even if they have been given Write Access to a Project Manager group.

By User:

  1. Select a user in the drop down list. The table will list all Project Managers
  2. Set read or write permissions on each Project Manager as desired

By Project Manager:

  1. Select a Project Manager in the drop down list. The table will list all esi.manage users
  2. Set read or write permissions for each user as desired

In the screen shot below, three users have different access to the projects of Project Manager Chandra Chanel. Curtis Tuckey can only read the projects. Ian McKay can read and write to the projects. Dane Meister can neither read nor write to the projects.

 

Toggle all read and write permissions by using the provided icons.       

The actions are Read All, Read None, Write All, Write None respectively. (Hover the mouse over the icon to see a tooltip)
