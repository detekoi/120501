Alberta Royalty Framework

This page sets all the global defaults and unit mapping that must be configured to calculate royalties. Alberta must be selected from the Fiscal Model Hierarchy list to be able to set fiscal model level defaults.

There are four functional areas on the configuration page:

 

1. Royalty Product Configuration

These products will need to be configured and mapped to calculate AB royalties.

Note: multiple products can be mapped to each product bucket if necessary.

Not all products need to be mapped to run Alberta royalty calculations.

 

2. Production Unit Binding

These units will need to be mapped when configuring Alberta Royalties. If a unit does not exist in the drop-down Pick-lists, it can be created on the Production Units Page.

 

3. Par Price Forecast Binding

Par price forecasts must be configured in appropriate units for the Royalty Model to run. The Alberta Royalty Framework (ARF) calculates Royalties in GJ for gas and cubic meters for oil, C3, C4 and C5. Select these par prices from available price forecasts.

 

4. Parameter Binding

Alberta royalties relies on a variety of asset-specific variables to be calculated correctly. This section allows the custom asset attributes to be mapped within esi.manage. See Alberta Royalties Custom Asset Attributes for more details.
