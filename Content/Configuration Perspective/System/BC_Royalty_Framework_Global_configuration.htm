BC Royalty Framework Global configuration

This page sets all the global defaults and unit mappings that must be configured to calculate royalties. British Columbia must be selected from the Fiscal Model Hierarchy list on the left of the page:

 

 There are four functional areas on the configuration page:

 

1. esi.manage Unit Mapping

These units will need to be mapped when configuring British Columbia Royalties.  If a unit does not exist in the drop-down Pick-lists, it can be created on the Production Units page.   

 

2. Product Classifications

The list of products existing in esi.manage  must be mapped to the specific definitions used by the royalty calculations. This mapping will let esi.manage know which production volumes to use for the calculations that are product specific.  If left unmapped, or a product doesn't exist in your data, the royalties can still be calculated.

To map a product, click and drag it from UNCLASSIFIED to the desired product.

 

3. Prices

A price forecast  of the Third Tier Oil Threshold Price, the Heavy Oil Threshold Price and the Selected Gas Price must all be created.  The drop down lists will include all available price forecasts that have the appropriate units:

 

4. PCOS Facilities

PCOS (Producer Cost of Service) allowance facilities ID, facilities Name and PCOS rate can be entered here.  Multiple entries can be copied and pasted in from a PCOS list.
