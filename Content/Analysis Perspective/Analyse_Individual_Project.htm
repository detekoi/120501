Options for Running Economic Analysis

There are several alternative workflows available to run an Economic Analysis against a Project, collection of Projects or Project Set(s). The choice of which workflow is dependent on your particular needs, and there are some underlying capabilities and relationships that must be understood.

  1. An Analysis Case is made up of Properties, Metrics and Project Sets.
    • The Project Sets can be temporarily ignored, applying the Analysis Case Properties and Metrics against a Project, collection of Projects, or Project Set(s) directly.
  2. The Properties of an Analysis Case create a consistent basis for evaluation:
    • There is a single Discounting Schedule, Price File, and Escalation File contained within an Analysis Case.
  3. However, multiple Analysis Cases can be run simultaneously to streamline the evaluation.
    • For example, to compare the same set of Projects under two scenarios (High and Low Price Decks, Escalation or Not, etc.), Analysis Cases can be created for each of the desired scenarios, with the same Project Set(s) in each.
    • Multiple Analysis cases can then be selected to be evaluated simultaneously. The highest level of roll up will be the Analysis Case, the same Projects will be present under each.

 

Analyze an Individual Project

When a Project is opened, the Analyze button allows you to run the Project within an existing Analysis Case.  This button is visible on every tab within the Project Editor, so you can choose to run an Analysis Case at any time.

 

The Select Analysis Case view will open, allowing you to choose an existing Analysis case whose Properties and Metrics will be used against the selected Project's data. If you are intending to perform this type of workflow repeatedly, choose the Remember Selection option.

Running an Analysis Case against a Project Set(s)

One of the Options with Project Sets is to apply esi.manage tools against them.

  1. From the Navigator, select (singly or multiple selections) the Project Set(s) to be evaluated.
  2. Right Click and select Analyze from the context menu.

 

The Select Analysis Case window will open, prompting you to select the Analysis Case to run against the project.

Select the Analysis Case and click OK to run the case.

Refer to Analysis Variance for more details on using esi.manage to compare the results of two Analysis Cases.
