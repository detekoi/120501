Analysis Variance

The Analysis Variance window is used when comparing the results of two different Analysis Cases (with the same or different Project Set) or two different Project Sets using the same Analysis Case.  This can be used to determine differences between any combination of variables or metrics that can be configured within an Analysis Case, such as Project Set, Discount Rate, Price Scenario, Reserve Consolidation, etc.  Please note that this window is best viewed when maximized, as the two windows with data in them have a specified size in order to display identical data and scroll simultaneously.

 

To run an Analysis Variance:

  1. Create an Analysis Case, and save it. (eg.Cash Flow)
  2. Change whatever variable you wish to see the effect of.  For example, If you wish to compare two different Project Sets, change the Project Set(s) selected on the third tab of the Analysis Case.
  3. Click "Save as" and save the new Analysis Case with a different name.
  4. Select both Analysis cases from the Navigator using Ctrl+click
  5. Right click on an Analysis Case and select "Analyze Selected Analysis Cases"
  6. Click OK if prompted to switch to the Analysis Perspective.

Analysis Variance on Project Sets

If you want to use the same Analysis Case to compare two different Project Sets, simply select the Project Sets in the Navigator and choose to Analyze Project Sets using your Analysis Case.

 

Analysis Mode

There are three Analysis modes available when you choose to run an Analysis Case.

  • Simple - This will sum all items being analyzed into a single top level line item which can be drilled into to expose any grouping within it (eg. Project Set, Fiscal model).
  • Variance - This requires more than one item to be selected to behave differently than a simple variance. This can be multiple Projects selected from the Project List, multiple Project Sets, or multiple Analysis Cases. It will provide the option to subtract any two of the following pieces of information from each other:
    • Total of Analysis Case
    • Each individual item selected (Project, Project Set, or Analysis Case)
    • Null value
  • Variance (+ base): This requires a Project Set to be set inside of the Analysis Case to behave differently than a Variance Analysis. The Project Set selected in the Analysis Case must also have projects that are set as Base Projects, and these function as the +base layer. This allows a user to run incremental analysis of individual items in addition to the base layer, using the same functionality as the variance analysis.

 

Resizing Columns

You can resize one or all columns in the Cashflow tab. Right-click on the header or table, and choose to either resize a column or all columns.

 

Show / Hide Zeros

There is an option in the top right corner of the Analysis Variance window, called "Hide Zeros". This will hide rows that are all zeros in both Analysis Cases. This can lead to different numbers of rows in different roll up levels.

For example:

  • Project set in Canada has data prior to the start date of my Analysis Cases (Jan 2013), so it will display a "<2013 - Jan" row with values in it.
  • Project set in USA has no data prior to the start of the Analysis Case (Jan 2013), so when hide zeros is selected it will no longer display a "<2013 - Jan" row
  • This can cause problems if exporting to Excel for the sake of further comparison

Note: the "Hide Zero" option will not hide columns that have no data within them, as these are un-editable within an Analysis Case (see Analysis Case Metrics).
