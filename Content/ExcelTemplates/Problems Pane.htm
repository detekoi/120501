Problems Pane

***THIS FEATURE IS BEING DEPRECATED. PLEASE MIGRATE ALL EXCEL DATA LOAD TEMPLATES TO THE NEW FORMAT***

The problems pane resides within the EDL underneath the EDL tabs. It can be expanded to fit the screen for a better view. The problems collected here are to help the user correctly create a template, and are not always errors. For example, whenever an attribute is selected to load tabular data on the attributes tab, an error message will appear stating that ‘Tabular Attribute “X” is not defined in any Data Series - it is just a reminder to map that attribute(s) on the series tab.

 

Some errors will appear during the template build while others will appear when attempting to import a broken template.

 

When applicable, the Source will provide some information to help solve the error.  Double clicking on the error message will take you to a location housing the error. The source will be in the form “X,Y”, where X is the location within the template, and Y is the location within the Excel document. In the example above, X is Production. By navigating to the attributes page and expanding projects>assets>production, the error will be attempting to load from here. This example explicitly states the production rate and is easy to navigate to, but some errors are not as clear. The Y is Column 1:B, which refers to the first column in the data series, in this case B. Once the location of the error is found, there are a couple ways to solve the problem. In the example above, the rate nbbl/d is not found. So the unit nbbl/d is not defined in esi.manage configuration and cannot enter unknown data. In this case, it would likely be a typo referring to mbbl/d. The ways to solve this would be to either:

  1. Directly change the Excel production rate to mbbl/d (assuming that is what is wanted)
  2. Use the translations tab to translate nbbl/d to mbbl/d
  3. Use the esi.manage Configuration Perspective to define a production rate as nbbl/d

For a template to load, all problems must be solved. Sometimes a resolved problem will not be cleared from the problems pane until the template has been reimported.
