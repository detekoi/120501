Expression Evaluation

***THIS FEATURE IS BEING DEPRECATED. PLEASE MIGRATE ALL EXCEL DATA LOAD TEMPLATES TO THE NEW FORMAT***

  • Expressions are not type-safe and will fail at time of evaluation if incorrect types are passed to functions or operators
  • Values are of two distinct types:
  • Scalar values, for example constants or values extracted from a single Excel cell
  • Vector values which are extracted via Time-Based Data Series
  • Rules of evaluation of operators (+, -, *, /)
  • Scalar and Scalar: operator works as expected
  • Scalar and Vector: operator is applied to each element of the vector, resulting in a vector
  • Vector and Vector: operator is applied to each pair of elements in the vectors consecutively, resulting in a vector whose size is the size of the longest vector in the input
  • The special dot (.) operator.
  • As a convenience the dot (.) operator may be used as an alternate form of calling functions with multiple arguments. So, for example, GET(SPLIT(“Hi_There”, “_”), 2) is equivalent to SPLIT(“Hi_There”, “_”).GET(2). The result in both cases is “There”.
