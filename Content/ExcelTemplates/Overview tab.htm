Overview Tab

The Overview tab inputs are discussed below; the mandatory inputs are Template Type and Original Excel File. Top down:

  • Owner: the user logged in to esi.manage who created the template
  • Created: the date the template was created
  • Template Name: the name of the template
  • Template Description: a description can be entered here to describe the template, or to instruct users of how to use the template. Any useful information can be stored here.
  • Template Type: A drop-down list to define the template
  • Add Production Curves: The add production curves type is used when there is an already existing project within esi.manage, and there is production information in Excel to be added to the project. This type requires that there is production already within the production tab of a project. This feature will overwrite the existing production.
  • Add Production Curves – Multiple: This feature is similar to the previous but instead has the capability to add multiple products and rates to an existing project. Again production must already exist within the projects and will be overwritten by the template.
  • Create Non-forecast: Non-forecast data has the ability to be imported into the Actuals tab of the Project Editor. Existing data for Production, Fixed Opex, or Revenue can be imported.
  • Create Non-forecast – Multiple: The multiple feature for Non-forecast imports is effectively the same as the Create Non-forecast template type.
  • Write Hierarchy Type: Write Hierarchy Type template allows Hierarchy Types and Hierarchy Elements to be written in to configuration.
  • Write Hierarchy Type – Multiple: The multiple feature for Write Hierarchy Type is effectively the same as the singular template.
  • Write Budget: The Write Budget template imports budgets into configuration. Budgets, Current Forecasts, and Archives can all be imported.
  • Write Budget – Multiple: Write Budget – Multiple works the same as Write Budget but can import more than one budget at a time.
  • Write Project: The write project function will create an entire project from scratch, loading its assets and measures from the provided Excel document. This is used when the Excel document only contains a single project.
  • Write Project – Multiple: This function is the same as Write Project, but it is used when an Excel document houses multiple projects to be loaded into esi.manage.
  • Write Type Curves: Used to write a single type curve from an Excel Workbook.
  • Write Type Curves – Multiple: Used to write multiple type curves from a single Excel Workbook.

 Template Options: Additional options to shape how the EDL loads data:

o Merge New Data With Existing: Available in Write Project and Write Project – Multiple Template Types. This feature will update a project with new data mapped in the EDL template.

o Trim Leading Zeros from Tabular Forecasts: When loading tabular data, leading zeros will be trimmed and start dates/onstream dates updated accordingly.

o Trim Trailing Zeroes from Tabular Forecasts: When loading tabular data, trailing zeroes will be trimmed.

 

  • Date Formats: In this text box, the format of the dates within Excel are specified. For example, if the dates in Excel are all in the format 2014/1 to describe January 2014 or similar, yyyy/MM would be typed into the text box. This works for any format, as well as multiple formats. If there are multiple formats, the | character can be used to separate them. The usable characters are as follows (Case sensitive):
  • y = year
  • M = month
  • d = day in month
  • D = day in year
  • q = quarter
  • Quarters will have a format such as:
  • qQyyyy for values in Excel in the form 2Q2014 to describe April 1, 2014
  • yyQTRq for values in Excel in the form 15Q1 to describe January 1, 2015
  • Repeat across worksheets: This check box can be enabled if the data is in the same format from worksheet to worksheet and the template is to load in the same way on each sheet. Checking this box enables the Number of Header Worksheets numeric box and the Worksheet Repeat Group Size numeric box and the three options are tied together. If this is enabled, it will cause errors if the Worksheet Repeat Group Size is zero.
  • Number of Header Worksheets: In this numeric box, the number of worksheets before the actual data starts can be specified. This value must be defined if Repeat across worksheets is defined. Its default is zero.
  • Worksheet Repeat Group Size: If there is repeating data, but multiple sheets define a project, the pattern can be specified here. For example, if sheet 1 contains production, sheet 2 for capex, then a new project starts on sheet 3 which has production, then capex on sheet 4, the group size can be specified as 2. The default is 0, but must be changed to specify the pattern while Repeat across worksheets is enabled.
  • Original Excel File: The name of the Excel document from which to base the EDL template.

To create a basic template, only the Template Type and Original Excel File are needed to get started. The rest of the features are to enhance the template and to address more complex needs.
