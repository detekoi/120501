EID Import Requirements

External Identifiers (EIDs) can be imported into esi.manage. An EID is the Identifier used in an External System (i.e., Accounting or Field Capture system). The Integration Perspective within esi.manage allows EIDs to be mapped to an esi.manage Asset.  Multiple EID's can be mapped to the same Asset, which provides a more convenient mechanism to bring external data (Field Estimates, Actuals, etc.) into esi.manage and have it associated against the appropriate Asset. Refer to Mapped External Identifiers for how to map Assets to EID’s.

Context Key and Value are optional fields, however they must be entered as a pair. An example of a Key and Value pair are: Region (key) and Alberta (value). You can include multiple pairs of these in a single line, but they must be entered as pairs, and must be unique for each pair. (i.e., you could not enter a second Region key after the first one in the same import line.)

The Context Key pairs are provided to simplify the process of matching EID's to assets. You must add the Context Key string (e.g. Region) into the esi.manage Configuration.

The Context Key must already exist in the esi.manage Configuration (Configuration > External Identifiers > External Context Keys)
