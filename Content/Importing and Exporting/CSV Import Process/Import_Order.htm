General Order for Imports

There are two 'classes' of Imports, Configuration and Operational Data.

Configuration Type data (listed below) is generally imported prior to the population of esi.manage projects, however, new information (i.e., External Identifiers) can be imported at any time.

  • Project Hierarchy: Hierarchy Elements (information that is used to populate a project's Hierarchy).
  • External Identifiers: An External Identifier is used by esi.manage to determine which Project to associate data with during an Import. An External Identifier is made up of two parts, the System the data comes from, and the Key that is used in that system to identify a specific entity.

Operational data (Forecast, Field Estimates, Actuals) can be imported into esi.manage, either to create (or update) Forecast data, or to bring in actual data to facilitate Tracking.

esi.manage supports internal csv Imports for the following types of Project Data.

  • CAPEX Header - Import the Expenditure Line items into Projects.
  • AFE Header - Import AFEs to be assigned to the Expenditures.
  • CAPEX Detail - Import specific CAPEX schedule information.
  • AFE Detail - Import specific AFE schedules to track Actual expenditures.
  • Production - Import Production (and optionally revenue) against specific assets for tracking purposes.
  • OPEX - Import OPEX against specific assets to track production costs.
