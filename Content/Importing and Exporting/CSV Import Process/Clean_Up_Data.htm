Clear Data

When importing Production, CAPEX (AFEs of CAPEX Details) or OPEX data, prior data can be cleared by selecting the Clear option before importing. This Form will be initialized based on the data that is found in the import file:

  1. The Start and End Dates will respectively be the earliest and latest dates found in the import file.
  2. The Types will also be highlighted based on the contents of the import file.

There are several considerations in regards to how the Clear function is applied, and how your import files are formulated. The three parts of the Clear function are combined together, i.e:

  1. All records of type Actuals or Field Estimate
  2. Whose start and end dates fall between Feb 1, 2015 and Feb 28, 2015
  3. With the Clear Matching check box selected, the clear is applied to Projects identified within the import file (via the System/Identifier pair).
  4. If the Clear Matching check box is not checked, then the resultant Clear will be based on steps 1 and 2.

 

Considerations for Preparation of Import Files

  1. If bringing in data from different systems (i.e., Two different Field Capture Systems which are to be imported independently), then ensure the Matching check box is selected otherwise the second import file will erase the results of the first import file.
  2. Care should be exercised when combining data of different Types (i.e., Field Estimates and Financial Actuals). It is highly probable that in this situation there will be a quite wide range of Dates, for example, Field Estimates from June and Financial Actuals from February. This would result in a Start Date of Feb 1, 2015, and an End Date of June 30, 2015. Unless the import file contains all of the Field Estimate data between February 1 and June 30, 2015, when the Clear is applied it will remove all Field Estimate Data between the specified dates.

 

 

If the Clear Button is not Pressed

Depending on the type of Import, the treatment of existing data is different if the Clear Button isn't pressed.

  1. For OPEX and Production, the new data will be blended into existing data (provided the dates for the imported records match the date range for the existing record).
  2. For CAPEX and AFE Details, the new data will overwrite the existing data (provided the dates for the imported records match the date range for the existing record).
  3. If the dates in the import file don't match the existing dates, an error message will be generated regarding Overlapping Dates and the import record will be rejected.
