AFE Header Import Requirements

This table defines the requirements for AFE Header imports.

 

Description: Which field is being imported

Format: What the required import format is

Validation Rule: If there are any special rules for the field to validate

 

Description

Format

Validation Rule

AFE Number

Text

maximum of 50 characters

Description

Text

maximum of 100 characters

Approval Date

Date

yyyy-MM-dd format

WI%

Decimal or Percentage

Either a decimal (.805) without the % symbol, or a Percentage (80.5%) with the % symbol

Gross Amount

Number

 

Manager

Text

Must match an existing Project Manager in esi.manage

Status

Text

Must match an existing AFE Status Abbreviation in esi.manage

Original Amount

Number

 

Supplemental Amount

Number

 

Revised Amount

Number

 

Currency

Text

Must match an existing Currency Unit Abbreviation in esi.manage

Project

Text

If supplied, must match an existing project name in the target budget.

 

Bolded fields are mandatory, the import will fail for that record if a mandatory field is missing.

Note that:

  1. If WI% left empty, esi.manage use a value of 100% to calculate the Net AFE Amount.
  2. Supplying a Project Name will place the AFE into the AFE Table in the identified Project, but it will not map the AFE to an Expenditure.
