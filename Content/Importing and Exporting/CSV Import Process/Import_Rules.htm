Import Rules

Each Import has a required format and order - see the Requirements topic of each individual Import types for specific details.

In general, there are several key aspects to the Import Process.

  1. Each of the Import Types has a prescribed format which must be followed.
  2. For each import Type there is a part of each import record which identifies the corresponding esi.manage record:
    • For CAPEX Headers and CAPEX Details it is the combination of Project, Line Number, Expenditure Type and Start Date.
    • For AFE Headers and AFE Details, it is the AFE Number (AFE Name).
    • For Production it is either the Project/Asset/UWI combination or the EID System/Identifier pair.
    • For OPEX it is the EID System/Identifier pair.
  3. CAPEX and AFE Details will not aggregate records within an import file, i.e.; two rows with the same Date periods and AFE Number will not aggregate together.
  4. Production and OPEX will aggregate records, i.e.; two rows with the same Date periods and Production Segment will add together.
  5. The Clearing step of the import process may or may not be required, depending on your desired outcome.
    • CAPEX and AFE information (Headers or Details) will overwrite the data for the corresponding esi.manage record.
    • Production and OPEX information (Details) will 'blend' the new information into the existing data.
    • Production Forecast import will replace ALL forecast information in the associated project.
  6. Overlapping Dates will generate Import Errors.
    • For Import Types which aggregate (Production and OPEX) two records which would map to the same Project/Asset/Product but have overlapping dates (i.e., the End Date of one record is between the Start and End Date of the other record) will generate an Import Error.
    • If Clearing is bypassed, an Import record which has an overlapping date (described in 1) with an existing esi.manage record will generate an Import Error.

 

Dates must be entered in Excel in English UK (yyyy-mm-dd) format. Excel may try and auto-format them to another date style. Please follow these steps to avoid this:

  1. Open the CSV in Excel.
  2. Select a Date column.
  3. Right click and select "Format Cells"
  4. Format Date
  5. Choose "English UK" from the drop-down list.
  6. The yyyy-mm-dd format will be there.

 

If you are receiving an "Invalid date" error on your import, this is likely the cause and solution.
