Import Process

Importing data into esi.manage utilizes an import Wizard. Each screens of the wizard has a matching topic in this Help File. The general process is illustrated below:

Please refer to the specific import types found in this file for the process on individual areas.

 

  1. Access the Wizard by selecting File / Import Data.
  2. Select the Import Type (AFE Details, AFE Headers, CAPEX Details, CAPEX Headers, OPEX, Production, External Identifiers, or Project Hierarchy).
  3. Select the Budget the data is to be imported into (note that the Budget selection step is skipped for Hierarchy and EID imports as configuration information applies to your entire esi.manage Database).
  4. Choose the File to be imported, specify whether the First Line is a header, and Validate the selected File (i.e., the correct number of columns, columns have the correct data types).
    • If there are validation errors, the import can be cancelled or can Proceed with Valid Records only.
  5. Optionally Clear Data. The import file is analyzed for the data types (Field Estimates, Forecast, Actuals, etc.), and the earliest Start Date and Latest End Date.
    • The Clear Data page is initialized to those Types and Dates. The Clear Data function will remove existing records in the database that match the type and fall within the Dates.
    • Optionally select the Matching option, in which case the Clear (using the Types and Dates as described above) is limited to only those Projects found in the Import File (either through the mapped EID, or an identified project).
  6. Import the Data into esi.manage.
  7. Press Finish which will generate a log file with the import results (number of records, valid records, errors, etc.), and esi.manage will prompt to be restarted.
