Common Problems with CSV Files

Excel's formatting of CSV files can be strange at best to downright confusing at worst. Perfectly fine import records will sometimes return an error at the validation stage "Invalid Record Length" as a result of this. To diagnose this problem, always look at the record in Notepad or Wordpad, not Excel. Formatting errors will appear much clearer there.

In this example, we are importing some hierarchy items, but leaving the fourth column blank, as it is optional.

Correct:

  • Country, Canada, Can,
  • Country, US, US,

Notice the extra comma at the end of each, denoting that there is no value in the final column.

Incorrect:

  • Country, Canada, Can
  • Country, US, US

In this example, Excel has not properly formatted the CSV to reflect the empty cell at the end of the table. By dropping the comma, the importer is unaware there is data for the fourth import column and thus the import fails.
