External Prices Import File

The basic structure of the source file (csv form) is:

1. Definition of a Price Forecast:

a. Hierarchy Attribute Code.

b. Project.

c. The Forecast name.

d. Either the Base Forecast name (if the Forecast is related to another Forecast), or

e. Currency and Units if the Forecast is a Root Forecast.

f. Comments

2. Identification of the Price Scenario the following data will be read into.

3. Periodic Data

a. Date and Price for Root Forecasts.

b. Date, Scale and Offset for all other Forecasts.

4. The first column of any row must contain one of the following entries to identify how to interpret the rest of the data in the row:

a. Null (empty) – this row is ignored. This allows you to put in a Header row to guide you in the structure of the data. Header rows can be entered anywhere in the file.

b. Forecast – this indicates that the row identifies an existing Price Forecast (if data is being updated), or creates a new Price Forecast.

c. Scenario – this indicates which Scenario the following data is to be read into.

d. Price – indicates that the data pertains to a Root Forecast.

e. Diff – indicates that the data pertains to Forecast that has a Base Forecast.

 

The file is processed procedurally; i.e:

1. The first row identifies the Forecast. All subsequent rows apply to that Forecast until another Forecast row is encountered.

2. The next row identifies the Scenario. All subsequent rows apply the Forecast / Scenario combination.

3. Data rows (either Price or Diff). All subsequent rows are read into the Details for the Forecast / Scenario combination.

a. The second column (Diff or Price is in the first column) specifies the Date.

b. The Date can either be in YYYY-MM form (for monthly data), or YYYY for yearly data.

4. The “next” section can be identified by either the Forecast or Scenario keyword.

a. If Forecast, the procedure begins again at item 1.

b. If Scenario is the next Keyword, the procedure begins at item 2; i.e.; the subsequent data rows will apply to the Forecast from item 1, but the new Scenario.
