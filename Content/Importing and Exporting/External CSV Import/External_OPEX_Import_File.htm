Command Interpretation

How the Command (Update or Delete) is applied is dependent on what OPEX entries already exist for the Project/Asset. If there is some degree of matching (i.e., for the Project and Asset, there is an OPEX entry of the same OPEX Type).

 

When there is no Existing OPEX:

  • Update will add a new entry

 

When there is Existing OPEX:

  • Update will replace an existing entry (if there is an exact match in Dates)
  • Update will split an existing entry (if there is not an Exact match in Dates and Amount), and create a new Entry
  • Update will also combine lines if it is determined that there are several entries that can be represented by a single line.

 

  1. If the new entry falls within the time period of the existing entry (for example the existing entry is from Jan 1 2010 to Dec 31, 2020, and the new entry is Jan 1 2011 to Dec 31 2011)
    • the existing entry will be broken into two lines - Jan 1 2010 to Dec 31 2010 and Jan 1 2012 to Dec 31 2020.
    • the new entry will be introduced (Jan 1 2011 to Dec 31 2011)
  2. If the new entry (Update only) falls exactly within two existing entries:
    • i.e. the Update record is Jan 1 2011 to Dec 31 2011 and
    • the existing records are Jan 1 2010 to Dec 31 2010. and Jan 1 2012 to Dec 31 2020,
    • and the OPEX Amount for all three are identical
    • Update will combine all three into a single row from Jan 1 2010 to Dec 31 2020.
