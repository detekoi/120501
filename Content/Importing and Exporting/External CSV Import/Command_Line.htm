Running Import.exe from the Command Line

Import.exe can be invoked entirely from the command line, using various switches.

>import [OPTION]... [FILE]

eg.: >import -t opex:text/csv -v-h -d localhost:1432/esimanage -o rejects.csv opex.csv

The options that are Yes/No (verify, header, usenamedpipes) don't take an argument on the command line - their presence is interpreted as a Yes.

If your database has spaces in the name (i.e.,  Smart Oil and Gas), quote the database name as part of the -d (-d localhost:1432/"Smart Oil and Gas"

Note:

  1. When SERVER is unspecified localhost will be assumed. When PORT is unspecified 1433 will be assumed.
  2. Each of the Switches can be represented in a Properties File instead (i.e., Header=Y). Note that some options (database for example) expand to 3 lines in the Properties File.
  3. Where the same option is specified in the File and the command line the value from the command line will be preferred.
