Using a Properties File with Import.exe

The various settings for Import.exe can be defined in a Properties (text) file. This file can then be used as part of the calling argument.

For example:

 

> import -p properties.txt

 

In this case, all of the required parameters for import.exe will need to be included in the file Properties.txt.

A sample file would look like:

 

Type=opex:text/csv

Header=YES

Verify=YES

OutFile=rejects.csv

Server=127.0.0.1

DatabaseName=Smart Oil and Gas

Port=1432

File=Opex_2010_09.csv

 

Note:

  1. The File Syntax (OutFile for example) is case sensitive.
  2. When SERVER is unspecified localhost will be assumed. When PORT is unspecified 1433 will be assumed.
  3. Where the same option is specified in the Properties File and the command line, the value from the command line will be preferred.
