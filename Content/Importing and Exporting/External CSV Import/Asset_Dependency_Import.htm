Asset Dependency Import

Import Properties

The various settings for Import.exe can be defined in a Properties (text) file. This file can then be used as part of the calling argument.

For example: > import -p properties.txt

In this case, all of the required parameters for import.exe will need to be included in the file Properties.txt.

CSV Import Format

N

name of the asset to update

Y

name of the parent asset to assign to the asset being updated

 

depends on asset offset

Y

offset from the start of the parent asset

 

 

Notes:

If the 'depends on asset' column is set to null it will clear an existing dependency for an asset.

If the 'depends on asset' column is not null the parent asset must be within the same budget as the asset being updated and the 'depends on asset offset' must be specified.
