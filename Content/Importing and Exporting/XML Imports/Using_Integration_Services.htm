Using the Integration Service

esi.manage enables external processes or applications to Create or Update select data in the esi.manage database via Web Services interfaces. Data can either be exported from, or imported into, esi.manage. In most situations, this capability will be used as part of an automated process (Web Service interfaces invoked automatically, using machine generated import files or consuming esi.manage export results).

Generally the Integration Service will be run on a specific machine, which may or may not be a machine/server which runs esi.manage installation. However, the esi.manage.integration.service folder must be copied to the machine where the service is to be run. The esi.manage.integration.service folder contains the following:

 

In order to access the Web Services interface, the user would need to run the start.cmd script inside the esi.manage.integration.service folder. This script invokes a series of events that opens the communication channel that enables the user to access the Web Services interface, as well as the access to the esi.manage database via the Web Service interface, where data will be exported from or imported into.

To properly establish the communication channel to enable user access to Web Services, the network settings must be configured correctly. The network settings can be changed in the esi.manage.xml file located in the esi.manage.integration.service\conf\Catalina\localhost folder.

 

Open the XML file, and ensure the highlighted URL strings have the format jdbc:jtds:sqlserver://[esi.manage database server name]:[esi.manage database server port]/esi.manage (in the example, "localhost" is the name of the server where the esi.manage database is stored, and "1433" is the communication port for the server).

 

Aside from the URL strings, there are other parameters of the integration service that can be configured in the XML file:

Parameter Description
name = "jdbc/[name of the esi.manage database]"

name of esi.manage SQL database

username = [SQL authentication username]

only required if Database Authentication is used in esi.manage

password = [SQL authentication password]

only required if Database Authentication is used in esi.manage

connectionProperties = [Name of the instance]

examples: SQL2005, SQL2008, SQL2008R2
namedPipe = [true or false] turns the Named Pipe option on and off (same as the Database Settings in esi.manage)

 

After completing the network settings configuration, initiate the Web Services access procedure by running the esi.manage.integration.service\start.cmd script. This opens a command prompt console. When the last line reads "INFO: Server startup in xxxxx ms", the communication channel has established, and the user can proceed to access the Web Services interface.

 

To view the Schemas or sample Import Files in the Web Services interface, open a web browser (Internet Explorer, FireFox, Chrome, etc.) and go to the link http://localhost:8080/esi.manage, the following web page will successfully load if the communication channel has been properly established.
