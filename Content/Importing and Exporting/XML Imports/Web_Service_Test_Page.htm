Web Service Test Page

Once the Web Service interface has been configured and started, a Test Page can be accessed (follow the syntax as illustrated in the image below - i.e.; http://localhost:8080/esi.manage) to view the Schemas and Sample import files, as well as using existing XML files to import and export data to and from esi.manage database.

 

Within the Web Service interface, the user can choose to perform the following tasks:

Projects:

  • Export Projects into an XML document
  • Import Projects from an XML document
  • Import Project Attributes from an XML document
  • Shift Project start dates with an XML document
  • Export metadata of Projects into an XML document

Measures:

  • Invoke Measures export using an XML document
  • Export changes in Expenditure items since a given date
  • Import AFE data from an XML document
  • Import CAPEX data from an XML document
  • Import Production from with an XML document
  • Import OPEX data from a CSV document

Configuration:

  • Import Hierarchy elements from an XML document
  • Import External Identifier (EID) from an XML document

Capital Planning:

  • Export data from esi.manage to the Schlumberger Merak Capital Planning application
  • Invoke the importing of data from the Schlumberger Merak Capital Planning application using an XML document
  • Invoke the shifting of data from the Schlumberger Merak Capital Planning application using an XML document

 

Most of the importing interfaces provides access to a similar page (below is the Production Import page as an example):

  • Request XSD - provides access to the particular Import.xsd file
  • Import Schema - shows the Schema Documentation for the particular import.
  • Sample Import - provides access to documented Sample Import file.
  • Response - shows the Schema Documentation file regarding the Response to a given import.

 

A particularly useful feature is the Sample Import, which outputs a sample XML import file that follows the appropriate Schema for the particular import, from which the user can use as a template to generate a custom XML import file. In general, the Import Services would not be utilized via this Test Page. However, you can use this utility to test out your import file structure. Once you have created an Import File (following the prescribed schema), you can paste it into the XML to Send window and choose the Send Via Post button. This will pass the Pasted XML text to the import routine.

Note: the Project Export functionality is also accessible via the Integration Service. There is a set of documentation produced on that page (Project Export) which describes the calling syntax for this interface.
