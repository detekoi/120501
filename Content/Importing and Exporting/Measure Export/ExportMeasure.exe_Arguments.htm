Explanation of an example line

Using -c "Gross CAPEX" -m capex -e "Unrisked Gross" -t Blended -a 2010-08-31 -M -U kCAD as an example:

  1.  -c indicates that a column is required, which will be labelled "Gross CAPEX"
  2. -m capex identifies the Measure
  3. -e "Unrisked Gross"  specifies the Definition (esi.manage Configuration) for this column
  4. -t Blended specifies the Type
  5. -a 2010-08-31 specifies the Last Actuals date to use for Blending
  6. -M indicates that the Match Forecast option is to be used for Blending
  7. -U kCAD specifies the desired output unit for the Column

The same measure can be identified several times, using different Definitions and Types.

The specified Grouping will create appropriate columns in the output file, in the order that they are specified. The Measures will be written against each of the Grouping combinations that they are valid for - this can result in the same data showing up in a given column more than one time (depending on which combinations of the Group it is applicable to).

 

For more information, invoke the help file by putting it in a text file, or calling it directly into the command prompt using exportMeasure.exe --help
