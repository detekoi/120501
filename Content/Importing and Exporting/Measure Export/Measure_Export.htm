Measure Export

exportMeasure.exe (found in the esi.manage.export.measure folder or your installation) allows calculated data (CAPEX, OPEX, Production, Reserves, Revenue) to be exported in a csv format.

The creation of this new application allows the export to be invoked by an external application, as part of a pre-defined workflow. While this is similar to the XML export routines, it deals only with calculated results. As it is in a csv format, it can readily be consumed by Excel for the generation of pivot tables.

This application is documented - entering:

 

> exportMeasure --help  (from the command line in the folder - click here if you need step by step instructions for running the command line)

 

exportMeasure --help will bring up a description of the parameters that can be included in the Measure Export along with their options.

In addition to exported calculated results, Grouping can also be requested. There are a number of pre-defined Groups, as well as the option to specify Hierarchy elements to be included.

Note: a text file (i.e.; ExportProperties.txt) can be created which contains the desired results and grouping and used as part of the calling syntax.

 

A Parameters File to configure a MeasureExport is broken into 4 "sections"

  1. Configuration:
    1. Database connection: Server, Database Name, login information, etc.
    2. Header - a switch to write out Header information for each output column
    3. Output File - name of the output file, whether to Append to the file (Replace is the default behaviour)
    4. Pivot - a switch to output all calculated results into a single column, and insert a Measure Definition column to aid in pivoting
  2. Grouping:
    1. Pre-defined Groups - there is a wide collection of built-in Grouping options; Project and Asset Name, CAPEX/OPEX Category and Type, Project Code and Jurisdiction, etc.
    2. Custom Groups - Project Hierarchy, and Budget, Project or Asset Attributes
    3. Named Groups - AFE Amount, Estimate at Completion
  3. Measures - each output is defined by a number of parameters
    1. Column Name - the header for the output column
    2. Measure - Activity, Expenditure, OPEX, Production, Reserves, Resource Cost or Revenue
    3. Definition - one of the appropriate options found in the esi.manage Configuration (Unrisked Gross, Net Risked Sales, etc.)
    4. Type - one of the appropriate options found in the esi.manage Configuration (Forecast, Actuals, Blended, etc.)
    5. Basis - an alternative approach to represent the Definition, using Gross, Net or Net_Net in combination with Modifiers
    6. Units - the desired output unit
    7. Filtering - calculations can be 'filtered' to a desired subset (i.e., a specific Product or combination of Products)
    8. Modifiers - based on the selected Measure (for Revenue a Price File must be identified), Definition (NetRiskedRoyalty requires a Royalty Scheme) or Type (Blended requires a Last Actuals Date and a Blending Mode)
  4. Calculation Period and Source
    1. Start Date for calculations
    2. Calculation Period - how many calculation periods of what periodicity (Daily, Monthly, Quarterly, Annual)
    3. Source - an existing Project Set in esi.manage that provides the Projects to be calculated

 

Notes:

  1. The Syntax identifiers, Options and Samples are CASE SENSITIVE.
    1. When the Description refers to a Configuration table within esi.manage, the String must match the Descriptor (Name or Description) that appears within your company's Configuration.
  2. In the following Samples, some entries will have Quotes (") around them. If there are Spaces within the sample, the entire string must be quoted.
  3. The Database connection information should follow how your esi.manage database connection is Configured.
  4. Grouping options (particularly with respect to multiple grouping options) defines the pattern of the calculated data (Measures).
    • Consider relationships when specifying the sequence of the grouping options - Project.Name should likely come before Asset.Name.
  5. The Grouping options, combined with the identified measures, imply an AND relationship. For example:
    • The selected Grouping options are Project.Name, Asset.Name, CAPEX.Type and Product.Name, while the selected Measures are Expenditure and Production.
    • The output will show the same pattern of Project Name and Asset Name twice. For the first section, CAPEX Type will be populated as will the Date and Expenditure columns.
    • The second section will have same Project Name and Asset Name values, but only the Product Name, Date and Production columns populated.
  6. If the CAPEX Type and Product Name grouping options are not included, there will only be one section containing Project Name, Asset Name, Dates with the Expenditure and Production columns populated.
  7. Measure Definition (-e) and Basis (-b) with Modifiers (-R, -S, etc.) are mutually exclusive.
    • Definitions are based on the available Definitions within the Configuration and are pre-defined in terms of Modifiers.
    • The same results can be obtained by using the Basis and Modifier options, however, both cannot be defined for a single Measure calculation (-c).
  8. Items marked with an asterisk * in the description have options that can be user configured.  If these options have been renamed, then the new name must be used in place of the default.
  9. If using the "-t" flag to identify the measure type (eg. Forecast), it will use curtailment (as it is defined in Production Types).  If you do not wish to use all the settings that come included by using the "-t" flag, you must use the other modifiers available to describe what is wanted out of the measure export.
  10. If no consolidation setting is specified, by default the "Expected Value" consolidation is used.

If wanted, add other modifiers to option B (eg. risk) to return the desired value.

  1. -c "Forecast Gross Unrisked Raw" -m PRODUCTION -e "Gross Unrisked Raw" -t Forecast -U Oil:boe -X Default --consolidation "Reserve (Cumulative):1P" --pricescenario "Current Price"
  2. -c "Forecast Gross Unrisked Raw" -m PRODUCTION -b GROSS  -v FORECAST  -U Oil:boe --consolidation "Reserve (Cumulative):1P" --pricescenario "Current Price"

"exportMeasure --help" from command line.

Below is a copy of the file produced when "exportMeasure --help" is run from the command line and provides all the measure export expressions.

  • Click on start menu>All programs>Accessories>Command Prompt or type cmd in the search programs and files box.
  • From the command prompt you need to navigate to the exportMeasure --help. To do this type the 'cd ' (change directory command) followed by the path to the 'esi.manage.export.measure' folder in the esi.manage installation files, followed by "exportMeasure --help"

cd \

to navigate to your systems root directory (c drive)

cd esi.manage\3.22\product\esi.manage.export

to navigate to the Measure export folder within the esi.manage installation files ie here the measure export folder is in the following directory:

C:\esi.manage\3.22\product\esi.manage.export.measure

Once in the measure export folder type

exportMeasure --help or type --help > help.txt to have the text saved directly to a text file which will appear in the esi.manage.export.measure folder.

 

To copy the contents from the command prompt

  1. Right-click the title bar of the command prompt window, point to Edit, and then click Mark.
  2. Click the beginning of the text you want to copy.
  3. Press and hold down the SHIFT key, and then click the end of the text you want to copy (or you can click and drag the cursor to select the text).
  4. Right-click the title bar, point to Edit, and then click Copy.
  5. Paste the text into your new location.
