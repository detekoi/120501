Import & Export Overview

esi.manage provides a number of approaches for Data Import and Export, the choice of which is dependant on the particular requirement. There are two data formats which are supported, CSV (comma separated or comma delimited) which is a flat file format, or XML.

Importing Data via the CSV Import Process is always user driven from inside of esi.manage. There is an Import Wizard that guides you through the necessary steps; selecting the Data Type (Capital, Operational, etc), Selecting and Validating the file to be Imported, potentially Clearing existing data, and then finally Importing the data into esi.manage.

esi.manage contains a Schema that represents Project data in a structured format that is typically (although a file can be constructed manually) generated or consumed by an external application. This schema is used for Exports (the Export Job), as well as for select imports. Inside of esi.manage this form is available for the Project Hierarchy

In addition, esi.manage provides an Integration Service developed using Web Service protocols that enables an External Application (typically part of a larger overall business process) to invoked the Export and Import interfaces (using the XML schema) automatically - potentially based on a 'business process event' or 'time trigger'. An external application (Export.exe) is also part of esi.manage which allows Export Jobs to be invoked from a command line.

An additional Import approach also exists, currently for OPEX data only, as well as some configuration data (Prices & Currency information). This uses an external executable (import.exe) that is found in the esi.manage.imports.cmd folder of your installation.

There also exists an external Export approach, primarily for calculated data. This uses an external executable (exportMeasure.exe) that is found in the esi.manage.export.measure folder of your installation.

The final means if importing data into esi.manage exists as a feature called the Excel Data Loader (EDL). Using this tool, users can map out existing spreadsheets using templates, and import nearly every piece of data contained within an esi.manage project. The import template mapping file that was created can then be reused to import more excel files with identical formatting.
